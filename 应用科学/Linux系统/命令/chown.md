
chown：修改每个 <文件> 的所有者和/或组为给定的 <所有者> 和/或 <组>。  


```bash
用法：chown [选项]... [所有者][:[组]] 文件...  
　或：chown [选项]... --reference=参考文件 文件...  
使用 --reference 时，将每个 <文件> 的所有者和组修改为和 <参考文件> 一样。  
  
 -c, --changes  
        类似 verbose 选项，但仅在做出修改时进行报告  
 -f, --silent, --quiet  
        不显示大多数错误消息  
 -v, --verbose  
        为每个处理的文件输出一条诊断信息  
     --dereference  
        影响每个符号链接指向的文件（这是默认行为），而非  
        符号链接本身  
 -h, --no-dereference  
        影响符号链接本身，而非它指向的文件  
        仅当系统支持更改符号链接的所有权时，此选项才有用  
     --from=当前所有者:当前组  
        仅在文件的当前所有者和/或组和这里指定的一致时，才更改  
        它的所有权。其中一个可以省略，表示不对被省略的属性  
        作出要求  
     --no-preserve-root  
        不特殊对待 "/"（默认行为）  
     --preserve-root  
        不允许在 "/" 上递归操作  
     --reference=参考文件  
        使用 <参考文件> 的所有者和组，而非指定一个值。  
        如果 <参考文件> 是符号链接，则总是解引用它。  
 -R, --recursive  
        递归操作文件和目录  
  
指定了 -R 选项时，以下选项设置如何遍历目录层次。  
如果您指定了多于一个选项，那么只有最后一个会生效。默认值是 -P。  
  
 -H  
        如果命令行参数是一个指向目录的符号链接，则对其进行遍历  
 -L  
        遍历每一个遇到的指向目录的符号链接  
 -P  
        不遍历任何符号链接
```
