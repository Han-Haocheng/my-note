# 函数


函数function：组成C程序的基本单位，模块化程序设计的最小单位、基石
- 分类：库函数(标准函数)、自定义函数


模块化程序设计方法：自顶向下、逐步分解、分而治之，即将一个大的程序按功能分解成为一些功能单一、结构清晰、接口简单、容易理解的功能模块。


库函数(标准函数)：函数声明一般是放在系统的include目录下以.h为后缀的头文件中，如在程序中要用到某个库函数，必须在调用该函数之前用`#include<头文件名>`命令将库函数信息包含到本程序中。

```C
#include<math.h> 
abs(),sin(),cos()

#include<string.h> 
strcmp(),strcpy()

#include<stdio.h> 
fopen(),fclose()

#include<stdlib.h> 
malloc(),free()
```

自定义函数：模块化程序设计的需要，按要求设计的


## 函数格式

### 函数定义格式
```c
函数返回值类型 函数名(形参说明表)
{
	<函数语句体>
}
```



函数首部（函数头）
- 函数返回值类型：函数最后需要传出的结果的类型
- 函数名：一个合法的用户自定义标识符，见名知义
- 形式参数表：列出所需要提供的参数及对应类型（不共享），代表使用这个函数必须提供的初始数据，是一系列用逗号分开的形参变量说明
	- 无参函数：形参列表缺省

> [!warning] 注意
> 参数列表不能直接写成：`int x, y, z`


函数体：函数代码实现部分

```c
{
/* 说明性语句 */
/* 执行语句 */
// 返回语句
}
```


- 返回值类型是非void类型：则函数体内中要使用`return 表达式;`
	- 表达式类型和返回值类型一致
- 若返回值类型为void：返回点使用`return ;`语句
	- void 返回类型可以没有return语句，执行到函数体的右大括号结束

> [!warning] 注意
> 函数体内不可以出现其他函数的定义，即函数不能嵌套定义。


举例

```c
int sum(int x,int y,int c) //形参变量类型都是int型的x, y, z
{
	return x+y+c;
}

int abs_sum(int m,int n){
	m=m<0?-m:m;
	n=n<0?-n:n;
	return m+n;
}
```

### 函数声明格式


函数声明：函数应当先定义后调用，但如果出现了先调用后定义的情况，则必须在调用之前作原型声明。


先声明，再调用，后定义

```c
/*example7_1.C */
#include <stdio.h>
```

## 函数调用

### 函数调用过程

函数调用的完整过程
1) 转向：调用函数时，流程从主调函数转向被调用函数
2) 传参：如果调用有形参的函数，首先要用实际参数初始化形式参数；如果是无参函数，则不存在参数传递工作
3) 执行：执行被调用函数的函数体直到返回处
4) 返回：执行到被调用函数的return语句处（如果无此语句，则是到函数体的右大括号处），执行流程返回到刚才主调函数调用此函数处
5) 继续：继续执行主调函数后续语句至程序完

### 实参与形参

形参：在函数定义时不占内存空间，只有被调用时才占内存，且用实参的值初始化它。调用结束不再占用空间，等到下次被调用时重新再占用空间

实参：直接以表达式的形式给出，其个数与形参个数必须完全相同，对应数据类型最好完全一致


形参与实参的关系：形式参数决定了实际参数的个数和对应类型，按从左到右的顺序——匹配对应


| 参数性质  | 形式参数                       | 实际参数                |
| ----- | -------------------------- | ------------------- |
| 出现位置  | 函数定义时在首部形式参数表中             | 主调用函数中函数调用时提供       |
| 本质    | 变量值                        | 表达式（常量、变量为其特殊形式）    |
| 表达的含义 | 调用该函数的入口参数通用要求，需要几个、什么类型的值 | 某一次特定调用时所提供的实际入口参数值 |


### 传值调用和地址调用

形参传递（传值调用）：实参的值在函数调用后不会发生变化


```c
/*example7_2.C 函数传值调用*/
#include <stdio.h>
void exchange(int a,int b){
    int temp;
    printf("3--- a=%d,b=%d\n",a,b); //10 20.
    temp=a;
    a=b;
    b=temp;
    printf("4--- a=%d,b=%d\n",a,b); //20. 10.
}

int main(){
    int x=10,y=20;
    printf("1--- x=%d,y=%d\n",x,y); //10 20.
    exchange(x,y); // 函数调用
    printf("2--- x=%d,y=%d\n",x,y);
    return 0;
}
```

实参传递（传指针调用）：实参值在函数调用后会发生修改
```c
/*example7_2.C 函数传值调用*/
#include <stdio.h>
void exchange(int* a,int* b){
    int temp;
    printf("3--- a=%d,b=%d\n",*a,*b); //10 20.
    temp=*a;
    *a=*b;
    *b=temp;
    printf("4--- a=%d,b=%d\n",*a,*b); //20. 10.
}

int main(){
    int x=10,y=20;
    printf("1--- x=%d,y=%d\n",x,y); //10 20.
    exchange(&x,&y); // 函数调用
    printf("2--- x=%d,y=%d\n",x,y);
    return 0;
}
```

### 数组作为参数


数组元素作为函数的参数：数组元素的作用等同于简单变量
- 如果实参为数组元素，则形参必须是简单变量
- 函数的调用属于传值调用方式(实参的值单向传递给形参)

数组名作为实参：这种函数的调用方法
- 如果实参为数组名，则形参不允许为简单变量
- 函数的调用属于传址调用方式(实参与形参公用内存单元)，函数中形参值的改变会引起实参值的改变


- 形参数组和实参数组共占用一段内存单元
- 当形参的值发生变化时，实参的值也随之发生变化。
- 形参为数组时，可以不定义长度。


```c

	//...
}
```
### 变量作用域



![[应用科学/c&c++程序/01-c&c++基础/2026-01-25_212159.svg]]

变量的作用域：变量名应该在程序的哪一部分可以直接引用
- 变量的作用域取决于变量定义的位置

变量的生命周期：变量所占用的空间从创建到撤消的这段时间
- 变量的生命周期取决于变量的存储类型
- C语言中变量存储类型：分别用关键字auto、static、extern 、register

二者的关系：一个变量如果不在其生命周期，肯定无作用域可言；如果在生命周期，也未必一直起作用，即使起作用也是在特定的范围内。


#### 变量定义的位置
- 函数体外，外部（全局）变量，定义点到程序结束，但去掉同名局部量范围
- 函数体内，局部变量，作用范围为本函数体内
- 函数语句块内，局部变量，作用范围仅限于本语句块内

#### 变量的存储类型
存储类型：编译器为变量分配内存的方式，决定变量的生存期
- 在内存中，供用户使用的存储区由程序区、静态存储区和动态存储区3部分组成

```c
[存储类型关键字] 变量数据类型 变量名;
```

存储类型关键字
- `auto`：自动存储类型标识，为缺省存储类型，变量存储在内存的动态存储区；
- `static`：静态存储类型标识，表明该变量位于静态存储区；
- `extern`：外部存储类型标识，仅用于变量声明中，表示该变量在本文件的后面给出定义，或该变量来自于同一个程序的另一个文件中，变量存储在静态存储器。
- `register`：寄存器存储类型标识，表明该变量位于CPU的寄存器区域，加快了访问

默认存储类型
- 其中缺省的存储类型关键字为auto
- 局部变量的存储类型默认值为auto型。
- 全局变量的存储类型默认值为extern型。

> [!warning] 注意
> - 一般情况下，常用auto型register型定义局部变量
> - static型既可作为局部变量，又可作为全局变量
> - static局部变量：局部变量的值将被保留，若定义时没有赋初值，则系统会自动为其赋0值
> - static全局变量：其有效范围为其所在的源文件，其他源文件不能使用
> 

### 递归函数的设计和调用


函数递归：在定义函数时直接或间接调用自己
- 直接递归：A-->A
- 间接递归：A-->B-->A

```c
void func(){
	//...
	func();
	//...
}

```

递归求解条件
- 问题的解可以分解为几个子问题的解
- 问题与分解之后的子问题，除了数据规模不同，求解思路完全一样
- 存在递归终止条件

一些问题适合用递归方法求解：阶乘问题、乘幂问题、数值转换问题、汉诺塔问题等


- “递”即递推：表示将复杂的原问题转化为同类型同方法的简单问题的过程；
- “归”即回归：表示从递归调用终止处依次一层层向前返回处理结果。
