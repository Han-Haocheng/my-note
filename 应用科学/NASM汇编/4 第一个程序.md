
# 第一个程序

## 一个源程序从写出到执行的过程

1. 编写
2. 编译链接：masm.exe和link.exe
3. 执行可执行文件(程序、数据)

## 源程序

源程序：包含汇编指令、伪指令，其中程序中必须包含至少一个代码段
- 标号：一个标号指代了一个地址

```asm
assume cs:codesg

codesg segment
start:
  mov ax,0123H
  mov bx,0456H
  add ax,bx
  add ax,ax

  mov ax.4c00H
  int 21H
codesg ends

end
```


### 伪指令

**伪指令**：由编译器执行的不可被CPU识别和执行的指令

`segment-ends`：定义一个段
- `<段名> segment`：段的开始
- `<段名> ends`：段的结束

`end`：标志程序结束

`assume`：将段寄存器和程序中的段进行关联


任务：编程运算$2^3$
- 定义一个段
- 实现处理任务
- 程序结束
- 段与段寄存器关联

```asm
assume cs:abc

abc segment
start:
	mov ax,2
	add ax,ax
	add ax,ax
abc ends

end
```

### DOS中的程序运行

DOS系统运行：是一个单任务操作系统
- 一个程序p2在可执行文件中，则必须有一个正在运行的程序P1，将P2从可执行文件中加载入内存后，将CPU的控制权交给p2，p2才能得以运行。p2开始运行后，P1暂停运行
- 当P2运行完成后将CPU的控制权交还给P1，P1继续运行

### 程序返回

程序返回：程序运行结束后，将CPU的控制权交还给使这个程序运行的程序的过程
- 在程序的末尾添加返回的程序段
	- `mov ax,4C00H`
	- `int 21H`

### 创建源程序

- 创建文件1.asm
- 写入以下内容

```asm
assume cs:abc

abc segment
  mov ax,0002H
  add ax,ax
  add ax,ax
  mov ax,4C00H
  int 0021H
abc ends
end
```

- 保存文件

### 编译链接

- 在命令行使用`masm 1.asm`
- 在命令行执行`link 1.obj`

> [!tip]
> 
> - 跳过询问环节：在文件后添加`;`
> - 同时编译链接：使用`ML.EXE 1.asm`
> 

## 程序执行

### 可执行文件中的程序装入内存并运行的原理

1. 在命令行`command.exe`中执行`1`命令
	- command设置CPU的`CS:IP`，指向程序的入口(第一条命令)
2. 1.exe中的程序运行
	- 找到一个段起始地址为`SA:0000`的容量足够的空闲内存
	- 在这段内存区的前256个字节中，创建一个称为程序的前缀，PSP的数据区，DOS要利用PSP和被加载程序进行通信，psp描述在操作系统中
	- 从这段内存区的256字节，处开始将程序装入程序的地址被设为`SA+10H:0`
	- 将该内存区的断地址存入`DS`中。初始化其他相关寄存器后，设置`CS:IP`指向程序的入口
3. 运行结束返回

外壳程序(shell)：用于与系统进行交互，控制操作系统
