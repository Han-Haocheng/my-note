# 内中断
中断：可以在执行完当前正在执行的指令之后，检测到从 CPU 外部发送过来的或内部产生的一种特殊信息，并且可以立即对所接收到的信息进行处理
- 中断源：发出中断请求的来源
- 分类：硬件中断（内中断、外中断）、软件中断

## 内中断的产生

- 外中断：外部设备发出的指令，可屏蔽的中断
- 内中断：不可屏蔽的中断
- 软件中断：调用可被调用的程序功能
- 优先权从高到低
	1. 除法错误、溢出中断、软件中断
	2. 不可屏蔽中断
	3. 可屏蔽中断
	4. 单步中断

### 中断处理程序

中断处理程序：用来处理中断信息的程序，在内存中保存，其中存放着256个中断源所对应的中断处理程序的入口
- 类型码：中断信息中用于标识中断源，用于定位中断处理程序
- 

### 中断向量表

中断向量表，就是中断处理程序入口地址的列表
- 中断向量：中断处理程序的入口地址
- 位置：`0:0-0:3FF`
- 大小：1024byte
- 高地址字存放段地址，低地址字存放偏移地址。

检测点12.1
(1)用Debug查看内存，情况如下：
0000:0000 68 10 A7 00 8B 01 70 00-16 00 9D 03 8B 01 70 00
则3号中断源对应的中断处理程序的入口地址为：0070:018B。
(2)存储N号中断源对应的中断处理程序入口的偏移地址的内存单元的地址
为：4N。
存储N号中断源对应的中断处理程序入口的段地址的内存单元的地址为：4N+2。

## 中断过程

中断过程：CPU硬件用中断类型码，在中断向量表中找到中断处理程序的入口。找到这个入口地址的最终目的是用它设置CS和IP
1. 取得中断类型码N;
2. pushf
3. TF=0,IF=0
4. push CS
5. push IP
6. (IP)=(Nx4),(CS)=(Nx4+2)

序的入口地址设置IP和CS。

###  中断处理程序和iret指令
中断处理程序的编写方法和子程序的比较相似
1. 保存用到的寄存器
2. 处理中断
3. 恢复用到的寄存器
4. 用iret 指令返回

iret指令的功能用汇编语法描述，通常和硬件自动完成的中断过程配合使用

```
pop IP
pop CS
popf
```

### 除法错误中断的处理

除法错误中断（0号中断）：当CPU检测到这个信息，会引发中断过程并执行对应的中断处理程序

```
CODESG SEGMENT
	MOV AX,1000H
	MOV BH,1
	DIV BH
	MOV 4C00H
	INT 21H
CODESG ENDS
END
```

## 编程处理0号中断

编程：当发生除法溢出时，在屏幕中间显示“overflow!”,返回DOS。

(1)当发生除法溢出的时候，产生0号中断信息，从而引发中断过程。
- 取得中断类型码0;
- 标志寄存器入栈，TF、IF设置为0;
- CS、IP入栈；
- (IP)=(0x4),(CS)=(0x4+2)

(2)可见，当中断0发生时，CPU将转去执行中断处理程序。
只要按如下步骤编写中断处理程序，当中断0发生时，即可显示“overflow!”。
- 相关处理；
- 向显示缓冲区送字符串“overflow!”;
- 返回DOS。

我们将这段程序称为：do0。

(3)现在的问题是：do0应存放在内存中。因为除法溢出随时可能发生，CPU 随时都可能将CS:IP指向do0的入口，执行程序。

(4)将中断处理程序do0放到0000:0200后，若要使得除法溢出发生的时候，CPU转去执行do0,则必须将do0的入口地址，即0000:0200登记在中断向量表的对应表项中。


(1)编写可以显示“overflow!”的中断处理程序：do0;
(2)将do0送入内存0000:0200处；
(3)将do0的入口地址0000:0200存储在中断向量表0号表项中。


```
ASSUME CS:CODE,DS:DATA
DATA SEGMENT
       
DATA ENDS
STACK SEGMENT
        DW 16 DUP(0)
STACK ENDS
CODE SEGMENT
  ;DO0安装程序
  START:MOV  AX, STACK
        MOV  SS,AX
        MOV  SP,16*2
        
        MOV  AX,CS
        MOV  DS,AX
        MOV  SI,OFFSET DO0

        MOV  AX,0
        MOV  ES,AX
        MOV  DI,200H
  
  ;“-”是编译器识别的运算符号，编译器可以用它来进行两个常数的减法。
        MOV  CX,OFFSET DO0ED-OFFSET DO0
        CLD
        REP  MOVSB
  
  ;设置中断向量表
        MOV  AX,0
        MOV  ES,AX
        MOV  WORD PTR ES:[0],200H
        MOV  WORD PTR ES:[2],0
        
        MOV  AX,4C00H
        INT  21H

  ;显示字符串"OVERFLOW!"
  DO0:  JMP  SHORT DO0BG
        DB   "OVERFLOW!"

  DO0BG:MOV  AX,CS
        MOV  DS,AX
        MOV  SI,202H

        MOV  AX,0B800H
        MOV  ES,AX
        MOV  DI,12*160+36*2
        MOV  AH,111B

        MOV  CX,9
  S:    MOV  AL,[SI]
        MOV  ES:[DI],AL
        MOV  ES:[DI+1],AH
        INC  SI
        ADD  DI,2
        LOOP S

        MOV  AX,4C00H
        INT  21H
  DO0ED:NOP
CODE ENDS
END START
```

## 单步中断

单步中断：CPU每次执行一条命令都会发生中断，对应的标志位是`TF`，中断处理程序也是由一条条指令组成的
- 取得中断类型码1
- 标志寄存器入栈，并设置TF、IF为0
- CS、IP入栈
- (IP)=(1x4),(CS)=(1x4+2)。


中断过程
1. 取得中断类型码N;
2. 标志寄存器入栈，TF=0、IF=0;
3. CS、IP入栈；
4. (IP)=(Nx4),(CS)=(Nx4+2)。

## 响应中断的特殊情况

锁中断：CPU在执行完当前指令后，即便是发生中断，也不会响应
- 例如：设置ss:sp的时候

正确格式
```
mov ax,1000h
mov ss,ax
mov sp,0
```
