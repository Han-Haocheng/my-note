# 数据查询语言

```mysql
SELECT [ALL|DISTINCT|DISTINCTROW] <目标列表达式1> [,<目标列表达式2>] …
	FROM <表名1或视图名1> [,<表名2或视图名2> ]…  
	[WHERE <条件表达式> ]
	[GROUP BY <列名1> [HAVING <条件表达式>] ]
	[ORDERBY <列名2> [ASC|DESC]]
	[LIMIT [m,] n];
```

`ALL|DISTINCT|DISTINCTROW`：为可选项，用于指定是否返回结果集中的重复行。
- 默认为 ALL，即返回 SELECT 操作中所有匹配的行，包括存在的重复行；
- 若指定选项 DISTINCT 或 DISTINCTROW(任选)，则取消结果集中的重复行，


`FROM` 子句：用于指定数据来源于哪些表或视图；

`WHERE` 子句：为可选项，用于指定对记录的筛选过滤条件；

` GROUP BY` 子句：为可选项，用于将查询结果集按指定的字段值分组；
- 其 HAVING 子句为可选项，包含在 GROUP BY 子句后，用于指定分组时的筛选过滤条件；

`ORDER BY` 子句：为可选项，用于将查询结果集按指定字段值的升序或降序排序，
- `ASC` 为升序，`DESC` 为降序，默认为升序；

`LIMIT` 子句：为可选项，用于指定查询结果集包含的记录数。

| 子句       | 说  明        | 是否必须使用      |
| -------- | ----------- | ----------- |
| SELECT   | 指定返回的列或表达式  | 是           |
| FROM     | 指定检索数据的表    | 仅在从表选择数据时使用 |
| WHERE    | 指定行级过滤条件    | 否           |
| GKOUP BY | 指定分组字段      | 仅在按组计算聚合时使用 |
| HAVlNG   | 指定分组过滤条件    | 否           |
| ORDER BY | 指定排序字段及排序方式 | 否           |
| LIMIT    | 指定返回的记录数    | 否           |


## 单表查询


```mysql
SELECT [目标列表达式1 [,目标列表达式2[, …]]]  FROM <`表名`> ; 
```

`<目标列表达式序列>`：用于指定需要查询显示的内容
- 包括：字段名、算术表达式、字符串常量、函数和列别名

### 选择字段
#### 查询指定字段

查询指定字段：查询时若只需要显示数据表中的部分字段，这就是关系运算中的投影
- 可以在SELECT子句的“<目标列表达式序列>”中指定要查询的字段
- 若查询的字段有多个，则各个字段名之间需要用逗号分隔
- 查询返回显示结果时，结果集中各字段是依照SELECT子句中指定字段的次序给出
- 从查询结果可以看出，“<目标列表达式序列>”中各个字段的先后顺序可以与表中字段定义的顺序不一致，
- 用户可以根据应用的需要改变字段的显示顺序
- 如果想要去掉重复记录，必须在SELECT关键字后面指定关键字 `DISTINCT``或DISTINCTROW`，缺省为ALL

#### 查询所有字段




查询一个表中的所有字段有两种方法：
 一种方法是在SELECT关键字后面列出所有字段名；
 另一种更简单方法是在SELECT关键字后面直接使用星号(`*`)通配符，其语法格式如下：

```mysql
SELECT * FROM 表名 ;
```



#### 查询包括计算值

SELECT 子句的“<目标列表达式>”不仅可以是表中的字段名，也可以是表达式，还可以是字符串常量、函数等
- 可以使用`CURDATE()`函数返回当前日期，`YEAR()`函数返回指定日期对应的年份

#### 定义字段别名


当 SELECT 子句的目标列为英文字段名或表达式时，可以用 AS 子句为目标列定义别名，
- 结果集将显示别名
- 注意：当自定义的别名中含有空格时，必须使用单引号将别名括起来
- 字段的别名不允许出现在WHERE子句中

```mysql
字段名 [AS] 字段别名
```


#### 选择指定记录


在SELECT语句中，可以通过`WHERE`子句设置选择的条件(也称搜索条件或查询条件)。
- 查询时根据需要可以选择数据表中指定的记录，对数据进行过滤，这就是关系运算中的选择

```mysql
SELECT <目标列表达式序列> FROM <表名> WHERE <查询条件>
```

### 选择指定记录

常用查询条件

| 查询条件   | 操作符                                                      |
| ------ | -------------------------------------------------------- |
| 比    较 | = ,<> ,!= ,< ,<= ,> ,>= ,!< ,!> ，<br><br> NOT+含比较运算符的表达式 |
| 确定范围   | BETWEEN AND，NOT BETWEEN AND                              |
| 确定集合   | IN，NOT IN                                                |
| 字符匹配   | LIKE，NOT LIKE                                            |
| 空    值 | IS NULL，IS NOT NULL                                      |
| 多重条件   | AND，OR                                                   |
#### 比较大小


比较大小：比较运算符用于比较目标列表达式的值， 当目标列表达式的值与比较的值比较成立时，返回逻辑值 TRUE(真)，否则返回 FALSE(假)。
- “大于等于”也可以表达为“不小于”,所以该查询也可以使用关键字NOT来改写
 
#### 带BETWEEN …AND关键字的范围查询
 
 当查询的过滤条件被限定在某个取值范围时，可以使用BETWEEN…AND操作符。
 - 其中,表达式 1 的值不能大于表达式 2 的值
 - BETWEEN…AND操作符前可以加关键字NOT，表示指定范围之外的值，如果字段值不满足指定的范围，则这些记录被返回
 
```mysql
<表达式> [NOT] BETWEEN <表达式1> AND <表达式2> 
```
 
#### 带IN关键字的集合查询

 使用关键字IN可以用来查询字段值属于指定集合范围内的记录， 当要判断的值匹配集合范围内的任意一个值时，会返回结果TRUE，否则返回FALSE。
 - 注意，尽管关键字IN可用于范围内判断，但其最主要的作用是用于子查询。
 - 可以使用关键字NOT来查询属性值不属于集合范围内的记录
 
#### 带LIKE关键字的字符串匹配查询
 LIKE关键字用来查找与指定字符串匹配的记录，LIKE关键字通过通配符可实现模糊查询。其基本语法格式如下：

```mysql
<字段名> [NOT] LIKE <'匹配串'> [ESCAPE <'转义字符'>]
```

`<匹配串>` 可以是字符串常数，也可以含有通配符。
- 能进行匹配运算的字段可以是 CHAR、VARCHAR、TEXT、DATETIME 等数据类型，运算返回的结果是逻辑值 TRUE 或 FALSE


`LIKE` 关键字利用通配符，可以对不能完全确定的值进行搜索
- 通配符可以在指定字段值的任意位置，并且可以使用多个任意组合

MySQL 所支持的常用通配符有两种：`％` 和 `_`
- 任意长度的字符串“`％`
- 代表任意单个字符 `_`



 如果要匹配的字符串本身就含有通配符“`％`”或“`_`”，这时就要使用``ESCAPE``短语对通配符进行转义， 把通配符“`％`”或“`_`”转换成普通字符
- 由于通常情况下，下画线“`_`”是一个通配符，所以这里使用关键字ESCAPE指定一个转义字符“#”，’<匹配串>’中“`#`”后面的“`_`”不再是通配符，从而改变“`_`”原有的通配符特殊作用，使其在匹配串中成为一个普通字符。
- 注意：百分号“`%`”不能匹配空值`NULL`。 也可以使用"`\`"


#### 使用正则表达式的查询

 字符串查询除了按照给定的具体字符串外，还可以定义字符串规则进行匹配查询。 可以用来正则表达式描述一种字符串匹配的模式(规则)。 根据指定的匹配规则可以检查一个字符串是否含有某种子串，以及从字符串中取出符合某个条件的子串等。也就是按照某种规则去匹配符合条件的字符串进行查询。用正则表达式定义一个模板，将字符模式与所搜索的字符串进行匹配。


 - MySQL 中使用关键字 `REGEXP` 指定正则表达式的字符匹配模式
 - 运算符 `RLIKE` 是 `REGEXP` 的同义词
 
```mysql
[NOT][REGEXP|RLIKE] <正则表达式>
``` 


**正则表达式**：由普通字符以及描述规则的特殊字符（元字符）组成的文字模式，可以是单个的字符、字符集合、字符范围、字符间的选择或者所有这些组件的任意组合


正则表达式定义的模式用一个字符串表示，其中特殊字符(“元字符”)的含义如下：
- `[  ]`：为定义的匹配字符。
- `[^  ]`：用于定义字符串中不可匹配的字符，“^”表示除…之外。
- `{  }`：定义匹配的字符数量。
-  `.`：用于表示匹配除了空白外的任意字符。
- `|`：用于表示“或”的逻辑符号，只要符合两个规则中的一个即匹配。
- `(  )`：用于分块，与数学运算中的小括号作用类似。
- `+`：表示最少 1 个字符要符合，最多到无限多个字符。
- `\`：若规则表达式中包含有特殊意义的字符，必须在这些特殊符号之前加上“\” 
	- `\d`表示0-9的数字

#### 带 IS NULL 关键字的空值查询

 空值一般表示数据未知、不确定或以后再添加。空值 NULL 不同于 0，也不同于空字符串。
- 当需要查询某字段内容是否为空值时，可以使用关键字 `IS NULL` 来实现。
- 注意：`IS NULL`不能用`=NULL`代替，`IS NOT NULL`”也不能用`!=NULL`代替。

#### 带 AND 或 OR 的多条件查询

带 `AND` 或 `OR` 的多条件查询：逻辑运算符 AND 和 OR 可用实现多个查询条件。
- `AND`只有满足所有查询条件的记录才会被返回
- `OR` 只要满足其中一个查询条件的记录即可被返回
 - `AND` 的优先级高于 `OR`，可以用括号改变优先级

### 对查询结果排序

 在 SELECT 语句中，可以使用 `ORDER BY` 子句将查询结果集中的记录按一个或多个字段值的升序或降序排列。
- `ASC`：默认值，按升序排列
- `DESC`：按降序排列
- 注意：当对空值进行排序时，ORDER BY 子句会将该空值作为最小值来对待。


升序排序（不写为默认）
```mysql
SELECT <目标列表达式序列> FROM <`表名`> ORDER BY sex;
```

降序排序
```mysql
SELECT <目标列表达式序列> FROM <`表名`> ORDER BY sex DESC;
``` 

### 限制查询结果的数量

 当使用 SELECT 语句返回的结果集中行数很多时，为了便于用户对查向结果集进行浏览和操作，可以使用 LIMIT 子句来限制 SELECT 语句返回的行数(界限)。
 
 LIMIT 子句的语法格式是：
```mysql
LIMIT [初始位置,] <行数>
# 或者
LIMIT <行数> [OFFSET <位置偏移量>]
``` 

`行数`：指定需要返回的记录数

 `初始位置` (`位置偏移量`)：可选参数
 - 如果从第一条记录开始显示，则初始位置为 0，
 - 如果从第二条记录开始显示，则初始位置为 1，后面的记录位置依次类推。



## 分组及聚合查询

 SELECT 语句可以对查询结果进行聚合函数计算；
 还可以通过对查询结果进行分组，再利用聚合函数实现对各组进行汇总。
 查询中完成计算的函数称为聚合函数(如 COUNT()、SUM()、AVG() 、MAX()、MIN()等)，分组查询用 GROUP BY 子句完成分组。
### 使用聚合函数查询


聚合函数：MySQL 提供的一类系统内置函数，常用于对一组值进行计算，然后返回各组计算值。使用聚合函数可以对数据进行分析。

常用聚合函数表

| 函数名                           | 说明         |
| ----------------------------- | ---------- |
| `COUNT([DISTINCT\|ALL])`      | 统计数据表的记录数  |
| `COUNT([DISTINCT\|ALL])<列名>)` | 统计指定字段的记录数 |
| `SUM([DISTINCT\|ALL])<列名>)`   | 计算指定字段的和   |
| `AVG([DISTINCT\|ALL])<列名>)`   | 计算指定字段的平均值 |
| `MAX([DISTINCT\|ALL])<列名>)`   | 求指定字段的最大值  |
| `MIN([DISTINCT\|ALL])<列名>)`   | 求指定字段的最小值  |

不指定 DISTINCT 短语或指定 ALL 短语(ALL 为默认值)，则表示
- `DISTINCT`：要取消指定列中的重复值
- `ALL`：缺省值，不取消重复值

- 注意：除函数 COUNT(`*`)外，其余聚合函数（包括 COUNT(<列名>)）都会忽略空值


### 分组聚合查询

`GROUP BY` 子句：对数据进行分组计算
 - 分组计算：目的是为了分类汇总计算。 如果不对查询结果分组，聚合函数作用于整个查询结果的每一记录， 对查询结果分组后，聚合函数分别作用于每个组，查询结果按组聚合输出


`GROUP BY`对查询结果按字段列表进行分组，字段值相等的记录分为一组

```mysql
GROUP BY <分组字段列表> [HAVING <条件表达式>]
```

 GROUP BY 
 指定用于分组的字段列表可以是一列，也可以是多列，彼此间用逗号分隔；
 HAVING 短语设定分组条件，对分组的结果进行筛选过滤，仅输出满足条件的组。

注意：
 使用 GROUP BY 子句后，SELECT 子句的目标列表达式中只应包含 GROUP BY 子句中的分组字段和聚合函数，
 分组字段作为提示；
 聚合函数为分组计算值。
