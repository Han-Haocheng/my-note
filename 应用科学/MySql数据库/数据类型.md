# 数据类型

## 数值类型


很小整数（TINYINT）：M 表示数值的位数，
 - 带符号的范围是-128～127，无符号的范围是 0～255。

```mysql
TINYINT[(M)] [UNSIGNED] [ZEROFILL] 
```
 - `[UNSIGNED]`指定无符号正值
 - `[ZEROFILL]` 填充 0 。

普通大小整数
```mysql
{INTEGER|INT} [(M)] [UNSIGNED] [ZEROFILL] 
```

浮点数
```mysql
DOUBLE[(M,D)] # 普通大小(双精度)
FLOAT[(M,D)] # 小
DECIMAL(M,D) # 自定义准确浮点数
```

## 日期和时间类型

日期型。支持的范围为’1000-01-01’到’9999-12-31‘’
- 以’YYYY-MM-DD’格式显示DATE值
- 允许使用字符串或数字为DATE列分配值
```mysql
DATE
```

时间型。范围是’ -838:59:59’到’838:59:59‘
- 以’HH:MM:SS’格式显示TIME值
- 允许使用字符串或数字为TIME列分配值

```mysql
TIME
```

日期和时间的组合：支持的范围是’1000-01-01 00:00:00’到’9999-12-31 23:59:59‘’
- 以’YYYY-MM-DD HH:MM:SS’格式显示DATETIME值
```mysql
DATETIME
```

时间戳：数据在特定时间点已经存在并可验证，以证明用户的某些数据的产生时间
- 范围是’1970-01-01 00:00:00’到 2037 年
```mysql
TIMESTAMP[(M)]
```

年：两位或四位格式。默认是四位格式。
 - 四位格式，允许的值是 1901～2155 和 0000
 - 两位格式，允许的值是 70～69，表示从 1970 年～2069 年。
- 以YYYY格式显示YEAR值
- 允许使用字符串或数字为YEAR列分配值

```mysql
YEAR[(2|4)]
```


## 字符串类型

固定长度的字符数据类型：用于保存以文本格式存储的信息。
- M 表示列长度，其范围是 0～255 个字符

```mysql
{CHAR|CHARACTER}[(M)]
```

可变长的字符数据类型：用于保存以文本格式存储的信息
- M 表示最大列长度，其范围是 0～65535
```mysql
VARCHAR[(M)]
```

# 数据操作语句


## 数据查询

SELECT 语句：以关系数据模型中对表对象的关系运算为基础的操作，功能强大、使用灵活、相对复杂，是数据库操作的重点

```mysql
SELECT [ALL|DISTINCT|DISTINCTROW] <`fd_name`> [,<`fd_name2`>[,...]]
	FROM <`tb_name`|`view_name`> [,<`tb_name`|`view_name`>[,...]]
	[WHERE <conditions> ]
	[GROUP BY <`col_name`> [HAVING <conditions>] ]
 	[ORDER BY <`col_name`> [ASC|DESC]]
 	[LIMIT [m,] n]；
```

`[ALL|DISTINCT|DISTINCTROW]`：可选项，指定是否返回结果集中的重复行
- `ALL`：默认，返回所有匹配行，包括存在的重复行
- `DISTINCT|DISTINCTROW`：取消结果集中的重复行

| 子句       | 说  明        | 是否必须使用      |
| -------- | ----------- | ----------- |
| SELECT   | 指定返回的列或表达式  | 是           |
| FROM     | 指定检索数据的表    | 仅在从表选择数据时使用 |
| WHERE    | 指定行级过滤条件    | 否           |
| GKOUP BY | 指定分组字段      | 仅在按组计算聚合时使用 |
| HAVlNG   | 指定分组过滤条件    | 否           |
| ORDER BY | 指定排序字段及排序方式 | 否           |
| LIMIT    | 指定返回的记录数    | 否           |


`FROM` 字句：用于指定数据来源于哪些表或视图

```mysql
FROM <`tb_name`|`view_name`> [,<`tb_name`|`view_name`>[,...]]
```

`WHERE` 子句：可选项，用于将查询结果集按指定的字段值分组
```mysql
WHERE <conditions>
```

`HAVING` 子句：可选项，用于指定分组时的筛选过滤条件，包含在 GROUP BY 子句后

```mysql
GROUP BY <`col_name`> [HAVING <conditions>]
```

`ORDER BY` 子句：可选项，用于将查询结果集按指定字段值的升序或降序排序
```mysql
ORDER BY <`col_name`> [ASC|DESC]
```
- `ASC` ：默认，升序
- `DESC` ：降序
 `LIMIT` 子句：可选项，用于指定查询结果集包含的记录数

```mysql
LIMIT [m,] n
```

### 单表查询

查询指定字段：查询时若只显示数据表中的部分字段，是关系运算中的投影
- fd_name 的顺序与结果一致，可以根据需要显示不同的顺序
```mysql
SELECT [ALL|DISTINCT|DISTINCTROW] <`fd_name`> [,<`fd_name2`>[,...]]
	FROM <`tb_name`>;
```

查询所有字段


```mysql
SELECT * FROM 表名 ;
```
