### 用例关系：包含

#计算机/软件建模/用例/关系 包含Include：基用例中复用被包含用例的行为，提取公共步骤，便于复用。表示某个用例中包含了其他用例的行为，从两个或多个用例行为中提取公共部分的能力，主要用于支持用例行为的复用

### 用例关系：扩展

#计算机/软件建模/用例/关系 扩展Extend：通过扩展用例对基用例增加附加的行为。某个用例在特定情况下，包含其他用例(扩展用例)的行为，表示功能被扩展

- 为了将基用例的一些特殊情况分离出来，在保持基用例本身相对完整的情况下处理这些特殊行为
- 即不改变基用例，对基用例的行为进行扩展

提示：当基用例的行为非常复杂时，针对一些特殊的处理可以提炼为扩展用例
- 对用例基本行为的可选择特征的描述
- 复杂的异常处理行为的描述
- 面向特定用户的特定需求
- 范围管理和版本管理

### 扩展 VS. 包含
#计算机/软件建模/用例/关系 扩展包含 对比

| 出发点不同     | 包含关系                     | 扩展关系                      |
| ---------- | ------------------------ | ------------------------- |
|            |                          | 通过扩展点在不影响基用例的情况下附加行为      |
| 达到的效果不同    | 基用例中的一部分业务放在子用例中         | 基用例处理一般情况，一些特殊业务放在子用例中    |
| 执行子用例方式不同 | 基用例中直接引用子用例              | 主用例达到一定条件触发扩展点，子用例通过扩展点触发 |
| 使用方式不同    | 基用例不够完整，一般要联合子用例为参与者提供价值 | 基用例相对完整，可以单独为参与者提供价值      |
| 依赖方向不同    | 主用例依赖子用例，子用例相对独立         | 子用例依赖主用例，主用例相对独立          |

### 用例关系：泛化

#计算机/软件建模/用例/关系 泛化Generalization：派生用例继承泛化用例的行为并添加新行为。表示子用例继承了父用例
- 用例间的泛化关系表明子用例继承父用例中定义的所有属性、行为序列和扩展点
- 参与父用例中所有的关系
- 慎重用例间的泛化关系，将泛化转换成扩展

## 用例分包：组织模型

 
对于大规模系统，当用例数量很多时，难以在一个层次上一次性描述所有用例

#计算机/软件建模/用例 分包：让系统的用例图能够更为清晰的表现出系统的业务逻辑关系和层次，对系统进行模块的分割，这种分割将影响到系统今后的开发、系统的最终表现形式
### 分包策略
#计算机/软件建模/用例 分包 策略：基于业务主题、按照参与者、基于开发团队、基于发布情况

## 用例分级

#计算机/软件建模/用例 用例分级：由于用例和迭代开发，迭代开发中开发周期的定义是围绕用例来组织的，一个迭代周期要被指派一个到多个用例，如果完全版本的用例在一个迭代周期中处理起来太复杂的话，那就采用简化版本的用例

#计算机/软件建模/用例 用例分级原则：高级别用例是那些对系统核心架构影响最大的用例

- 对架构设计有重要影响的用例，如在领域层中增加多个类的用例或者需要持久化的用例
- 体现系统核心业务流程的用例
- 存在开发风险的用例
- 涉及新技术或需要创新的用例
- 能够尽快投入使用并带来直接经济效益的用例


用例分级策略
- 使用一个简单的但是有些不精确的分类方法，如将用例划分成高、中、低三个等级
- 依照上述的影响用例级别的特性给用例打分（特性也可能带有权值）
