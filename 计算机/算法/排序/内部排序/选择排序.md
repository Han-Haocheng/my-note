# 选择排序


选择排序：每一趟在后面 $n-i+1$ 个中选出关键字最小的对象，作为有序序列的第 $i$ 个记录。


## 简单选择排序

基本思想
- 第1趟从 $n$ 个记录中选出关键字最小的记录和第1个记录交换；
- 第2趟从第2个记录开始的 $n-1$ 个记录中再选出关键字最小的记录与第2个记录交换；
- 如此第 $i$ 趟则从第 $i$ 个记录开始的 $n-i+1$ 个记录中选出关键字最小的记录与第 $i$ 个记录交换
- 直到整个序列按关键字有序。
```c
// 简单选择排序主函数
void SelectSort(SqList &L) {
    // 对顺序表L做简单选择排序
    int i, j;
    for(i = 1; i < L.length; ++i) {
        // 选择第i小的记录，并交换到位
        j = SelectMinkey(L, i);        // 在L.r[i…L.length]中选择key最小的记录
        if(i != j) {
            Swap(L.r[i], L.r[j]);      // 与第i个记录交换
        }
    }
}
```
算法分析：
- 时间复杂度：$T(n)=O(n^2)$
- 空间复杂度：$S(n)=O(1)$
- 稳定性：是不稳定的排序方法

## 堆排序

堆排序是利用堆的特性进行排序的方法。堆的定义：$n$个元素的序列$\{k_1,k_2,...,k_n\}$，当且仅当任一$k_i$满足以下关系时，称为堆：$\begin{cases}k_i \leq k_{2i} \\k_i \leq k_{2i+1}\end{cases}$$或$$\begin{cases}k_i \geq k_{2i} \\k_i \geq k_{2i+1}\end{cases}$


分别称为小顶堆或大顶堆。如果将序列看成一个完全二叉树，非终端结点的值均小于或大于左右子结点的值。（树只是作为堆的描述工具，堆实际是存放在线性空间中的）
堆排序的基本思想
首先用待排序的记录序列构造出一个堆，此时选出了堆中所有记录的最大者（或最小者）为堆顶，随后将它从堆中移走（通常是将堆顶记录和堆中最后一个记录交换），并将剩余的记录再调整成堆，这样又找出了次大（次小）的记录，依次类推，直到堆中只有一个记录为止。
基于$n$个元素建立堆的方法如下：对于一个含有$n$个结点的完全二叉树，其最后一个结点是第$\lfloor n/2 \rfloor$个结点的孩子结点。对第$\lfloor n/2 \rfloor$个结点及其孩子结点进行调整（交换结点关键字），使之满足堆的定义，之后再向前依次对各个结点为根的子树进行调整，最终整个完全二叉树为一个堆。

从第$\lfloor n/2 \rfloor$（最后一个分支结点）个元素起，至第一个元素止（从下往上），进行比较与调整，来构建大顶堆。


```c
/* 创建大顶堆 */
/* 调整堆的时间复杂度与树的高度有关,故为O(h) */
/* 在建立时,关键字个数为n,比较的次数最大不超过4n,所以时间复杂度为O(n) */
void CreateMaxHeap( ElemType A[ ],int len) {
    for(int i=len/2;i>0;i--)
        HeadAdjust(A,i,len);      // 从i=[n/2]~1,反复调整堆
}

/* 堆排序 */
void Sort( ElemType A[ ],int len) {
    CreateMaxHeap(A,len);         // 构建堆
    for(i=len;i>1;i--) {
        Swap(A[i],A[1]);         // n-1趟的交换和建堆过程
        HeadAdjust(A,1,i-1);     // 将剩余的元素整理成堆
    }
}
```

算法分析：
- 时间复杂度：$T(n)=O(n\log_2 n)$
- 空间复杂度：$S(n)=O(1)$
- 稳定性：是不稳定的排序方法
