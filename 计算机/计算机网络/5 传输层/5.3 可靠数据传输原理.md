# 可靠数据传输原理

可靠数据传输协议：用于保证数据不错、不丢、不乱的协议，将不可靠传输数据处理为可靠的数据

- 基本结构：接口
- 状态机FSM（FiniteStateMachine）：用于刻画传输协议，描述运行时的状态变化情况和条件

![[计算机/计算机网络/attachments/2025-10-31_153554.svg]]

- `rdt_send()`：被上层应用调用，将数据交给rdt以发送给对方
- `udt_send()`：被rdt调用，在不可靠信道上向接收方传输数据
- `rdt_rcv()`：当数据包到达接收方信道时被调用
- `deliver_data()`：被rdt调用，向上层应用交付数据

## 可靠信道上的可靠数据传输

rdt1.0

可靠信道上的可靠数据传输：底层信道完全可靠，不会发生位错、乱序、丢组情况
- 发送方和接收方的FSM独立

## 会位错信道

### rdt2.0

产生位错的信道：底层信道只可能发生位错，而不会发送乱序和丢组情况
- 机制：校验和检测机制、确认机制、重传机制
- 确认机制ACK(Acknowlegedgements)：接收方显式告知发送方分组已正确接收
- NAK：接收方显式告知发送方分组有错，发送方收到后重传分组
- ARQ协议（Automatic Repeat reQuest）：基于重传机制的rdt协议

![[计算机/计算机网络/attachments/2025-11-03_164527.svg]]

```c++
bool rdt_send(data){
	snkpkt = make_pkt(data,checksum);
	do{
		udt_send(snkpkt);
	}while(rdt_rcv(rcvpkt) && isNAK(rcvpkt));
}

bool rdt_rcv(rcvpkt){
	if(corrut(rcvpkt)){
		udt_send(NAK);
	}else{
		extract(rcvpkt,data);
		devliver_data(data);
		udt_send(ACK);
	}
	
}
```

缺陷：如果ACK/NAK消息发生错误/被破坏corrupted
- 为ACK/NAK增加校验和，检错并纠错。代价大
- 添加额外的控制消息。消息也会错
- 如果ACK/NAK坏掉，发送方重传重传。简单重传会产生重复分组


### rdt2.1

通过在头部添加序列号保证接收方收到的数据是没有重复分组的
- 序列号Sequence number：发送方给每一个分组增加一个特殊ID
- 接收方丢弃重复分组

![[计算机/计算机网络/attachments/2025-11-03_181910.svg]]


```c++
int rdt_send(data){
	static int n = 0;
	snkpkt = make_pkt(n,data,checksum);
	udt_send(snkpkt);
	while(rdt_rcv(rcvpkt) && (corrupt(rcvpkt)||isNAK(rcvpkt)))
		udt_send(snkpkt);
	
}

int rdt_rcv(rcvpkt){
	while(corrupt(rcvpkt)){
		sndpkt = make_pkt(NAK,chksum);
		udt_send(sndpkt);
	}
	if(has_seq0(rcvpkt)){
		extract(rcvpkt,data);
		devliver_data(data);
		sndpkt=make_pkt(ACK,chksum);
		udt_send(ACK);
		while(has_seq0(rcvpkt)){
			sndpkt=make_pkt(ACK,chksum);
			udt_send(ACK);
		}
	}
	

	
	
}
```

## 会位错丢组信道

rdt3.0：信道会发生位错、丢组。在rdt2.2的基础上，发送方等待ACK有限时间
- 如果在规定时间内没有收到ACK，重传
- 如果分组或ACK延迟，会产生重复
- 定时器机制
- 响应延迟$=\frac{L/R}{RTT+L/R}$
