### ARP协议作用  RARP协议作用

ARP协议：用于将网络层的IP地址解析为数据链路层的MAC地址，是局域网中设备通信的重要协议

RARP协议：允许局域网中的设备通过已知的物理地址（MAC地址）来请求分配IP地址。这种协议通常用于无盘工作站在启动时获取IP地址，因为这些工作站没有存储设备来保存自己的IP配置信息

### 路由器、HUB、交换机、网关工作层次

集线器HUB：物理层。
- 对接收到的信号进行再生、整形和放大，以扩大网络传输距离；
- 连接多台设备形成星形局域网

网桥：数据链路层


交换机：数据链路层。
- 学习并维护 MAC 地址表，实现点对点精准转发

路由器（route）：网络层
- 连接不同网络
- 根据目标 IP 地址和路由表选择最佳路径，转发数据包

网关：网络层
- 连接异构网络，实现协议转换和数据转发

### TCP流量控制机制

TCP流量控制：通过滑动窗口机制，实现了发送速率与接收能力的动态匹配，是TCP可靠传输和高效通信的重要基础。其核心在于接收方通告窗口、发送方动态调整，以及零窗口探测、持续计时器等细节设计，保障了数据传输的稳定性与高效性

### 停止等待协议对重复报文处理


停止等待协议对重复报文处理
- 必须丢弃重复报文，不向上层交付；
- 必须再次发送ACK，防止发送方死循环重传

### CSMA/CD协议

#### 2.1 载波侦听（Carrier Sense）

- 发送前，先监听信道是否空闲；
- 若信道忙碌，则持续等待或按算法退避；
- 若信道空闲，则进入发送阶段。

#### 2.2 多路访问（Multiple Access）

- 多个设备共享同一总线，任何一个设备都可以在信道空闲时尝试发送数据；
- 若两个或多个设备同时发送，则可能发生冲突。

#### 2.3 冲突检测（Collision Detection）

- 发送数据的同时，持续监听信道；
- 若检测到发送信号与接收信号不一致（即发生冲突），立即停止发送；
- 发送一个“阻塞信号”（Jam Signal），强化冲突，确保所有节点都能感知到冲突。

#### 2.4 二进制指数退避（Binary Exponential Backoff）

- 冲突后，每个冲突节点随机选择一个退避时间，等待后再尝试发送；
- 退避时间范围随着冲突次数指数增长，即：
    - 第n次冲突后，等待时间为 0 到 $2^n−1$ 个时隙的随机值；
    - 通常最大重传次数为16次，超过则放弃发送

### 数据编码与调制

| 编码方式       | 编码规则                      | 特点及适用场景                           |
| ---------- | ------------------------- | --------------------------------- |
| 非归零编码（NRZ） | 高电平=1，低电平=0，码元内不归零        | 实现简单，但无自同步能力，需额外时钟；易受直流干扰。        |
| 归零编码（RZ）   | 每个码元结束归零                  | 自同步，但效率低，带宽浪费严重。                  |
| 曼彻斯特编码     | 每个码元中间跳变，0=低→高，1=高→低（或相反） | 自同步，抗干扰强，效率50%；用于经典以太网（10BASE-T）。 |
| 差分曼彻斯特编码   | 每个码元中间跳变，0=边界跳变，1=边界不变    | 自同步，抗干扰优于曼彻斯特，用于令牌环网。             |

| 调制方式        | 调制原理                    | 特点及适用场景                      |
| ----------- | ----------------------- | ---------------------------- |
| 幅移键控（ASK）   | 用载波振幅变化表示0和1            | 实现简单，抗干扰差；用于光通信、低成本RFID。     |
| 频移键控（FSK）   | 用载波频率变化表示0和1            | 抗干扰较强，需较大带宽；用于电话拨号、ZigBee。   |
| 相移键控（PSK）   | 用载波相位变化表示0和1            | 抗干扰好，带宽效率高；用于数字微波、卫星通信。      |
| 正交幅度调制（QAM） | 幅度+相位联合调制（如16QAM、64QAM） | 高频谱效率，多bit/符号；用于WiFi、数字电视、5G |

### DNS功能

**DNS（Domain Name System，域名系统）** 是互联网的“电话簿”，负责将人类可读的域名（如 `www.example.com`）解析为机器可识别的IP地址（如 `192.0.2.1`），是互联网通信的基础服务

#### 2.1 域名解析（正向解析）

- **功能**：将域名转换为IP地址，是DNS最核心的功能。
- **流程**：浏览器/客户端发起DNS查询，本地DNS服务器（如ISP DNS）逐级向根、顶级域（TLD）、权威DNS服务器查询，最终返回IP。
- **查询方式**：
    - **递归查询**：本地DNS代替客户端完成全部查询，最终返回IP，客户端只需发一次请求。
    - **迭代查询**：客户端/本地DNS依次向各级服务器查询，每次得到下一级服务器地址，直至获得结果。

#### 2.2 反向解析（Reverse DNS）

- **功能**：将IP地址解析为域名，常用于邮件服务器验证、安全审计、日志分析。
- **记录类型**：PTR记录用于实现反向解析。

#### 2.3 负载均衡与流量分发

- **原理**：为同一域名配置多个IP地址，DNS服务器按轮询、加权、IP哈希等策略返回不同IP，将用户请求分配到不同服务器，实现负载均衡和故障转移。
- **优点**：实现简单、成本低，适合多机房、多活架构；
- **缺点**：TTL缓存可能导致切换延迟，无法感知服务器实时状态。

#### 2.4 邮件路由（MX记录）

- **功能**：通过MX记录指定域名的邮件服务器地址，实现电子邮件的投递路由。
- **流程**：发送方邮件服务器查询目标域的MX记录，将邮件投递到优先级最高的可用邮件服务器。

#### 2.5 安全扩展（DNSSEC）

- **功能**：通过数字签名验证DNS响应的来源和完整性，防止DNS劫持、缓存投毒等攻击。
- **记录类型**：DNSKEY、RRSIG、DS、NSEC等。
- **局限性**：不加密查询内容，仅保证数据真实性和完整性。


### 香农定理

**香农定理（Shannon’s Theorem）**，又称**信道容量定理**，由信息论创始人克劳德·香农于1948年提出，给出了在**带宽受限、受高斯白噪声干扰**的信道中，**信息传输速率的理论极限**，是现代通信系统设计的理论基础。


$C = B \cdot \log_2\left(1 + \frac{S}{N}\right)$

- **C**：信道容量，单位比特/秒（bps），即无差错传输的最大速率；
- **B**：信道带宽，单位赫兹（Hz）；
- **S**：信号平均功率，单位瓦特（W）；
- **N**：噪声平均功率，单位瓦特（W）；
- **S/N**：信噪比（Signal-to-Noise Ratio），无量纲，常用分贝（dB）表示。

### 传输层协议特性（TCP vs UDP）

| 特性   | TCP                 | UDP             |
| ---- | ------------------- | --------------- |
| 连接方式 | 面向连接（三次握手建立，四次挥手释放） | 无连接（直接发送）       |
| 可靠性  | 可靠（确认、重传、排序、校验）     | 不可靠（尽力交付）       |
| 传输模式 | 面向字节流               | 面向数据报           |
| 流量控制 | 有（滑动窗口）             | 无               |
| 拥塞控制 | 有（慢启动、拥塞避免、快重传、快恢复） | 无               |
| 头部开销 | 最少20字节              | 固定8字节           |
| 顺序保证 | 保证顺序                | 不保证顺序           |
| 通信模式 | 一对一（点对点）            | 一对一、一对多、广播      |
| 传输效率 | 较低（机制复杂）            | 高（简单、开销小）       |
| 典型应用 | 网页浏览、文件传输、邮件、数据库    | 视频/语音、直播、游戏、DNS |

### 信道复用技术

#### 2.1 频分复用（FDM, Frequency Division Multiplexing）

- **原理**：将信道总带宽划分为若干不重叠的子频带，每个用户独占一个子频带，所有用户可同时传输。
- **工作流程**：调制器将各路信号调制到不同载波频率，混合后发送；接收端用带通滤波器分离、解调还原。
- **举例**：广播电台（FM 97.4MHz/103.9MHz）、有线电视、ADSL（语音/上行/下行分频段）。
- **优点**：技术成熟、实现简单、适合模拟信号、低延迟。
- **缺点**：需保护频带、频谱效率低、易受非线性干扰、扩展性受限。
- **适用场景**：传统广播、有线电视、模拟微波通信。

#### 2.2 时分复用（TDM, Time Division Multiplexing）

- **原理**：将时间划分为固定长度的帧，每帧划分为若干时隙，每个用户在固定时隙占用整个信道带宽，轮流传输。
- **工作流程**：各用户数据按顺序插入帧内时隙，接收端按固定时隙顺序分离。
- **举例**：T1/E1数字电话系统（T1：24路，E1：30路）、SDH传输网。
- **优点**：适合数字信号、抗干扰能力强、无需保护频带。
- **缺点**：固定时隙分配，空闲时隙浪费、需严格同步。
- **适用场景**：数字电话网、PCM数字传输、同步通信系统。

#### 2.3 统计时分复用（STDM, Statistical TDM）

- **原理**：动态分配时隙，仅当用户有数据时才分配时隙，按需发送，提高信道利用率。
- **工作流程**：集中器缓存数据，按需组成STDM帧，帧内时隙需附带地址信息。
- **举例**：数字电视复用器、分组交换网、现代路由器。
- **优点**：信道利用率高、适应突发数据、支持更多用户。
- **缺点**：需地址开销、控制复杂、延迟不确定。
- **适用场景**：计算机网络、突发数据传输、动态多路复用。

#### 2.4 波分复用（WDM, Wavelength Division Multiplexing）

- **原理**：光的频分复用，将不同波长（频率）的光信号在同一光纤中传输。
- **分类**：WDM（粗波分）、DWDM（密集波分，几十至上百路）。
- **举例**：8路2.5Gbps光信号复用，单光纤速率达20Gbps；骨干网Tbps级传输。
- **优点**：极大提升光纤容量、低串扰、灵活配置。
- **缺点**：设备成本高、波长数量受限。
- **适用场景**：光纤骨干网、数据中心互联、大容量传输。

#### 2.5 码分复用（CDM, Code Division Multiplexing）

- **原理**：所有用户共享同一频带和时隙，用唯一正交码序列区分信号，接收端通过码相关运算分离。
- **举例**：CDMA移动通信（3G）、GPS、军事通信。
- **优点**：抗干扰强、频谱效率高、安全性高、软容量。
- **缺点**：需精确码同步和功率控制、设备复杂、远近效应。
- **适用场景**：移动通信、卫星通信、安全通信。

### 五层网络体系结构及数据传递过程

- 物理层：在物理设备之间通过有线或无线介质传输数据流
- 数据链路层：在两个相邻的节点间传输帧
- 网络层：在两个主机IP之间传输IP数据报
- 运输层：在两个应用端口之间传输可靠或不可靠数据报
- 应用层：在两个应用进程之间传输

### 虚电路 vs 数据报服务的对比

| 比较的方面         | 虚电路服务                   | 数据报服务                     |
| ------------- | ----------------------- | ------------------------- |
| 思路            | 可靠通信应当由网络来保证            | 可靠通信应当由用户主机来保证            |
| 连接的建立         | 必须有                     | 不需要                       |
| 终点地址          | 仅在连接建立阶段使用，每个分组使用短的虚电路号 | 每个分组都有终点的完整地址             |
| 分组的转发         | 属于同一条虚电路的分组均按照同一路由进行转发  | 每个分组独立选择路由进行转发            |
| 当结点出故障时       | 所有通过出故障的结点的虚电路均不能工作     | 出故障的结点可能会丢失分组，一些路由可能会发生变化 |
| 分组的顺序         | 总是按发送顺序到达终点             | 到达终点的时间不一定按发送顺序           |
| 端到端的差错处理和流量控制 | 可以由网络负责，也可以由用户主机负责      | 由用户主机负责                   |

### 课后题3-04

【3-04】数据链路层的三个基本问题(封装成帧、透明传输和差错检测)为什么都必须加以解决?

简答：
- 封装成帧：用于界定数据，使接收方能识别帧的起止。
- 透明传输：可防止数据中的特定比特组合被误判为帧的控制信息，确保数据内容不受限制。
- 差错检测：能及时发现并丢弃出错的帧，避免无效数据继续在网络中传输，浪费带宽和处理能力。
- 三者缺一不可，共同保障了数据在链路上传输的可靠性和效率

### 课后题4-03

问题：作为中间设备,转发器、网桥、路由器和网关有何区别?

解答：将网络互相连接起来要使用一些中间设备。根据中间设备所在的层次,有以下四种
不同的中间设备:
(1)转发器：物理层使用的中间设备
(2)网桥或交换机：数据链路层使用的中间设备
(3)路由器：网络层使用的中间设备
(4)网关：在网络层以上使用的中间设备。用网关连接两个不兼容的系统需要在高层进行协议的转换

### 课后题4-04

【4-04】试简单说明下列协议的作用:IP,ARP和ICMP。

解答:
- 网际协议IP：将世界上的异构网络统合到一个虚拟的网络
- 地址解析协议ARP：把主机或路由器的IP地址转换为相应的MAC地址
- 网际控制报文协议ICMP：使主机或路由器报告差错情况和提供有关异常情况的报告，更有效地转发IP数据报和提高交付成功的概率

### 课后题4-07

【4-07】试说明IP地址与MAC地址的区别。为什么要使用这两种不同的地址?
解答:
- 从层次的角度看,MAC地址是数据链路层和物理层使用的地址,而IP地址是网络层和以上各层使用的地址,是一种逻辑地址
- 全世界存在各种的网络,其使用不同的MAC地址，异构网络互相通信需要完成非常复杂的MAC地址转换工作，不可能由用户或用户主机完成
- 统一的IP地址使所有主机和异构网络连接到同一个互联网络，软件通过调用ARP完成IP-MAC地址转换
