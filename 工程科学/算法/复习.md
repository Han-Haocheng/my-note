## 三、判断题

1. ❌用主定理法可以解决所有的递归问题的复杂度计算。
2. ❌算法的时间复杂度和空间复杂度总是相同的，因此每个算法只要分析时间复杂度即可。
3. ❌递归算法与迭代算法比较，具有编程简单，结构清晰，运行速度快的特点。
4. ⭕迭代法、递推法、尝试法、积分法、主定理法和递归树是算法分析的主要分析方法。
5. ⭕算法可以没有输入但是必须有输出。
6. ❌算法分析中的迭代法包括：直接迭代、换元迭代、消元迭代和循环迭代分析法。
7. ⭕算法的表达方式可以是自然语言、伪代码、程序框图及程序设计语言。
8. ❌解决某一问题的算法复杂度最优的算法是唯一的。
9. ⭕健壮的算法不会因为非法的输入数据而出现莫名其妙的状态。
10. ❌算法的优劣与描述算法的语言无关，而与所用的计算机的性能有关。
11. ⭕二分搜索方法在最坏的情况下用$O(\log n)$时间完成搜索任务。
12. ❌分治法的基本思想是将一个规模较大的问题分解成若干个规模较小的子问题，这些子问题之间并不一定相互独立。
13. ❌能否利用分治法完全取决于问题是否具有如下特征：利用该问题分解出的子问题的解可以合并为该问题的解。
14. ⭕反复应用分治手段能使子问题与原问题类型一致而其规模却不断缩小。
15. ⭕采用分治策略的芯片测试问题的算法复杂度可以达到$O(n)$。
16. ⭕Strassen算法的重要意义在于他证明了矩阵相乘问题的算法复杂度$O(n^{3})$不是最优解。
17. ❌分治法解决平面点对问题的算法复杂度为$O(n\log^{2}n)$
18. ⭕原始数据27,99,0,8,13,64,86,16,7,10,88,25,90，以第一个数做为划分标准做快速排序，第一轮交换之后数据变为16,25,0,8,13,10,7,27,86,64,88,99,90。
19. ⭕快速排序算法的平均复杂度是基于比较排序法中最优的排序算法。
20. ⭕用分治法求解Fibonacci数列第n项值时，算法时间复杂度为$O(logn)$。
21. ⭕动态规划算法的要求是由原问题分解出的各个子问题的解必须是最优解
22. ❌矩阵序列相乘问题不适合动态规划算法。
23. ⭕m元钱,n个项目的投资问题,可以通过动态规划算法实现。
24. ❌完全背包问题中，求背包最大价值的不可以使用动态规划法。
25. ⭕动态规划的实质是分治思想和解决冗余，因此，动态规划是一种将问题实例分解为更小的、相似的子问题，并存储子问题的解而避免计算重复的子问题，以解决最优化问题的算法策略。
26. ⭕最优子结构性质是指原问题的最优解包含其子问题的最优解。
27. ❌动态规划算法求解问题时,分解出来的子问题相互独立。
28. ⭕动态规划算法最基本要素是最优子结构。
29. ⭕动态规划法与分治法和贪心法类似，它们都是将问题实例归纳为更小的、相似的子问题，并通过求解子问题产生一个全局最优解。
30. ❌最长公共子序列LCS问题适合于回溯法。
31. ❌贪心法可以进行贪心选择的关键在于该问题的最优子结构性质。
32. ❌用贪婪算法解决零钱兑换问题时,总能找到问题的最优解。
33. ⭕贪心法必需满足贪心选择性质。

40. ❌对于使用最小数目硬币找零钱的问题，贪心法总能给出最优解。
41. ⭕贪心法常以当前情况为基础作最优选择，而不考虑各种可能的整体情况，所以贪心法不要回溯。
42. ⭕回溯法中常见的两类典型的解空间树是子集树和排列树。

11. 贪心法的基本思想是什么？
	1. 它采用逐步构造最优解的思想，从最简单的子问题到复杂的本原问题进行逐步求解，在问题求解的每一个阶段，都作出一个在一定标准下看上去最优的决策；
	2. 决策一旦作出，就不可再更改；
	3. 制定决策的依据称为贪心(婪)选择性质。
12. 贪心法的特点有哪些？
	1) 逐步构造最优解。每一步都是在一定标准下的最优决策，是只顾眼前的最优；
	2) 贪婪法常以当前情况为基础作最优选择，而不考虑整体情况，是一种“短视的”行为，所以贪婪法不要回溯。
	3) 每一步做出的决策,在以后的步骤中都不可更改；
	4) 贪婪法是一种不追求最优解，只希望得到较为满意解的方法。贪婪法一般可以快速得到满意的解，因为它省去了为找最优解要穷尽所有可能而必须耗费的大量时间；
	5) 贪心法必须进行正确性证明。
	6) 算法简单，时间和空间复杂性低。

13. 贪心法的解题步骤有哪些？

	4) ==合并局部最优解==：把子问题的解局部最优解合成原来解问题的一个解
	5) ==算法正确性证明==

15. 贪心法与动态规划的异同点
	1. 共同点：贪心算法和动态规划算法都要求问题具有==最优子结构性质==，这是两类算法的一个共同点。
	2. 不同点
		1. 贪心法需要满足==贪心选择性质==，动态规划法不需要贪心选择性质

### 问题1

给定含有n个不同的数的数组$L=<x_{1},x_{2},\dots,x_{n}>$，如果$L$中存在$x_{i}$使得$x_{1}<x_{2}<\dots x_{i-1}<x_{i}>x_{i+1}>\dots>x_{n}$，则称$L$是单峰的，并称$x_{i}$是$L$的“峰顶”，假设L是单峰的，设计一个算法找到L的峰顶。 要求：(1)进行算法分析 (2)写出算法伪代码 (3)分析算法复杂度
1. 算法分析：此题是找最大值，普通算法复杂度为Θ(n)，不是本题的最优算法。本题可利用单峰的特点，每次取剩余数据的中间点与紧邻其后的数据进行比较，如果大于等于它则向后查找峰顶，否则向前查找峰顶。此算法等同于分治法的折半查找，所以算法复杂度大幅降低
2. 伪代码
	```c
	peak(int x[],int p,int r)：
		if p>=rthen //小于2个元素算法结束
			if p> rthen return 0 //无解
			else then return p+1
		q<-(p+r)/2
		if x[q] < x[q+1] then
			return peak(x,q+1,r) //峰点在右边
		else return peak(x,p,q) //峰点在左边
	```
3. 算法复杂度：递推方程：$\begin{cases}W(n) = W(n/2) + c\\W(1)= 1\end{cases}$ ，本算法复杂度：$W(n)=\Theta(\log n)$

### 问题2

设A是含有n个元素的数组，如果元素x在A中出现的次数大于n/2，则称x是A的主元素。如果A中的元素只能进行“是否相等”的测试，但是不能排序。根据上述条件，设计一个算法，判断A中是否存在主元素。要求：(1)进行算法分析 (2)写出算法伪代码 (3)分析算法复杂度
1. 算法分析
   本题目采用分治策略--芯片测试算法：首先将A复制到B，若B中数据为奇数，取最后一个元素与其它所有元素比较，若它是主元素则算法结束，若不是主元素则扔掉；然后把B中数据两两一组进行比较，相等则扔一个留一个，不等则两个都扔。若B中元素大于1则重复以上过程；否则若B中元素为1个且此元素是主元素，则存在主元素，否则不存在主元素。
2. 伪代码
	```c
	mainElement(A, n)：
		B<-A,m<-n
		while m>1 do
			if m为奇数 then //检查最后一个是不是主元素
				if check(B,B[m-1],m) then return 1//存在主元素
				else m<-m-1 //去掉最后一个元素。
			k<-0
			for i¬0 to <m step i<-i+2 do //两两比较，每次i+2
				if B[i]==B[i+1] then
					B[k]=B[i]//留下一个元素。
					k<-k+1
				m<-k
		if m==1 and check(A,B[0],n)then//剩下的元素可能是主元素
			return 1 //存在主元素
		else return 0 //不存在主元素
	```
	```c
	check(A,x,n)//伪代码:检查x是不是数值A的主元素
		count<-0
		for i<-0 to n-1 do
			if A[i]==x then count<-count+1
		if count>n/2 return 1 //最后一个元素是主元素
else return 0 //最后一个元素不是主元素
	```
3. 本算法的时间复杂度为： $W(n)=O(n)$

### 问题3

设A和B都是从小到大已经排好序的n个不等的整数构成的数组，如果把A和B合并后的数组记作C，设计一个算法找出C的中位数(注:数组的中位数是ën/2û下取整位置的数)。要求：(1)进行算法分析 (2)写出算法伪代码 (3)分析算法复杂度
算法分析： 
$a1，a2，…，am，…，an-1，an$
$b1，b2，…，bm，…，bn-1，bn$
采用分治策略：
1. 设i为A、B数组合并后有序数组的左半部中包含A的元素最大下标，j为左半部中B的元素最大下标，则$j=n-i-2$
2. 取A中从left到right的中间位置作为i，判断$A[i]$是不是合并后的中位数，若是算法结束
3. 否则若偏大在A中从left到(i-1)中找中位数
4. 偏小在A中从(i+1)到right中找中位数
5. 重复以上过程，直到找到中位数或达到边界条件(left>right)为止

算法伪代码median()://对于均有n个元素的A、B有序数组，在left~right之间找数组a、b合并后的中位数
```c
//初始值left¬0，right¬n-2
median(A, B, n, left, right):
	//i、j为合并成有序数组后左半部中包含的A、B数组元素最大下标
	i<-(left+right)/2, j<-n-i-2 
	if left>right then 
		return min(A[left],B[n-left-1]);//边界条件，算法结束
	if A[i]>B[j+1] then
		right¬i-1
		return median(A,B,n,left,right) 
	else if B[j]>A[i+1] then
		left<-i+1
		return median(A,B,n,left,right)
	else return max(A[i],B[j])
```

算法的时间复杂度为：$W(n)=O(\log n)$

if|S|<5 then 算法结束

### 问题6

建模优化函数：
- 使用动态规划算法，令$F_{k}(y)=0,1$，$F_{k}(y)=1$ 表示用前k种钱币、总钱数为y时可以付款，那么Fk(y)满足以下递推方程：$\begin{cases}F_{k}=max\{ F_{k-1}(y),F(k-1)(y-x_{k}) \} & k=2,3,\dots,n;y=0,1,2,\dots,M \\ F_{1}(y)=\begin{cases}1 & 若y=x_{1} \\ 0 & 否则\end{cases} \\ F_{k}(y)=1 & 若y=0,k=0,1,2,\dots,M\end{cases}$
