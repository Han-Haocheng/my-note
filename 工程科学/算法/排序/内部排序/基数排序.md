# 基数排序

**基数排序**：不需要关键字之间的比较，而是将正整数按位数分割为不同数字，按每个位数的值分别比较。
- 排序方式：
  - **最低位优先（LSD）**：从最右边的单关键字开始排序。
  - **最高位优先（MSD）**：从最左边的单关键字开始排序。
## 基数排序的方法
- **桶的设置**：因十进制数有0 - 9共10个数字，故设置10个桶，标记为$B_0, B_1, ..., B_9$。
- **预处理**：将所有待比较正整数的位数统一为相等长度，位数少的整数前面补零。
- **排序步骤**：
  1. **分配**：将右起第$j$位数字相同的数放入同一桶。例如，数字为1的数（若位数不同则左边补0，视为01）放入桶$B_1$，其余类推。
  2. **收集**：按$B_0, B_1, ..., B_9$的顺序收集各桶中的数。
- **重复过程**：重复步骤（1）和（2），从最右位（个位）到最左位（最高位）依次处理，直至所有位排序完成。
## 3. 算法分析

**时间复杂度**：
- 设待排序序列有$n$个记录，最长关键字为$d$位（需重复执行$d$趟“分配”与“收集”）
- 单趟分配时间复杂度：$O(n)$（遍历所有记录分配到桶）
- 单趟收集时间复杂度：$O(r)$（$r$为基数，十进制下$r=10$，需遍历所有桶收集）
- 总时间复杂度：$O(d(n + r))$

**空间复杂度**：$O(n + r)$。空间消耗包括存储每轮排序结果的$n$个记录，以及记录每个基数出现频率的计数数组（大小为$r$）。

**稳定性**：是稳定的排序方法（分配和收集过程中，相同关键字的相对顺序不会改变）。
