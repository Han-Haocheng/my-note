# 插入排序

## 直接插入排序

直接插入排序：基本操作是将一个待排序的记录按其关键字的大小插入到一个已经完成排序的有序表中，从而得到一个新的，记录数增1的有序表。

算法思路：边插入边排序，保证子序列中随时都是排好序的
- 通过构建有序序列，对于未排序数据，在已排序序列中从后向前扫描
- 从而找到相应的位置（基于顺序查找）并插入
- 在从后向前扫描过程中，需要反复把已排序的元素逐步向后挪位，为待插入的新元素提供插入空间


排序过程：整个排序过程为$n-1$趟插入，即先将序列中第1个记录看成是一个有序子序列，然后从第2个记录开始，逐个进行查找与插入，直至整个序列有序。


```c
void InsertSort (SqList &L) {  // 对顺序表工作直接插入排序
    for(i=2; i<=L.length; ++i) {
        if(L.r[i].key >= L.r[i-1].key) continue;
        
		//"<",需将L.r[i]插入有序子表
		L.r[0]=L.r[i];          // 复制为哨兵
		L.r[i]=L.r[i-1];
		for(j=i-2; L.r[0].key< L.r[j].key; --j)
			L.r[j+1]=L.r[j];    // 记录后移
		L.r[j+1]=L.r[0];        // 插入到正确位置
	}
}
```

算法分析：
- 最好情况：待排序列为正序，每个元素仅需比较一次且无需移动（比较和移动次数最少），时间复杂度为$O(n)$
- 最坏情况：待排序列为逆序，每个新元素都需要与已排序序列中的所有元素进行比较并移动（比较和移动次数最多），时间复杂度为$O(n^2)$
- 平均情况：时间复杂度$O(n^2)$
- 空间复杂度：$O(1)$
- 稳定性：是稳定的排序方法


## 折半插入排序

折半插入排序：查找插入位置的方法改为折半查找，减少关键字间的比较次数，以此来提升排序的性能


```c
void InsertSort (SqList &L) {
    int i,j;
    for(i=2; i<=L.length; ++i) {
        L.r[0]=L.r[i];          // 将L.r[i]暂存到L.r[0]
        int low=1;
        int high=i-1;
        while(low<=high) {
            int mid=(low+high)/2;    // 在r[low...high]中折半查找有序插入的位置
            if(L.r[0].key < L.r[mid].key)
                high=m-1;           // 插入点在低半区
            else
                low=mid+1;          // 插入点在高半区
        }
		for(j=i-1;j>=high+1;--j)
			L.r[j+1]=L.r[j];     // 记录后移
		L.r[high+1]=L.r[0];      // 插入
    }
}
```



算法分析：
- 折半插入排序减少了比较次数，但没有减少移动次数。
- 时间复杂度为$O(n^2)$，空间复杂度：$O(1)$
- 稳定性：是稳定的排序方法


## 希尔排序

希尔排序：先将整个待排记录序列分割成若干子序列，分别进行直接插入排序，待整个序列中的记录“基本有序”时，再对全体记录进行一次直接插入排序

子序列不是简单“逐段分割”，而是将相隔增量$d_i$的记录组成一个子序列，$d_i$逐趟缩短（比如选取2的整数次方，如4、2、1，所以又称缩小增量排序），在对各子序列排序过程中，小元素是在跳跃式前移，所以平均性能要优于直接插入排序。
$d_k$值较大，子序列中对象较少，速度较快；$d_k$值逐渐变小，子序列中对象变多，但大多数对象已基本有序，所以排序速度仍然很快。
```c
void ShellInsert (SqList &L,int dk) {
    // 对顺序表工作一趟希尔插入排序。本算法是和一趟直接插入排序相比，做了以下修改
    // 前后记录位置的增量是w，而不是1
    // r[0]只是暂存单元，不是哨兵。当<=0时，插入位置已找到
    int i,j;
    for(i=dk+1;i<=L.length;++i) {
        if(L.r[i].key>= L.r[i-dk].key)
			continue;
		// 需将L.r[i]插入有序增量子表
		L.r[0]=L.r[i];          // 暂存在L.r[0]
		for(j=i-dk; j>0&&L.r[0].key<L.r[j].key;j-=dk)
			L.r[j+dk]=L.r[j];   // 记录后移，查找插入位置
		L.r[j+dk]=L.r[0];       // 插入
    }
}
void ShellSort(SqList &L,int dlta[],int t) {
    // 按增量序列dlta[0…t-1]对顺序表L做希尔排序
    for(k=0;k<t;++k)
        ShellInsert(L,dlta[k]);     // 一趟增量为dlta[k]的插入排序
}
```

算法分析：
- 时间复杂度：由于不同增量序列的“分组策略”差异极大，时间复杂度无统一结论，增量序列的选择直接决定了希尔排序的效率上限。通常认为希尔排序的时间复杂度约为$O(n^{1.3})$
- 空间复杂度：$O(1)$
- 稳定性：是不稳定的排序方法
