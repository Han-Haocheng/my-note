# 数


定点数：小数点的位置是固定不变的
- 定点小数：小数点位于符号位（如果有符号）之后，所有数值之前
- 定点整数：小数点位于数值位之后

浮点数：小数点的位置通过


- 真值：带符号的数，是真正的数值
- 机器数：存储在机器内的数，符号数字化的数，小数点位置是约定的

| 真值        | 机器数       |
| --------- | --------- |
| $+0.1011$ | $0 .1011$ |
| $-0.1011$ | $1 .1011$ |
| $+1100$   | $0,1100.$ |
| -1100     | $1,1100.$ |



## 无符号数

无符号数：只有数值位，没有符号位，只能表示从 0 开始的数

## 有符号数

有符号数：包含数值和一位符号，通常以 0 为中心


### 原码表示法

原码表示法：通过在二进制数的最高位（符号位）来表示数的正负，0 表示正数，1 表示负数，其余位表示数值的绝对值
- 整数： $[x]_原=\begin{cases}0,x &  2^n>x \ge 0\\2^n-x &0\ge x >-2^n \end{cases}$
	- 真值x
	- 整数的位数n
	- 表示范围：$[-2^n-1, 2^n-1]$
	- 记作：用逗号将符号位和数值部分隔开
- 小数：$[x]_原=\begin{cases}x&1>x≥0\\1-x&0≥x>-1\end{cases}$
	- 记作：用**小数点**将符号位和数值部分分开

| 真值            | 原码        |
| ------------- | --------- |
| $-0.0011$     | $1. 0011$ |
| $-1100$       | $1, 1100$ |
| $+0.1101$     | $0.1101$  |
| $[+0.0000]_原$ | $0.0000$  |
| $[-0.0000]_原$ | $1.0000$  |
| $[+0]_原$      | $0, 0000$ |
| $[-0]_原$      | $1, 0000$ |

特点：简单直观、不易运算

|要求|数1|数2|实际操作|结果符号|
|-|-|-|-|-|
|加法|正|正|加|正|
|加法|正|负|减|可正可负|
|加法|负|正|减|可正可负|
|加法|负|负|加|负|

### 反码表示法

反码表示法：用于表示负数。对于正数，其反码与其原码相同。对于负数，其反码是将原码除符号位外的所有位取反（0 变 1，1 变 0）

- 整数：$[x]_{反}=\begin{cases}0,x&2^n>x≥0\\(2^{n+1-1}+x&0≥x>-2^n\end{cases}$
- 小数：$[x]_反=\begin{cases}x&1>x≥0\\(2-2^{-n})+x&x≥x>-1(\mod2-2^{-n})\end{cases}$

| 真数      | 反码                         |
| ------- | -------------------------- |
| +1101   | 0, 1101                    |
| -1101   | $(2^{4+1}-1)-1101=1, 0010$ |
| $[+0]$  | $[0,0000]$                 |
| $[-0]$  | $[1, 1111]$                |
| +0.1101 | 0.1101                     |
| -0.1010 | $(2-2^{-4})-0.1010=1.0101$ |

### 补码表示法

推理过程
- 将时钟进行调整：逆时针6-3=3；顺时针6+9=15-12=3
- 可见-3可以用+9代替**将减法转化为加法**
- 称+9是-3以12为模的**补数**
- 记作$-3\equiv+9(\mod 12)$
- 同理$-4\equiv+8(\mod 12)$
- 结论
	- 一个负数加上模即为该负数的补数。
	- 一个正数和一个负数互为补数时，它们的绝对值之和即为模数。
- 正数的补数即为其本身

补码表示法：为了简化计算机中的加减运算，使得符号位和数值位可以统一处理
- 其中 $[+0]_补=[-0]_补$

- 整数：$[x]_补\begin{cases}0,x&2^n>x≥0\\2^{n+1}+x&0>x≥-2^n(\mod2^{n+1})\end{cases}$
- 小数：$[x]_补=\begin{cases}x&1>x≥0\\2+x&0>x≥-1(\mod2)\end{cases}$

| 真值          | 补数                           |
| ----------- | ---------------------------- |
| +1010       | $0, 1010$                    |
| -101 1000   | $1,010 1000$                 |
| +0.1110     | $0.1110$                     |
| -0.110 0000 | $2+(-0.110 0000)=1.010 0000$ |

技巧：直接将最后一位值为1的前面所有的位取反，直接得到补码的数值部分


### 移码表示法

移码表示法：很难直接判断其真值大小

| 十进制 | 二进制  |   补码    |
| ------ | ------ | -------- |
| +21    | +10101 | 0,10101 |
| -21    | -10101 | 1,01011 |
| +31    | +11111 | 0,11111  |
| -31    | -11111 | 1,00001  |

### 定义

$[x]_移=2^n+x(2^n>x≥-2^n)$

小数部分比较阶数和数值部分，都是整数


|  真值   |         移码          |
| ------ | -------------------- |
| 10100  | $2^5+10100=1,110100$ |
| -10100 | $2^5-10100=0,011000$ |

### 移码和补码的比较

|   真值    |    补码     |    移码     |
| -------- | ---------- | ---------- |
| +1100100 | 1，1100100 | 0，1100100 |
| -1100100 | 0，0011100 | 1，0011100 |

补码与移码只差符号位

### 真值、补码和移码的对照表

| 真值x（n=5）     | $[x]_补$       | $[x]_移$       | $[x]_移$对应的十进制整数 |
| ------------ | ------------- | ------------- | --------------- |
| $-$ 10 0000  | $-$ 10 0000   | $-00 0000$    | 0               |
| $-$ 1 1111   | $-$ 10 0001   | $-00 0001$    | 1               |
| $-$ 1 1110   | $-$ 10 0010   | $-00 0010$    | 2               |
| ...          | ...           | ...           | ...             |
| $-$ 0 0001   | $-$ 11 1111   | $-01 1111$    | 31              |
| $\pm$ 0 0000 | $\pm$ 00 0000 | $\pm$10 0000$ | 32              |
| $+$ 0 0001   | $+$ 00 0001   | $+10 0001$    | 33              |
| $+$ 0 0010   | $+$ 00 0010   | $+10 0010$    | 34              |
| ...          | ...           | ...           | ...             |
| $+$ 1 1110   | $+$ 01 1110   | $+11 1110$    | 62              |
| $+$ 1 1111   | $+$ 01 1111   | $+11 1111$    | 63              |

### 特点

- 当x=0时$[+0]_移=2^5+0=1,0 0000$

    $[-0]_移=2^5-0=1,0 0000$

    $[+0]_移=[-0]_移$

- 当$n=5$时，最小的真值为$-2^5=-10 0000$

    $[-10 0000]_移=2^5-10 0000=00 0000$

    可见，**最小真值的移码全为0**

    用移码表示浮点数的阶码

## 数的编码

### BCD 码

BCD 码：用于表示 10 进制的数，每一位十进制数对应 4 位二进制数
- $[34]_{10}=[0011\,\,0100]_{BCD}$
