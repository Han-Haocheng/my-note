# 数据通信基础知识

## 数据通信系统的模型

![[工程科学/计算机网络/attachments/Pasted image 20250915084918.png]]

数据通信系统：由源系统（发送端、发送方）、传输系统（传输网络）、目的系统（接收端、接收方）组成 #通信/系统

消息message：用于通信传送的内容 #通信/系统
- 如话音、文字、图像、视频等

数据data：运送消息的实体 #通信/系统
- 根据 RFC 4949 给出的定义，数据是使用特定方式表示的信息，通常是有意义的符号序列，这种信息的表示可用计算机或其他机器（或人）处理或产生

信号signal：数据的电气或电磁的表现 #通信/系统
- 模拟信号analogous signal（连续信号）：—代表消息的参数的取值是<font color="#c0504d">连续</font>的
- 数字信号digital signal（离散信号）：代表消息的参数的取值是<font color="#c0504d">离散</font>的


码元：在使用时间域（时域）的波形表示数字信号时，代表不同离散数值的==基本波形== #通信/系统
- 二进制编码：两种码元，代表 0 的状态和代表 1 的状态。


### 源系统

源系统：由源点和发送器组成 #通信/系统

源点source(源站、信源) ：源点设备产生要传输的数据 #通信/系统
- 例如，从计算机的键盘输入汉字， 计算机产生输出的数字比特流

发送器：源点生成的数字比特流通过发送器编码后在传输系统中进行传输 #通信/系统
- 调制器：计算机使用内置调制解调器（包含调制器和解调器），用户在计算机外面看不见调制解调器

### 目的系统

目的系统：由接收器和终点组成 #通信/系统

接收器：接收传输系统信号，转为能被目的设备处理的信息 #通信/系统
- 解调器：它把来自传输线路上的模拟信号进行解调，提取出在发送端置入的消息，还原出发送端产生的数字比特流。 

终点destination(目的站，信宿) ：从接收器获取传送来的数字比特流，然后把信息输出 #通信/系统
- 举例：计算机接收汉字数据并把汉字在计算机屏幕上显示出来

## 信道

信道(channel)：表示向某一个方向传送信息的媒体 #通信/信道 

### 通信方式

通信方式：单工通信、半双工通信、双工通信  #通信/方式 

单向通信（单工通信）：只能有一个方向的通信而没有反方向的交互，只需要一条信道
- 举例：无线电广播、有线电广播、电视广播 

双向交替通信（半双工通信）：通信双方都可以发送信息，但不能双方同时发送/接收。一方发送另一方接收，过一段时间后可以再反过来。需要两条信道

双向同时通信（全双工通信）：通信双方可以同时发送/接收信息。 需要两条信道。传输效率最高

## 基带信号

基带信号（基本频带信号）：来自信源的信号，包含有较多的低频成分，甚至有直流成分 #通信/信号

调制modulation：解决许多信道并不能传输这种低频分量或直流分量问题  #通信/信号
- 调制方式：基带调制、带通调制

### 基带调制

![[工程科学/计算机网络/attachments/Pasted image 20250915093217.png]]

基带调制：仅仅对基带信号的波形进行变换，使它能够与信道特性相适应。变换后的信号仍然是基带信号。把==数字信号转换为另一种形式的数字信号==，把这种过程称为==编码(coding)== #通信/信号/基带调制
- 基带调制方式：归零码、不归零码、曼彻斯特编码、差分曼彻斯特编码 #通信/信号/基带调制

- 不归零码：正电平代表 1，负电平代表 0 #通信/信号/基带调制
- 归零制 ：正脉冲代表 1，负脉冲代表 0 #通信/信号/基带调制
- 曼彻斯特编码：位周期中心的向上跳变代表 0，位周期中心的向下跳变代表 1。 可反定义 #通信/信号/基带调制
- 差分曼彻斯特编码：在每一位的中心处始终都有跳变。位开始边界有跳变代表 0，而位开始边界没有跳变代表 1 #通信/信号/基带调制

| 编码类型   | 信号频率 | 自同步 |
| ------ | ---- | --- |
| 不归零制   | 低    | 不行  |
| 曼彻斯特编码 | 高    | 可以  |

自同步：从信号波形本身中提取信号时钟频率

### 带通调制


![[工程科学/计算机网络/attachments/Pasted image 20250915095801.png]]
带通调制：用<font color="#c0504d">载波(carrier)</font>进行调制，把基带信号的频率范围搬移到较高的频段，并<font color="#00b0f0">转换为模拟信号</font>  #通信/信号/带通调制
- 带通信号：经过载波调制后的信号，仅在一段频率范围内能够通过信道
- 带通信号调制方式：调幅、调频、调相 #通信/信号/带通调制

- 调幅AM：即载波的振幅随基带数字信号而变化 #通信/信号/带通调制
	- 举例：0 或 1 分别对应于无 载波或有载波输出。
- 调频FM：即载波的频率随基带数字信号而变化 #通信/信号/带通调制
	- 举例：0 或 1 分别对应于频率 f1 或 f2。 
- 调相PM：即载波的初始相位随基带数字信号而变化 #通信/信号/带通调制
	- 举例：0 或 1 分别对应于 相位 0 度或 180 度
- 正交振幅调制 QAM (Quadrature Amplitude Modulation)：更为复杂的多元制的振幅相位混合调 制方法 #通信/信号/带通调制

## 信道极限容量


![[工程科学/计算机网络/attachments/Pasted image 20250915103505.png]]

#通信/信道/容量极限 原因

- 任何实际的信道都不是理想的，都不可能以任意高的速率进行传送
- 码元传 输的速率越高，或信号传输的距离越远，或噪声干扰越大，或传输媒体质量越差，在接收端 的波形的失真就越严重

#通信/信道/容量极限 提升方法

- 限制码元传输速率：信道能够通过的频率范围、信噪比
- 提高信息的传输速率：让每一个码元携带更多比特的信息量
- 信道的带宽或信道中的信噪比越大，信息的极限传输速率就越高


#通信/信道/容量极限 计算：$C=W \log_{2}{1+\frac{S}{N}}(bit/s)$
- W：信道的带宽（以 Hz 为单位）
- C：信道的极限信息传输速率
- S：信道内所传信号的平均功率
- N：信道内 部的高斯噪声功率

#通信/信道/容量极限 奈氏准则：激励工程人员不断探索更加先进的编码技术，使每一个码元携带更多比特的信息量。

#通信/信道/容量极限 香农公式：告诫工程人员，在实际有噪声的信道上，不论采用多么复杂的编码技术，都不可能突破信息传输速率的绝对极限。


### 信道能够通过的频率范围

#通信/信道/频率范围

具体的信道所能通过的频率范围总是有限的。信号中的许多高频分量往往不能通过信 道

码间串扰：接收端收到的信号波形就<font color="#c0504d">失去</font>了码元之间的清晰界限

奈氏准则：码元传输的最高速率=2W(码元/秒)
- W：带宽

在任何信道中，码元传输的速率是有上限的，传输速率超过此上限，就 会出现严重的码间串扰的问题，使接收端对码元的判决（即识别）成为不可能。

### 信噪比

#通信/信道/信噪比 定义：信号的平均功率和噪声的平均功率之比
#通信/信道/信噪比 计算：$信噪比(dB)=10\log_{10}{\frac{S}{N}}(dB)$
