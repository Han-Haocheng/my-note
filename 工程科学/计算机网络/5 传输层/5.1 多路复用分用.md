# 多路复用分用

![[工程科学/计算机网络/attachments/2025-10-31_135529.svg]]

多路复用/分用：解决一个协议对应直接上层的多个协议实体冲突问题

1. 主机接收到的IP数据报datagram：携带源IP地址、目的IP地址、一个传输层的段Segment。每个段携带源端口号和目的端口号
2. 主机收到Segment：传输层协议提取IP地址和端口号信息，将Segment导向响应的Socket。TCP协议处理更多问题

## 无连接分用

一个端口与一个端口连接

```c++
DatagramSocket mySocket1 = new DatagramSocket(99111);
DatagramSocket mySocket2 = new DatagramSocket(99222);
```

1. 利用端口号创建Socket
2. UDP 的Socket用二元组标识：`(目的IP, 目的端口)`
3. 主机收到UDP：检查段中的目的端口号，将UDP段导向绑定在该端口号的Socket
4. 来自不同源IP地址和源端口号的IP数据报被导向同一个Socket

## 面向连接分用

一个端口与多个端口连接

1. TCP的Socket用四元组标识：`(源IP, 源端口, 目的IP, 目的端口)`
2. 接收端利用所有的四个值将Segment导向合适的Socket
3. 服务器可能同时支持多个Socket：每个Socket用自己的四元组标识
	- 例如：Web服务器为每个客户端开不同的Socket
