# 内存使用与分段

## 内存使用


```c
int main(int argc,char*argv[]){
	int i,to,sum = 0;
	to =atoi(argv[1]);
	for(i = 0;i<=to;++i){
		sum += i;
	}
	printf("%d",sum);
	return 0;
}
```

- 编译程序

```
.text
_entry:
	call _main          ; call 40
	call _exit
_main:                ; addr-40
	mov [300],0
	ret
```

- 内存使用：将程序存放到内存中，PC指向开始地址
- 内存指针需要根据程序的位置重新定位

```
		.text
```

### 编译时重定位

在编译时期，定位程序存放位置
- 优点：简单直接，效率高
- 缺点：系统运行死板，不能交换运行程序
- 适用：固定系统的单片机

### 运行时重定位


在程序从磁盘中载入时，根据程序PCB中保存的基地址(base)，进行重定位
- 优点：程序运行灵活，
- 缺点：效率降低，内存利用率低
- 地址翻译：每执行一条指令，要根据基地址和偏移地址计算出物理地址
- 交换(swap)：当程序进行等待时，将程序存放进磁盘，将就绪的程序从磁盘中加载到内存中

## 内存分段

分段：将程序，根据用途和特点划分为不同的段，其中代码段/数据段不会增长

内存分段：将程序各个段，分别放入内存，通过段地址+偏移获取对应地址，通过PSD中的进程段表保存对应的地址址
