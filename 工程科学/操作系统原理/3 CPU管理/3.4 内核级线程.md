## 内核级线程

![[工程科学/操作系统原理/attachments/20250704142716.svg]]


内核级线程（Kernel Threads）：线程由内核管理，发挥多核并行性能

### 内核栈和用户栈

- 在用户态执行的代码，使用的是用户栈
- 在内核态执行的代码，使用的是内核栈，并且与用户栈进行关联
	- 包含：用户栈`SS:SP`，执行函数地址

```c
100:A(){
	B();
104:}
```

```c
200:B(){
	read();
204:}
```

```c
300:read(){
	int 0x80;
304:}
```

```asm
system_call:
	call sys_read;
1000:
2000:sys_read(){}
```

### 内核线程切换

![[工程科学/操作系统原理/attachments/20250704150100.svg]]

```c
500:C(){

}
interrupt:
	call sys_xxx;
3000:
4000:sys_xxx(){}

```

```c
sys_read(){
	//启动硬盘读
	//将自己变成阻塞;
	//找到TCB next
	switch_to(cur,next);
}
```


```c
interrupt_entry:
	push ds;
	...
	pusha;
	mov ds,内核段号
	...
	call int_process
	popa
	...
	pop ds
	iret

interrupt_process:
	int //磁盘读 时钟
	call schedule
	ret

schedule:
	mov ..,next
	call switch_to
	ret

void switch_to(cur,next){
	mov %esp,TCB[cur].esp
	mov TCB[next],exp,%esp
	ret
}
```

### 内核线程创建

创建函数是系统调用，由内核管理TCB，并负责切换线程
- 包含：内核栈，TCB

```c
void thread_create(){
	TCB tcb = get_free_page();
	*krlstack = ...;
	*userstack ;
	tcb.esp=krlstack;
	tcb.state = ready;
	Queue.push(tcb);
}
```
