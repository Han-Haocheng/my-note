- 操作系统记录进程，并按照一定次序分配资源、调度

```c
int main(int argc,char*argv[]){
	//....

		if(!fork()){

}
```

多进程组织：PCB+状态+队列
- PCB（Process Control Block）：记录进程信息的数据结构
- 状态：新建态、就绪态、运行态、阻塞态、终止态
- 队列：就绪队列、等待队列

### 多进程交替

多进程交替：队列操作+调度+切换

#### 切换流程

1. 启动磁盘读写
2. `pCur.state = 'W';`
3. 将`pCur`放到`DiskWaitQueue;`
4. `schedule();`

```c
void schedule(){
	pNew = getNext(ReadyQueue);
	switch_to(pCur, pNew);
}
```

#### 调度方式

调度
- FIFO：公平策略，没有任务区别
- Priority：优先级策略，会产生饥饿进程

#### 切换方式

```c
void switch_to(pCur,pNew){
	pCur.ax = CPU.ax;
	pCur.bx = CPU.bx;
	//...
	pCur.cs = CPU.cs;

	CPU.ax = pNew.ax;
	CPU.bx = pNew.bx;
	//...
	CPU.cs = pNew.cs;
	CPU.retpc = pNew.pc;
}

```

### 多进程影响

问题：多进程同时存在于内存中，会导致进程跨越访问另一个进程数据

解决方式：在每个进程中建立一个映射表，映射代码的数据访问


### 多进程合作

打印过程
1. 应用进程提交打印任务
2. 打印任务放进打印队列
3. 打印进程从队列中取出任务
4. 打印进程控制打印机打印

问题：进程会抢夺打印资源，类比于消费者-生产者问题
解决方式：进程同步，在写入共享资源时，阻断其他进程访问该资源
