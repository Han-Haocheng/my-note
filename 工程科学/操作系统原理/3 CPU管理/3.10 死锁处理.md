# 死锁处理

## 死锁

死锁：多个进程互相等待对方持有的资源而造成谁都无法执行

```
Produce（item){
	P(mutex);
	P(empty);
	// 临界区
	V(mutex);
	V(full);
}

Consumer(){
	P(mutex);
	P(full);
	// 临界区
	V(mutex);
	V(empty);
}
```

死锁产生原因：资源互斥使用，进程占用资源不释放再去申请其他资源

死锁产生条件
- 互斥使用（Mutual exclusion）：资源固有特性
- 不可抢占（No preemption）：资源只能自愿放弃
- 请求和保持（Hold and wait）：进程必须占有资源再去申请
- 循环等待（Circular wait）：在资源分配图中存在一个环路

## 处理死锁方式

- 死锁预防：破坏死锁出现的条件
- 死锁避免：检测每个资源请求，如果造成死锁就拒绝
- 死锁检测恢复：检测到死锁出现时，让一些进程回滚，让出资源
-  死锁忽略

### 死锁预防

一次性申请所有需要的资源，不会占有资源再去申请其他资源
- 编程困难：需要预读代码
- 资源利用率低：许多资源分配后很长时间后才使用

对资源类型进行排序，资源申请必须按序进行，不会出现环路等待
- 资源浪费

### 死锁避免

安全状态：系统中的所有进程存在一个可完成的执行序列P1...Pn
- 安全序列：可完成的执行序列P1...Pn

#### 银行家算法

```c++
int Available[m];
int Allocation[n][m];
int Need[n][m];
int Work[m] = {Available};
bool Finish [n]={false};

while(true){
	for(i=1;i<n;i++){
		if(Finish[i]==false&&Need[i]<=Work){
			Work= Work+Allocation[i];
			Finish[i] = true;
			break;
		}
		for(i=1;i<=n;i++)
			if(Finish[i]==false) return "deadlock";
	}
}
```

- 时间复杂度T(n)：$O(mn^2)$
