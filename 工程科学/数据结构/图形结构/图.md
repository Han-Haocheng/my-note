# 图

图Graph：有穷且非空的顶点集合，及顶点间的关系集合（边集合）组成的一种数据结构，记$G=(V, E)$
- 其中G表示图
- V是顶点的非空有限集合
- E是V中顶点偶对的有限集合

子图：设图$G=(V,E)$和$G'=(V',E')$，若$V' \subseteq V$且$E' \subseteq E$，图$G'$是图$G$的子图

## 1 图的顶点

- 顶点Vertex：图中的数据元素

### 1.1 顶点的邻接

顶点的邻接
- 对于无向图$G=(V,E)$，如果边$(v,w) \in E$，则称顶点$v$、$w$互为邻接点（即$v$与$w$相邻接），边$(v,w)$依附于顶点$v$和$w$。 

- 对于有向图而言，若弧$<v,w> \in E$，则称顶点$v$邻接到顶点$w$，顶点$w$邻接自顶点$v$，或者说弧$<v,w>$与顶点$v$、$w$相关联

### 1.2 顶点的度

- **无向图顶点的度**：与顶点$v$关联的边的数目，记作$\deg(v)$。
	- #计算机/数据结构/公式 $无向图顶点度的和=2\times边$  
- **有向图顶点的度**
	- **顶点的入度**：是以顶点$v$为弧头(==终点==)的弧的数目，记作$\text{indeg}(v)$ 
	- **顶点的出度**：是以顶点$v$为弧尾(==起点==)的弧的数目，记作$\text{outdeg}(v)$
	- #计算机/数据结构/公式 $有向图顶点度的和\deg(v)=入度\text{indeg}(v)+出度\text{outdeg}(v)$

## 2 图的路径

路径Path：连续的边构成的顶点序列
- 路径长度Path length：路径上边或弧的数目之和
- 简单路径Simple path：在一条路径中，没有重复相同的顶点
### 2.1 回路

回路Cycle（环）：第一个顶点和最后一个顶点相同的路径
- 简单回路Simple cycle（简单环）：除路径起点和终点相同外，其余顶点均不相同的回路

## 4 抽象数据类型
### 4.1 数据对象
图的顶点集合 $V$。$V$ 是具有相同特性的数据元素的集合，称为顶点集。
### 4.2 数据关系

图的边的集合 $E$。
$E$ 是 $V$ 上二元关系的集合，通常表示为：
*   若 $<v, w> \in E$，则 $<v, w>$ 表示从 $v$ 到 $w$ 的一条弧（有向边），且 $v$ 称为弧尾，$w$ 称为弧头。此时图称为有向图。
*   若 $(v, w) \in E$，则 $(v, w)$ 表示 $v$ 和 $w$ 之间的一条边（无向边）。此时图称为无向图。
### 4.3 基本操作
假设图 $G$ 的类型为 `Graph`，顶点类型为 `VertexType`，边的相关类型（如权重）为 `InfoType`。
*   **CreateGraph(&G, V, VR)**
    *   **初始条件**：$V$ 是图的顶点集，$VR$ 是图中边的集合。
    *   **操作结果**：按 $V$ 和 $VR$ 的定义构造图 $G$。
*   **DestroyGraph(&G)**
    *   **初始条件**：图 $G$ 已存在。
    *   **操作结果**：销毁图 $G$，释放其占用的存储空间。
*   **LocateVex(G, u)**
    *   **初始条件**：图 $G$ 存在，$u$ 和 $G$ 中顶点有相同特征。
    *   **操作结果**：若 $G$ 中存在顶点 $u$，则返回该顶点在图中的位置；否则返回其它信息（如 -1 或 NULL）。
*   **GetVex(G, v)**
    *   **初始条件**：图 $G$ 存在，$v$ 是 $G$ 中某个顶点。
    *   **操作结果**：返回 $v$ 的值。
*   **PutVex(&G, v, value)**
    *   **初始条件**：图 $G$ 存在，$v$ 是 $G$ 中某个顶点。
    *   **操作结果**：对 $v$ 赋值 value。
*   **FirstAdjVex(G, v)**
    *   **初始条件**：图 $G$ 存在，$v$ 是 $G$ 中某个顶点。
    *   **操作结果**：返回 $v$ 的第一个邻接顶点。若 $v$ 没有邻接顶点，则返回“空”。
*   **NextAdjVex(G, v, w)**
    *   **初始条件**：图 $G$ 存在，$v$ 是 $G$ 中某个顶点，$w$ 是 $v$ 的邻接顶点。
    *   **操作结果**：返回 $v$ 的（相对于 $w$ 的）下一个邻接顶点。若 $w$ 是 $v$ 的最后一个邻接点，则返回“空”。
*   **InsertVex(&G, v)**
    *   **初始条件**：图 $G$ 存在，$v$ 和图中顶点有相同特征。
    *   **操作结果**：在图 $G$ 中增添新顶点 $v$。
*   **DeleteVex(&G, v)**
    *   **初始条件**：图 $G$ 存在，$v$ 是 $G$ 中某个顶点。
    *   **操作结果**：删除 $G$ 中顶点 $v$ 及其相关的弧/边。
*   **InsertArc(&G, v, w)**
    *   **初始条件**：图 $G$ 存在，$v$ 和 $w$ 是 $G$ 中两个顶点。
    *   **操作结果**：在 $G$ 中增添弧 $<v, w>$（若是有向图）或边 $(v, w)$（若是无向图）。
*   **DeleteArc(&G, v, w)**
    *   **初始条件**：图 $G$ 存在，$v$ 和 $w$ 是 $G$ 中两个顶点。
    *   **操作结果**：在 $G$ 中删除弧 $<v, w>$（若是有向图）或边 $(v, w)$（若是无向图）。
*   **DFSTraverse(G, Visit())**
    *   **初始条件**：图 $G$ 存在，Visit 是顶点的应用函数。
    *   **操作结果**：对图进行深度优先遍历。在遍历过程中对每个顶点调用函数 Visit 一次且仅一次。一旦 Visit() 失败，则操作失败。
*   **BFSTraverse(G, Visit())**
    *   **初始条件**：图 $G$ 存在，Visit 是顶点的应用函数。
    *   **操作结果**：对图进行广度优先遍历。在遍历过程中对每个顶点调用函数 Visit 一次且仅一次。一旦 Visit() 失败，则操作失败。


## 5 图的存储结构

图存储的特点
- 无法用物理位置表示元素关系
- 顶点的度的差异很大

图存储的方式：
- 顺序存储：邻接矩阵
- 链式存储：邻接表、十字链表、邻接多重表

### 5.1 邻接矩阵

邻接矩阵：顺序存储方式，一个一维数组表示顶点（数据元素），一个二维数组表示边之间的关系（边、弧的信息）
- 图的邻接矩阵唯一
- 容易查找边和，不容易判断边的数量


```c
#define MAX_VERTEX_NUM 30

typedef char VertexType;
typedef int EdgeType;
typedef struct{
	VertexType vexs[MAX_VERTEX_NUM];
	EdgeType vdegs[MAX_VERTEX_NUM][MAX_VERTEX_NUM];
	int vexnum; // 顶点数量
	int arcnum; // 弧的数量
}MGraph;
```
#### 5.1.1 无向图邻接矩阵
无向无权图$G = (V, E)$有$n$（$n \geq 1$）个顶点，其邻接矩阵是$n$阶对称方阵
- 无向图邻接矩阵一定是对称矩阵，可压缩
- 对于顶点$V_i$，其度数是第$i$行(第$i$列)的非0元素(非$\infty$元素)的个数
- 无向图的边数是上(下)三角型矩阵中非0元素的个数


无向无权图：$A[i][j] = \begin{cases} 1 & (V_i, V_j) \in E \\0 & (V_i, V_j) \not\in E\end{cases}$

- 邻接矩阵
$$\begin{bmatrix}0 & 1 & 1 & 1 \\1 & 0 & 1 & 1 \\1 & 1 & 0 & 1 \\
1 & 1 & 1 & 0\end{bmatrix}$$

![[工程科学/数据结构/attachments/2026-01-28_204453.svg]]


无向带权图：$A[i][j] = \begin{cases} W_{ij} & (V_i, V_j) \in E \\\infty & (V_i, V_j) \not\in E\end{cases}$

![[工程科学/数据结构/attachments/Pasted image 20260128204651.png]]

#### 5.1.2 有向图邻接矩阵

有向图$G=(V,E)$有$n,(n\geq 1)$个顶点
- 对于顶点$V_i$，第i行的非0元素的个数是其出度$OD(V_i)$
- 第i列的非0元素的个数是其入度$ID(V_i)$
- 邻接矩阵中非0元素的个数就是图的弧的数目


有向无权图：$A[i][j] = \begin{cases} 1 & <V_i, V_j> \in E \\0 & <V_i, V_j> \not\in E\end{cases}$

有向带权图：$A[i][j] = \begin{cases} W_{ij} & <V_i, V_j> \in E \\\infty & <V_i, V_j)>\not\in E\end{cases}$

### 5.2 邻接表

邻接表：链式存储，每个顶点建立一个链表，存储邻接节点及相关信息，每个单链表设置一个表头，表头通过顺序结构存储

![[工程科学/数据结构/attachments/2026-01-28_211144.svg]]
- 数据域data：存储顶点$V_{i}$信息
- 度degree：表头节点出度或入度
- 链域firstarc：指向单链表的第一个节点
- 邻接点域adjvex：表示与$V_{i}$的邻接的顶点位置
- 数据域info：与边有关信息，通常是权值
- 链域nextarc：指向下一个与$V_{i}$邻接边节点


```c
#define MAX_VEX 30
// 边节点
typedef struct ArcNode {
    int adjvex;// 该弧所指向的顶点的位置
    struct ArcNode *nextarc; // 指向下一条边的指针
    int info;// 与边或弧相关的信息,如权值等
} ArcNode;

//顶点节点
typedef struct VNode {
    VertexType data;// 顶点信息
    ArcNode *firstarc;// 指向第一条依附该顶点的弧的指针
} VNode, AdjList[MAX_VEX];

typedef struct {
    AdjList vertices;
	int vexnum; // 顶点数量
	int arcnum; // 弧的数量
    int kind;// 图的种类标志
} ALGraph;
```

### 5.3 十字链表

十字链表：链式存储结构，多用于有向图，每个顶点有一个节点，==每一条弧也有一个节点==

![[工程科学/数据结构/attachments/2026-01-28_212206.svg]]

- 数据域data：存储和顶点相关信息
- 指针域firstin：指向该顶点为==弧头==的第一条弧所对应的弧结点
- 指针域firstout：指向该顶点为==弧尾==的第一条弧所对应的弧结点

![[工程科学/数据结构/attachments/2026-01-28_212452.svg]]

- 尾域tailvex：指示弧尾顶点在图中的位置
- 头域headvex：指示弧头顶点在图中的位置
- 指针域hlink：指向弧头相同的下一条弧
- 指针域tlink：指向弧尾相同的下一条弧
- info域：指向该弧的相关信息，如权值

## 6 图的操作

### 6.1 图的遍历

图的遍历Traversing Graph：从图的某一顶点出发，按照某种搜索方法沿着图的边访遍图中的其余顶点，且每个顶点仅被访问一次
- 被依赖算法：图的连通性问题、拓扑排序、求关键路径
- 如果给定的图是连通的无向图或是强连通的有向图，则遍历过程一次就能完成，并可按访问的先后顺序得到由该图所有顶点组成的一个序列
- 根据搜索方法不同：深度优先搜索、广度优先搜索

#### 6.1.1 深度优先搜索

深度优先搜索Depth First Search-DFS：类似树的先序遍历，是树的先序遍历的推广


步骤
1. 从图中某个顶点$V_{i}$出发，访问$V_{i}$
2. 然后找到$V_{i}$的一个邻接顶点$V_{i1}$
3. 从$V_{i1}$出发，深度优先搜索访问和$V_{i1}$相邻接且未被访问的所有顶点
4. 转1，直到和$V_{i}$相邻接的所有顶点都被访问为止
5. 继续选取图中未被访问的顶点$V_{j}$作为起始顶点转1，直到图中所有顶点都被访问为止

![[工程科学/数据结构/attachments/2026-01-29_213727.svg]]

`v1->V2->v4->v8->v5->v3-v6-v7`


基于[[#邻接矩阵]]的深度优先搜索


```c
void DFS(MGraph *G, int v,bool vflags[]){
    vflags[v] = true;
    // 在这执行代码是 标准DFS搜索序列
    for(j = 0; j < G->vexnum ;j++) {
	    if(vflags[j] == true || G->vdegs[v][j] != 1)
			continue;
        DFS(G, j,vflags);
    }
    // 在这执行代码是 拓扑有序序列
}
```

基于[[#邻接表]]的深度优先算法

```c
ArcNode *p;
void DFS(ALGraph *G, int v, bool vflags[]) {
    vflags[v] = true;          // 置访问标志，访问顶点v
    for(p = G->AdjList[v].firstarc; p!=NULL;p=p->nextarc){
	    // 从v的未访问过的邻接顶点出发深度优先搜索
        if (vflags[p->adjvex])
            continue;
        DFS(G, p->adjvex,vflags); 
    }
}

void DFS_traverse_Grapg(ALGraph *G) {
    int v;
    bool vflags[MAX_VEX]={0};
    
    for (v = 0; v<G->vexnum; v++){
		if (vflags[v]) 
			continue; // 如果执行到这，说明图是不联通的
        DFS (G, v,vflags);
    }

}
```

#### 6.1.2 广度优先搜索

广度优先搜索：Breadth First Search-BFS：遍历类似与树的层序遍历
- 时间复杂度为$O(n+e)$
	- 顶点数量n
	- 边数量e

1. 从图中某个顶点$V_i$出发，访问$V_i$
2. 访问$V_i$的所有相邻接且未被访问的所有顶点$V_{i1}$，$V_{i2}$，…，$V_{im}$
3. 以$V_{i1}$，$V_{i2}$，…，$V_{im}$的次序，以$V_{ij}$（$1 \leq j \leq m$）依次作为$V_i$，转1
4. 继续选取图中未被访问顶点$V_k$作为起始顶点，转1，直到图中所有顶点都被访问为止

基于[[#邻接表]]的广度优先搜索
```c
typedef struct Queue{
	int elem[MAX_VEX];
    int front, rear;
}Queue;
void BFS_traverse_Grapg (ALGraph *G){
    int k, v, w;
	bool vflegs[MAX_VEX] = {0};
    ArcNode *p;
    Queue *Q;
    Q=(Queue *)malloc(sizeof(Queue));
    Q->front=Q->rear=0;        // 建立空队列并初始化
    
    for(k=0; k<G->vexnum; k++){
	    v = G->vertices[k].data;
        if(vflegs[v])
	        continue;
	    Q->elem[++Q->rear]=v; // 将顶点入队
	    
		for(;Q->front!=Q->rear;){
			w = Q->elem[++Q->front]; // w出队
			vflegs[w]=true;  // 置访问标志
			Visit(w);        // 访问队首元素
			for(p=G->AdjList[w].firstarc;p != NULL;p = p->nextarc){
				if(vflegs[p->adjvex])
					continue;
				Q->elem[++Q->rear] = p->adjvex;
			}
		}
    }
}
```
