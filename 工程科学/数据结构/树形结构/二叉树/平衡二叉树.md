# 平衡二叉树

平衡二叉树（Balanced Binary Tree）：一棵二叉树的每个结点的平衡因子只可能是-1、0、1，即绝对值≤1的二叉树
- 平衡因子BF（Balance Factor）：某结点的左子树深度和右子树深度之差，用于衡量一棵二叉树的均衡，深度差值

平衡旋转：在平衡二叉树中插入一个新结点，会使之成为不平衡二叉树，将其重新调整为平衡状态的过程


### 平行化调整
平行化调整：对最小不平衡子树进行调整
- 最小不平衡子树：距离节点最近的不平衡子树

- LL型调整（以B为旋转轴）：若在A的左孩子的左子树上插入结点，使A的平衡因子从1增加至2，需要进行一次==顺时针旋转==![[工程科学/数据结构/attachments/2026-02-09_201134.svg]]
- RR型调整（以B为旋转轴）：若在A的右孩子的右子树上插入结点，使A的平衡因子从-1增加至-2，需要进行一次==逆时针旋转==![[工程科学/数据结构/attachments/Pasted image 20260209190552.png]]
- LR型调整（以C为旋转轴）：若在A的左孩子的右子树上插入结点，使A的平衡因子从1增加至2，需要先进行逆时针旋转，再顺时针旋转![[工程科学/数据结构/attachments/Pasted image 20260209191046.png]]
- RL型调整（以C为旋转轴）:若在A的右孩子的左子树上插入结点，使A的平衡因子从-1增加至-2，需要先进行顺时针旋转，再逆时针旋转![[工程科学/数据结构/attachments/Pasted image 20260209191602.png]]

举例：将序列$(13,24,37,,90, 53)$构成平衡二叉排序树

# 红黑树


红黑树RBT（RB-Tree）：栖牲严格的平衡，换取插入、删除操作时少量的旋转操作，并每个结点额外存储color字段（RED or BLACK）用于确保树在插入和删除时保持相对的平衡，一种自平衡的二叉搜索树

- 查找、插入和删除时间：$O(log_{2}n)$内完成


![[工程科学/数据结构/attachments/2026-02-09_192310.svg]]

## 红黑树的特性

- 颜色特征：结点为红色或黑色  
- 根特征：根结点是黑色  
- 外部特征：每个叶子结点都是黑色的空结点  
- **内部特征**：每个红色结点的两个子结点都是黑色的，即不允许两个连续的红色结点  
- 深度特征：从任意结点到其每个叶子结点的所有路径上的黑色结点数量相同（黑高一致）

![[工程科学/数据结构/attachments/Pasted image 20260209192718.png]]
- NIL(Not In List)：表示终止

### 红黑树的插入

红黑树的插入：新结点初始为红色，完成插入后需根据插入结点及相关结点的状态进行修正以满足上文提到五条性质

- 令插入的结点为N，其父结点为P，祖父结点为G，叔结点为U
- 从插入的位置开始向上递归维护，若P为黑色即可终止，否则分为 3 种情况来进行调整

1. P和U均为红色：需重新染色![[工程科学/数据结构/attachments/Pasted image 20260209193137.png]]
2. P为红色，U为黑色，P的方向和N的方向相同：旋转G结点以将P转为子树的根，之后交换P和G的颜色![[工程科学/数据结构/attachments/Pasted image 20260209193332.png]]
3. P为红色，U为黑色，P的方向和N的方向不同。此时我们需要旋转P结点来转为第2种情况![[工程科学/数据结构/attachments/Pasted image 20260209193726.png]]
