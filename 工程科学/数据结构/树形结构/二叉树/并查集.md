# 并查集

并查集：使用树形数据结构，处理多个不相交集合的合并和查询问题
- 代表元：用一棵树来代表一个集合，树的根结点
- 判断两个元素是否属于同一集合，只需看它们是否有相同的根结点（代表元）；
- 合并：将其中一棵树的根结点连接到另一棵树的根结点上
- 特点：擅长解决涉及动态连通性的问题，特别是在需要频繁进行连通判断和集合合并的场景
- 只能判断联通与否，不能回答具体的联通路径

## 并查集操作

三个操作的协同工作，使得并查集能够高效地维护动态变化的联通关系

`make_set(x)`：建立一个新的集合
- 它的唯一成员是x，也即根结点（或叫代表元），因为各个集合是不相交的，故x不会出现在别的某个集合中

`find_set(x)`：返回包含x的集合的代表元
- 此操作是判断两个元素是否属于同一集合的基石
- 用于直接回答连通性问题
- 是union操作能够正确执行的前提

`union(x, y)`：将两个元素所在的集合合并为一个集合
- 将包含x和y的两个动态集合$S_{x}$和$S_{y}$进行合并，成为一个新的集合
- 先通过两次查找，判断元素是否属于同一个集合
