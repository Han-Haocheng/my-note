# 二叉搜索树

二叉搜索树BST（Binary Search Tree、二叉排序树、二叉查找树）：或者是一棵空树，或者是具有下列性质的[[工程科学/数据结构/树形结构/二叉树/二叉树|二叉树]]
- 若左子树不空，则左子树上所有结点的值均小于根结点的值
- 若右子树不空，则右子树上所有结点的值均大于根结点的值
- 左右子树是二叉搜索树

- 通常采用二叉链表进行存储。
- 中序搜索序列是有序递增的


## 1 二叉搜索树的操作
### 1.1 查找元素


1. 将给定值key与树的根结点关键字进行比较，若相等则查找成功；否则执行以下操作：
	- 若给定值key小于根结点关键字，则在以该树的左孩子为根结点的子树上执行②
	- 若给定值key大于根结点关键字，则在以该树的右孩子为根结点的子树上执行②


#### 1.1.1 递归查找算法
```c
BSTNode * BST_Serach(BSTNode *T，KeyType key){
    // 递归查找某关键字等于key的数据元素
    if (T == NULL) return(NULL);
	if(T->key == key) return(T);
	
	if( key < T->key )
		return BST_Serach(T->Lchild，key);
	else 
		return BST_Serach(T->Rchild，key);
}
```


#### 1.1.2 查找性能分析

查找性能：与树的结构相关
- 最好时间复杂度：$\log_2 n$（越接近[[工程科学/数据结构/树形结构/二叉树/完全二叉树]]越好）
- 最坏时间复杂度：$(n+1)/2$（单支树）
- 平均查找长度：$\log_2 n$
- 比折半查找维护简单

### 1.2 插入元素

- 不同插入次序的序列生成不同形态的二叉排序树
- 每次插入的新结点都作为二叉搜索树一个新的叶子结点
- 一个无序序列可以通过构造一棵二叉搜索树而变成一个有序序列（中序遍历）


步骤：
1. 若二叉搜索树为空，则令新结点为插入后二叉搜索树的根结点，否则执行②
2. 将新结点的关键字与树（子树）的根结点关键字进行比较，执行以下操作
    - 若相等，则说明树中已有此关键字，不需要插入
    - 若小于根结点关键字且左孩子为空
        - 则新插入结点作为树（子树）的根结点的左孩子
        - 否则执行②
    - 若大于根结点关键字且右孩子为空
        - 则新插入结点作为树（子树）的根结点的右孩子
        - 否则执行②

注：二叉搜索树的构造就是插入的过程

若从空树出发，经过一系列的查找、插入操作之后，可生成一棵二叉树

### 1.3 删除元素

删除一个结点后需要依旧保持二叉搜索树的特性。
- 删除叶结点，只需将其双亲结点指向它的指针清除，再释放它即可；
- 被删结点只有左子树或只有右子树，可以将其左孩子或右孩子结点顶替它的位置，再释放它即可；
- 被删结点的左右子树都存在
	- 其一：将其左子树作为其双亲结点的左子树，而其右子树作为其直接前驱的右子树，如图c；
	- 其二：将其直接前驱（或直接后继）替代自身位置，再释放它即可，如图d
