# 有向图
有向图：由有向边组成的图
- 有向边(弧)：若顶点$v$和$w$间的边有方向，用$<v,w>$表示
- 弧尾$v$(始点)、弧头$w$(终点)

![[数据科学/数据结构/attachments/2026-01-28_192718.svg]]

- 顶点集合：$V = \{v_1, v_2, v_3, v_4\}$  
- 边集合：$E = \{<v_1, v_2>, <v_1, v_3>, <v_3, v_4>, <v_4, v_1>\}$

## 1 最短路径

带权图路径长度：在==带权有向图==中，把从一个顶点到图中其余任一个顶点的一条路径上所经过边上的权值之和

最短路径：带权图路径长度最短（权值和最小）的路径
- 最短路径长度（最短距离）：权值之和

- 迪杰斯特拉Dijkstra算法：单源最短路径
- 弗洛伊德Floyd算法：所有顶点间的最短路径


#### 1.1.1 迪杰斯特拉算法

迪杰斯特拉Dijkstra算法：更新-选择
1. 初始化：先找出从源点$v_0$到各终点$v_k$的直达路径$(v_0, v_k)$，即通过一条弧到达的路径。
2. 选择：从这些路径中找出一条长度最短的路径$(v_0, u)$。
3. 修改更新：然后对其余各条路径进行适当调整：
	- 若在图中存在弧$(u, v_k)$，且$(v_0, u) + (u, v_k) < (v_0, v_k)$
	- 则以路径$(v_0, u, v_k)$代替$(v_0, v_k)$。
4. 在调整后的各条路径中，再找长度最短的路径，依此类推

#### 1.1.2 弗洛伊德算法

Floyd（弗洛伊德）算法的基本思想：更新
1. 初始化：用二维数组$A[i][j]$保存从$V_i$到$V_j$的路径长度，二维数组$Path[i][j]$保存从$V_i$到$V_j$最短路径所经过的顶点编号
2. 修改（更新）：从$V_i$到$V_j$的路径上加入图中的一个顶点$V_k$，更新$A[i][j]$的路径长度，如果$A[i][j] > A[i][k] + A[k][j]$，那么就更新$A[i][j]$和$Path[i][j]=k$，否则不更新
