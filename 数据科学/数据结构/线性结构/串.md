# 串

串（字符串）：是由零个或多个字符组成的有限序列
- 记作：$$s ="a_1a_2a_3\cdots a_n" \ (n \geq 1)$$
- 串名s
- 串值：双引号引起来的字符序列
- $a_i\ , (1 \leq i \leq n)$：可以是字母、数字或其他字符
- 串的长度：串中字符的数目
- 空串：长度为0的串，它不包含任何字符

子串：串中任意个连续的字符组成的子序列
主串：包含子串的串

字符位置：子串的首字符在主串中的序号称为子串在主串中的位置。

例：s1= "12345" s2= "23"
s₂是s₁的子串，s₁为主串
s₂在s₁中的位置是2


## 存储表示
### 顺序存储表示
顺序存储表示（静态存储分配的顺序表）：用一组连续的存储单元来存放串中的字符序列。所谓定长顺序存储结构，是指直接使用定长的字符数组来实现的。

```c
//----串的定长顺序存储表示
#define MAXSTRLEN 255    // 最大串长为255
typedef unsigned char SString[ MAXSTRLEN + 1 ];// 0号单元存放串的长度
```

该结构内存布局

| 下标 | 0 | 1 | 2 | 3 | …… | MAXSTRLEN |
| ---- | -- | -- | -- | -- | ---- | -------- |
| 用途 | 串的实际长度(n) | 第1个字符 | 第2个字符 | 第3个字符 | …… | (备用空间) |
| 示例 | 5 | "H" | 'e' | 'l' | …… | (未使用) |
### 链串

链串（Linked String，串的这种链式存储结）：便于进行插入和删除运算，但存储密度较低。结点可以存放一个字符，也可存放多个字符。

![[数据科学/数据结构/attachments/Pasted image 20260213105631.png]]


为便于进行串的操作，当以链表存储串值时，除头指针外还附设一个尾指针指示链串的最后一个结点，并给出当前串中的结点个数，即串的长度。串的链式存储结构表示如下：

```c
#define chunksize 100
typedef struct chunk{
    char ch[chunksize];
    chunk *next;
}chunk;

typedef struct{
    chunk *head,*tail;
    int curlen;  //串的当前长度
}Lstring;
```
## 串的基本操作

- 串赋值：生成一个其值等于chars的串T
- 串复制：由串S复制得到串T
- 判空：若串S为空串，则返回真
- 串比较：若S > T，返回值> 0；若S = T，返回0；若S < T，返回值< 0。比较规则是字符的编码值(如ASCII码)
- 求串长：返回串S的元素个数，即串的长度
- 清空串：将S清为空串
- 串联接：用T返回由S 1和S 2联接而成的新串
- 求子串：用Sub返回串S的第pos个字符起、长度为len的子串
- 定位(模式匹配)：若主串S中存在和串T值相同的子串，则返回它在主串S中第pos个字符之后第一次出现的位置；否则返回0
- 串替换：用V替换主串S中出现的所有与T相等的不重叠子串
- 子串插入：在串S的第pos个字符之前插入串T
- 子串删除：从串S中删除第pos个字符起、长度为len的子串

## 串的模式匹配

模式匹配（Pattern Matching）：设S和T是两个给定的串，在串S中找等于T的子串的过程
- S一般称为主串或目标串，T称为模式串
- 如果找到，称为匹配成功，否则称为匹配失败

### BF算法

BF算法（Brute Force）：古典的、经典的、朴素的、穷举的

1. 执行循环：将主串的第i个字符和子串的第1个字符比较
	- 相等，继续逐个比较后续字符
	- 不等，回溯到主串的下一字符，重新与子串的第1个字符比较
2. 退出循环
	- 成功：返回S中与T匹配的子序列第1个字符的序号
	- 失败：返回0

BF算法实现

![[数据科学/数据结构/attachments/Pasted image 20260213110820.png]]

```c
int Index (SString S, SString T, int pos) {
    // 返回子串T在主串S中第pos个字符之后的位置。若不存在，则函数值为0
    // 其中，T非空，1 ≤ pos ≤ StrLength (S)
    i = pos; j = 1; // 初始化
    while (i <= S.length && j <= T.length) { // 两个串均未比较到串尾
        if (S[i] == T[j]) {
            ++i; ++j; // 继续比较后继字符
        } else {
            i = i - j + 2;
            j = 1; // 指针后退重新开始匹配
        }
    }
    if (j <= T.length) {
	    return 0;
    }
    return i - T.length;
}
```


### KMP算法

KMP算法(knuth-Morris-Pratt)：如何避免指针i回溯。

- 主串的$k-1$位后缀与字串的$k-1$位前缀相同


可以提前创建一个数组`next[]`，用来存储当模式中第j个字符与主串中相应字符不匹配时，在模式串中需要重新和主串该字符进行比较的字符位置`next[j]`。由此可引出模式串的next函数的定义：

$next[j]=$
- $0,  \text{当}j=1\ (s_{i} \neq t_{1}\ \text{时，下步比较}t_{1}\text{和}s_{i+1})$
- $\max\{ k|1<k<j,且"t_{1}\dots t_{k-1}" = "t_{j-k+1}\dots t_{j-1}" \}$
- $1, \text{当}k=1\ (\text{不存在相同的子串，下一步比较}\text{t}[1]\text{和}\text{s}[i])$

手动推导`next[]`数值

| j       | 1234 | 5678 |
| ------- | ---- | ---- |
| 模式串     | abaa | bcac |
| next[j] | 0112 | 2312 |

KMP算法步骤
1. 失配时，i不变，j回退`next[j]`重新匹配
2. 当j退到0，i和j需要同时加1（即主串的第i个字符和模式的第1个字符不等时，应从主串的第i+1个字符起重新进行匹配）


![[数据科学/数据结构/attachments/Pasted image 20260213134653.png]]


```c
// KMP 算法
int Index_KMP (SString S, SString T, int pos, int next[]) {
    // 利用模式串T的next函数求T在主串S中第pos个字符之后的位置
    // 其中,T非空,1 ≤ pos ≤ StrLength(S)
    int i = pos, j = 1;
    while (i <= S[0] && j <= T[0]) {
        if (j == 0 || S[i] == T[j]) {
            ++i;
            ++j; // 继续比较后继字
        }
        else
            j = next[j]; // 模式串向右移动
    }
    if (j > T[0]) return i - T[0]; // 匹配成功
    else return 0;
}
```
### KMP改进算法


发现 $t[6]=t[3]$ 时，可以直接使 $next[6]=next[3]=1$
$next[]$ 数组的优化方法：修正值$nextval[]$，在$j++$ 以及 $k++$ 之后，在向
$nextval[]$数组中填入数值时，如果 $t[j] == t[k]$，直接使 $next[j] = next[k]$ 即可
