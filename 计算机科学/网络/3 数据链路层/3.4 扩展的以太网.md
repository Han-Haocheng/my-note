## 在数据链路层扩展以太网

网桥(bridge)：对收到的帧根据其 MAC 帧的目的地址进行转发和过滤

#### 以太网交换机的特点

以太网交换机：多接口的网桥，通常都有十几个或更多的接口
- 相 互通信的主机都是独占传输媒体，无碰撞地传输数据

集线器

- N 个用户共享集线器提供的带宽 B
- 平均每个用户仅占有 B/N 的带宽。

交换机

- 交换机为每个端口提供带宽 B
- N 个用户，每个用户独占带宽 B。
- 交换机总容量达 B xN。

#### 交换机交换方式

直通(cut-through)：不必把整个数据帧先缓存后再进行处理，而是在 接收数据帧的同时就立即按数据帧的目的 MAC 地址决定该帧的转发接口
- 缺点：不检查差错就直接将帧转发出去

### 以太网交换机的自学习功能

![[计算机科学/网络/3 数据链路层/attachments/Pasted image 20251013092907.png]]


1. A 先向 B 发送一帧，从接口 1 进入到交换机
2. 交换机收到帧后，先查找交换表
	1. 在 MAC 地址这一列中，找不到目的地址为 B 的项目
	2. 交换机把这个帧的源地址 A 和接口 1 写入交换表中
	3. 向除接口 1 以外的所有接口广 播这个帧
	4. B收下这个目的地址正确的帧
	5. B向A发送帧
3. 按照表进行发送

生成树协议 STP (Spanning Tree Protocol)：不改变网络的实际拓扑，但在逻辑上则切断某些链路，使得从一 台主机到所有其他主机的路径是无环路的树状结构，从而消除了兜圈子现象

### 从总线以太网到星形以太网

- 总线以太网使用 CSMA/CD 协议，以半双工方式工作
- 以太网交换机不使用共享总线，没有碰撞问题，因此不使用 CSMA/CD 协议，而是以全双工方式工作，仍然采用以太网的帧结构

## 虚拟局域网
![[计算机科学/网络/3 数据链路层/attachments/Pasted image 20251013102808.png]]
虚拟局域网 VLAN (Virtual LAN)：由一些局域网网段构成的与物理位置无关的逻辑组，而这些网段 具有某些共同的需求。每一个 VLAN 的帧都有一个明确的标识符，指明发送这个帧的计算 机属于哪一个 VLAN



划分虚拟局域网的方法
- 基于交换机端口：最简单常用的方法，在第一层划分，不允许用户移动
- 基于计算机网卡的 MAC 地址：在第二层划分虚拟局域网，允许用户移动，需要管理大量MAC地址，在MAC改变时，需要重新分配
- 基于协议类型：根据协议额外的类型确定，在第二层划分虚拟局域网
- 基于 IP 子网地址：根据以太网帧第三个字段类型和IP，在第三层层划分虚拟局域网
- 基于高层应用或服务：根据应用层或服务划分，灵活复杂

交换机之间的协议

- 标记类型：固定
