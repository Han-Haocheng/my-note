# 调度策略

CPU 调度：选择等待中的进程进行执行的方式
- 方式 1: 先进先出
	- 优点：容易实现
	- 缺点：简单任务等待时间长
- 方式 2: 优先级策略
	- 优点：根据不同的进程进行调度
	- 缺点：实现方式复杂，会出现饥饿情况


进程调度方式是指当某一个进程正在处理机上运行的过程中，如果存在一个优先级更高的进程进入就绪队列，此时处理机应该如何进行分配的方式。


## 非剥夺调度策略


非剥夺调度方式（非抢占方式）：指当一个进程正在处理机上运行时，即使有优先级更高的进程进入就绪队列，当前进程也不会把处理机分配给新进的进程而是继续执行，直到该进程完成或等待某种事件发生而进入阻塞状态时，才把处理机分配给优先级高的进程
- 优点：实现简单、系统开销小、适用于大多数的批处理系统
- 缺点：不能用于分时系统和大多数的实时系统。


## 剥夺调度策略

剥夺调度方式（抢占方式）：指当一个进程在处理机上运行的过程中，若有优先级更高的进程进入就绪队列，则当前进程立即暂停正在运行的进程，将处理机分配给优先级高的进程
- 在一定程度上可以提高系统吞吐率和响应效率
- 不能随意执行，必须遵循如优先权、短进程优先和时间片等原则。

### 先来先服务

先来先服务(FCFS,First Come,First Served)：先来的任务，优先执行
- 平均周转时间：$\frac{10+39+42+49+61}{5}=40.2$
- 平均响应时间：$\frac{0+10+39+42+49}{5}=28$

### 短作业优先

短作业优先(SJF)：越短的作业，优先执行
- 平均周转时间：$\frac{3+10+20+32+61}{5}=25.2$
- 平均响应时间：$\frac{0+3+10+20+32}{5}=13$

![[计算机科学/操作系统/attachments/20250711173747.svg]]

### 时间片轮转

时间片轮转(RR,Round Robin):按时间片来轮转调度，时间片是切换时间的 100 倍时合适
- 平均周转时间：$\frac{10+23+30+52+61}{5}=35.2$
- 平均响应时间：$\frac{{0+10+20+23+30}}{5}=16.6$

### 优先级调度

优先级调度：前台任务使用时间片轮转，后台任务使用短作业优先

时间片最大的任务，率先执行，如果是 IO 约束型任务，则时间片会随着 IO 增长，将时间片作为优先级

```c
while (1) {
	c = -1;              // 最大值依据
	next = 0;            // 时间片最大的任务
	i = NR_TASKS;        // 索引
	p = &task[NR_TASKS]; // 任务地址
	while (--i) {
		if (!*--p)
			continue;
		if ((*p)->state == TASK_RUNNING && (*p)->counter > c)
			c = (*p)->counter, next = i;
	}
	if (c) break;        // 找到时间片最大的任务
	// 所有就绪态任务时间片用完
	for(p = &LAST_TASK ; p > &FIRST_TASK ; --p)  // 将所有时间片
		if (*p) 
			// C <= 2P  收敛集数，保证响应时间有界
			(*p)->counter = ((*p)->counter >> 1)+(*p)->priority; 
}
```
