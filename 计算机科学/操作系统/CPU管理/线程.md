# 线程



线程：操作系统能够进行运算调度的最小单位。它被包含在进程之中，是进程中的实际运作单位。一个线程指的是进程中一个单一顺序的控制流，一个进程中可以并发多个线程，每条线程并行执行不同的任务。
- 在线程的操作系统中，进程的两个基本属性是分开的，进程是拥有资源的独立单位，线程是独立调度和分派的基本单位。

## 属性


- **轻型实体**：线程中的实体基本上不拥有系统资源，只是有一点能保证其独立运行的资源，比如线程实体包括程序、数据和 TCB 等。线程控制块是线程控制运行的唯一标识。
- **独立调度和分派的基本单位**：在多线程 OS 中，线程是能独立运行的基本单位，因而也是独立调度和分派的基本单位。由于线程很“轻”，故线程的切换非常迅速且开销小。
- **可并发执行**：在一个进程中的多个线程之间，可以并发执行，甚至允许在一个进程中所有线程都能并发执行；同样，不同进程中的线程也能并发执行。
- **共享进程资源**：在同一进程中的各个线程，都可以共享该进程所拥有的资源

## 状态

- NEW 状态：线程对象已创建但未调用 start()方法
- 就绪状态(READY)：线程对象创建后，调用该线程的 start()方法，该线程就进入了“可运行线程池”中，变得可运行，只需等待获取 CPU 的使用权即可运行
- 运行状态(RUNNING)：就绪状态的线程获得 CPU 后，开始执行该线程的代码
- 可运行状态(RUNNABLE)：(可运行状态=就绪状态+运行状态)，线程已调用 start()方法，等待 CPU 调度执行
- BLOCKED：阻塞状态，线程因资源竞争而被阻塞
- WAITING：等待状态，线程等待其他线程唤醒
- TIMED_WAITING：计时等待状态，线程等待一定时间后被唤醒或超时
- TERMINATED：终止状态，线程已执行完毕或异常退出

## 用户级线程



进程包含资源和指令执行序列

![[计算机科学/操作系统/attachments/20250703154745.svg]]

用户级线程（User Threads）：将进程的资源和指令序列分离，在一个进程中只有一个资源，但可有多个指令序列，完全在用户态
- 可以实现并发的同时，避免进程切换带来的资源切换浪费
- TCB(Thread Control Block)：线程控制块，用于存储一个线程的函数栈等资源
- 内核线程阻塞时，进程等待

### 含有中断的代码

```c
void main(){
	A();
	B();
}

void A(){
	fork();
}
```

对应代码段

```
! 100:0
main:
	call A
	call B
	ret

A:call fork
	ret
B: ret

fork:
	mov %eax,__NR_fork
	int 0x80           ! 100:123
	mov res,%eax       ! 100:125
user_stack:
	.dword 0,0,...,0
! 100:256
```

> [!NOTE] 注意：保护模式下 int 0 x 80 中断会导致以下操作
> 1. 检查中断向量号有效性
> 2. 从 IDT 中获取中断描述符
> 3. 特权级检查：`CPL<DPL`
> 4. 切换栈
> 	- `mov ss,ax`
> 	- `mov esp,bx`
> 	- `mov IDT[ss],ss` 
> 	- `mov IDT[esp],esp` 
> 	- `push ax`
> 	- `push bx`
> 5. 保存上下文
> 	- `push EFLAGS`
> 	- `push CS`
> 	- `push EIP`
> 6. 跳转到目标代码

kernel\sched.c

```c
#include <linux\sched.h>

void schedule(void){
	int i,next,c;
	struct task_struct ** p;

/* check alarm, wake up any interruptible tasks that have got a signal */

	for(p = &LAST_TASK ; p > &FIRST_TASK ; --p)
		if (*p) {
			if ((*p)->alarm && (*p)->alarm < jiffies) {
					(*p)->signal |= (1<<(SIGALRM-1));
					(*p)->alarm = 0;
				}
			if (((*p)->signal & ~(_BLOCKABLE & (*p)->blocked)) &&
			(*p)->state==TASK_INTERRUPTIBLE)
				(*p)->state=TASK_RUNNING;
		}

/* this is the scheduler proper: */

	while (1) {
		c = -1;
		next = 0;
		i = NR_TASKS;
		p = &task[NR_TASKS];
		while (--i) {
			if (!*--p)
				continue;
			if ((*p)->state == TASK_RUNNING && (*p)->counter > c)
				c = (*p)->counter, next = i;
		}
		if (c) break;
		for(p = &LAST_TASK ; p > &FIRST_TASK ; --p)
			if (*p)
				(*p)->counter = ((*p)->counter >> 1) +
						(*p)->priority;
	}
	switch_to(next);
}
```

### 复制进程为子进程

```c
int copy_process(int nr,
	long ebp,long edi,long esi,long gs,long none,
	long ebx,long ecx,long edx,
	long fs,long es,long ds,
	long eip,long cs,long eflags,long esp,long ss){
	//申请内存
	p=(struct task_struct *)get_free_page();
	// 创建内核栈
	p->tss.ss0 =0x10;
	p->tss.esp0=PAGE_SIZE + (long)p;
	//创建用户栈
	p->tss.ss  = ss&0xffff;
	p->tss.esp = esp;
	p->tss.eip = eip
	p->tss.cs  = cs&0xffff;
	p->tss.eax = 0
	p->tss.ecx = ecx;
	p->tss.ldt = _LDT(NR);
	set_tss_desc(gdt+(nr<<1))+FIRST_TSS_ENTRY,&(p->tss))
	set_idt_desc(gdt+(nr<<1))+FIRST_IDT_ENTRY,&(p->idt))
	p->state=TASK_RUNNINT;
}
```
