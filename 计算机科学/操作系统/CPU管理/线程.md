# 线程



线程：操作系统能够进行运算调度的最小单位。它被包含在进程之中，是进程中的实际运作单位。一个线程指的是进程中一个单一顺序的控制流，一个进程中可以并发多个线程，每条线程并行执行不同的任务。
- 在线程的操作系统中，进程的两个基本属性是分开的，进程是拥有资源的独立单位，线程是独立调度和分派的基本单位。

## 用户级线程

进程包含资源和指令执行序列

![[计算机科学/操作系统/attachments/20250703154745.svg]]

用户级线程（User Threads）：将进程的资源和指令序列分离，在一个进程中只有一个资源，但可有多个指令序列，完全在用户态
- 可以实现并发的同时，避免进程切换带来的资源切换浪费
- TCB(Thread Control Block)：线程控制块，用于存储一个线程的函数栈等资源

## 内核级线程

![[计算机科学/操作系统/attachments/20250704142716.svg]]


内核级线程（Kernel Threads）：线程由内核管理，发挥多核并行性能

### 内核栈和用户栈

- 在用户态执行的代码，使用的是用户栈
- 在内核态执行的代码，使用的是内核栈，并且与用户栈进行关联
	- 包含：用户栈 `SS:SP`，执行函数地址

```c
100:A(){
	B();
104:}
```

```c
200:B(){
	read();
204:}
```

```c
300:read(){
	int 0x80;
304:}
```

```asm
system_call:
	call sys_read;
1000:
2000:sys_read(){}
```

```c
interrupt_entry:
	push ds;
	...
	pusha;
	mov ds,内核段号
	...
	call int_process
	popa
	...
	pop ds
	iret

interrupt_process:
	int //磁盘读 时钟
	call schedule
	ret

schedule:
	mov ..,next
	call switch_to
	ret

void switch_to(cur,next){
	mov %esp,TCB[cur].esp
	mov TCB[next],exp,%esp
	ret
}
```
