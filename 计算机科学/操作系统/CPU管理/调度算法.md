# 调度算法


调度算法：根据系统的资源采用某分配策略对资源进行分配的方式方法
- 为实现不同的系统目标需采用不同的调度算法


任务调度举例

| 任务      | 到达时间 | CPU 区间(ms) |
| ------- | ---- | --------- |
| $P_{1}$ | 0    | 10        |
| $P_{2}$ | 2    | 29        |
| $P_{3}$ | 3    | 3         |
| $P_{4}$ | 4    | 7         |
| $P_{5}$ | 5    | 12        |

## 先来先服务

![[计算机科学/操作系统/attachments/20250711172537.svg]]
- 平均周转时间：$\frac{10+39+42+49+61}{5}=40.2$
- 平均响应时间：$\frac{0+10+39+42+49}{5}=28$


先来先服务(FCFS,First Come,First Served)：先来的任务，优先执行，不可剥夺算法。从等待时间上看，它是相对公平的。
- 对短作业来说，如果在之前有长作业先占有 CPU，则短作业就会等待很长时间。
- 算法简单但效率低
- 对长作业比较有利，对短作业不利
- 有利于 CPU 繁忙型作业，不利于 I/O 繁忙型作业。

## 短作业优先
![[计算机科学/操作系统/attachments/20250711173747.svg]]

- 平均周转时间：$\frac{3+10+20+32+61}{5}=25.2$
- 平均响应时间：$\frac{0+3+10+20+32}{5}=13$


短作业优先(SJF)：越短的作业，优先执行
- 对长作业不利，SJF 调度算法中长作业的周转时间会增加。如果有一个长作业进入系统的后备队列，由于调度程序总是优先调度短作业，将导致长作业长期不被调度，可能会出现“饥饿”现象。
- 没有考虑作业的紧迫程度，因而不能保证紧迫性作业会得到及时处理。
- 由于作业的长短只是根据用户所提供的估计执行时间而定的，而用户有可能会有意或无意地缩短其作业的估计运行时间，致使此算法不一定能真正做到短作业优先调度

注意：SJF 调度算法的平均等待时间、平均周转时间最少。


## 时间片轮转
![[计算机科学/操作系统/attachments/20250711174431.svg]]
- 平均周转时间：$\frac{10+23+30+52+61}{5}=35.2$
- 平均响应时间：$\frac{{0+10+20+23+30}}{5}=16.6$

时间片轮转(RR,Round Robin)：是指将所有就绪进程按其到达时间的先后次序排成一个队列，进程调度程序总是选择就绪队列中第一个进程执行，并按规定执行一个时间片，当执行完一个时间片后，即使进程并没有运行完成，系统计时器也立刻发出时钟中断请求，调度程序通过此信号停止该进程的运行，并将该进程送往就绪队列的队尾；然后，把处理机分配给就绪队列中下一个首进程。这样所有的进程按照顺序依次轮流获得、释放时间片方式循环执行，直到任务完成。
- 按时间片来轮转调度，时间片是切换时间的 100 倍时合适
- 在分时系统中，进程调度算法通常选择时间片轮转调度算法。在系统运行过程中，时间片轮转调度算法(时间片的大小固定，如 100 ms)。



时间片轮转（RR）调度算法：就绪进程按到达顺序排队列，调度程序选队首进程执行固定时间片；时间片耗尽时，进程回队尾，处理机分配给新队首，循环直至任务完成
- 时间片大小需远大于上下文切换开销（如 100 倍），以降低调度开销。  
- 分时系统中，该算法通过固定时间片（如 100 ms）实现多进程公平调度，满足交互需求。


时间片长度是影响系统性能的核心参数  
- 若时间片过长，进程可在单时间片内完成，算法退化为 FCFS（先来先服务），无法满足分时系统的交互性需求；  
- 若时间片过短，短作业虽可快速完成，但进程频繁中断导致上下文切换开销激增

优化原则：时间片长度应略大于典型交互操作所需时间，确保多数进程在一个时间片内完成。  

决定因素：时间片长度由系统响应时间、就绪队列进程数量及系统处理能力共同决定。


## 优先级调度

优先级调度：前台任务使用时间片轮转，后台任务使用短作业优先

时间片最大的任务，率先执行，如果是 IO 约束型任务，则时间片会随着 IO 增长，将时间片作为优先级

```c
while (1) {
	c = -1;              // 最大值依据
	next = 0;            // 时间片最大的任务
	i = NR_TASKS;        // 索引
	p = &task[NR_TASKS]; // 任务地址
	while (--i) {
		if (!*--p)
			continue;
		if ((*p)->state == TASK_RUNNING && (*p)->counter > c)
			c = (*p)->counter, next = i;
	}
	if (c) break;        // 找到时间片最大的任务
	// 所有就绪态任务时间片用完
	for(p = &LAST_TASK ; p > &FIRST_TASK ; --p)  // 将所有时间片
		if (*p) 
			// C <= 2P  收敛集数，保证响应时间有界
			(*p)->counter = ((*p)->counter >> 1)+(*p)->priority; 
}
```
