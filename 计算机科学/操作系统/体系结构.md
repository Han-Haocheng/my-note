# 体系结构
随着软件开发技术的不断发展，OS 结构也在不断地更新换代。其中，传统结构的 OS 包括早期的无结构 OS、模块化结构的 OS 和分层式结构的 OS，而微内核结构的 OS 为现代结构的 OS。
## 无结构

无结构：很多操作系统在最初都是小而简单且功能有限的系统，是众多过程的集合，并且这些过程可以相互调用。缺点是操作系统内部混乱无章，结构复杂。
## 分层结构

分层结构：比较成熟的 OS 结构，操作系统按功能分为多个模块，按相互作用关系划分不同的层次，各层之间单向依赖，没有构成循环，简化关系、修改扩充容易，一致性、可靠性和可适应性较高，所有功能模块都在核心态工作。

优点
- 构造和调试简单化
- 每层都有低一层为上一层提供功能和支持的服务

缺点
- 对于层的详细定义困难
- 效率差。
## 模块化结构

模块化结构：无序的模块化结构 OS。每个模块设一个子程序，相互调用，成网状结构，模块直接与硬件有关，难以维护，每一次扩充需要先打破多个关系，移植性差。这是早期系统常用的体系结构。
## 客户/服务器模式

客户/服务器系统主要由客户机、服务器和网络系统三部分组成。单机微内核操作系统都采用客户/服务器模式，将操作系统中最基本的部分放入内核中，而把操作系统的绝大部分功能都放在微内核外面的一组服务器中实现。
## 面向对象的程序设计模式

面向对象的程序设计模式：基于面向对象技术中的“抽象”和“隐藏”原则控制系统的复杂性，再进一步利用“对象”“封装”和“继承”等概念来确保操作系统的“正确性”“可靠性”“易修改性”“易扩展性”等，并提高操作系统的设计速度。目前面向对象技术也被广泛应用于现代操作系统的设计中。


## 宏内核结构

宏内核结构：宏内核是一种传统的内核结构，将整个核心程序以核心空间及监管者模式的身份运行，所有服务都在一个大型的核心地址空间中运作，这个地址空间被称为核心空间

优点：
- 性能较高，由于内核服务在内核空间直接运行，减少了用户态和内核态的切换开销，因此性能通常较高。
- 设计简洁，宏内核设计相对简单
缺点：
- 内核代码量大，维护难度高，且一个服务崩溃可能导致整个系统崩溃。
- 宏内核缺乏模块化，添加、移除或升级服务可能比较困难，甚至需要重启系统。

## 微内核结构

微内核结构：微内核 OS 结构以客户/服务器体系结构为基础、采用面向对象技术的结构，能有效地支持多处理器，非常适用于分布式系统。微内核是一个能实现 OS 功能的小型内核，运行在核心态，且常驻内存，它不是一个完整的 OS，只为构建通用 OS 提供基础

微内核的基本功能包括进程管理、存储器管理、进程间通信、I/O 设备管理。因此操作系统由运行在核心态的内核和运行在用户态并以客户／服务器方式运行的进程两大部分组成。
