# 生磁盘的使用

## 磁盘

![[计算机科学/操作系统/attachments/2025-09-15_165832.svg]]


- 磁盘访问单位：扇区
- 扇区大小：512KB
- 扇区的大小是传输时间和碎片浪费的折中

## 磁盘IO过程

### 磁盘读取宏观过程

1. 移动磁头到指定磁道
2. 旋转磁盘
3. 根据磁生电原理读取数据到内存缓存

### 磁盘写入宏观过程

1. 移动磁头到指定磁道
2. 旋转磁盘
3. 修改内存缓存
4. 根据电生磁原理从内存缓存写入磁盘数据

## 使用磁盘


### 直接使用磁盘

- 在控制器中写入柱面（Cyl）、磁头（Head）、扇区（Sec）、缓存的位置
![[计算机科学/操作系统/attachments/2025-09-21_131813.svg]]

```c
void do_hd_request(void){
	hd_out(dev,nsect,sec,head,cyl,WIN_WRITE,...);
	port_write(HD_DATA,CURRENT->buffer,256);
}

void hd_out(drive,nsect,sec,head,cyl,cmd...){
	port = HD_DATA; // 数据寄存器端口（0x1f0）
	outb_p(sect,++port);
	outb_p(cyl,++port);
	outb_p(cyl>>8,++port);
	outb_p(0xA0|(drive<<4)|head,++port);
	outb_p(cmd,++port);
}
```

### 通过盘号读写磁盘

![[计算机科学/操作系统/attachments/2025-09-21_132400.svg]]


- 磁盘驱动负责从block计算出CHS（cyl，head，sec）
- 磁盘访问时间=写入控制器时间+寻道时间+旋转时间+传输时间
	- 寻道时间：8~12ms
	- 旋转时间：7200转/min，4ms/半周
	- 传输时间：50MB/s，0.3ms
- 编址原则：block相邻的盘块可以快速读出
- 编址方式：$block=C\times(Heads\times Sectors)+H\times Sectors+S$
	- $S=block\%Sectors$
- 扇区到盘块：通过调整一个盘块对应n个扇区，从而提高读写速度
	- 每次读写1K；碎片0.5K；读写100K/秒
	- 每次读写1M；碎片0.5M；读写速度约40M/秒

void di_hd_request(void){
	unsigned int block = CURRENT->sector;
	__asm__("divl %4":"=a"(block),"=d"(sec):"0"(block),"1"(0),"r"(hd_info[dev].sect));
	__asm__("divl %4":"=a"(cyl),"=d"(head):"0"(block),"1"(0),"r"(hd_info[dev].head));
	hd_out(dev,nsect,sec,head,cyl,WIN_WRITE,...);
	port_write(HD_DATA,CURRENT->buffer,256);

### 多个进程通过队列使用磁盘

![[计算机科学/操作系统/attachments/2025-09-21_133014.svg]]


- 多个磁盘访问请求出现在请求队列，需要进行磁盘调度
- 调度目标：平均访问延迟最小
	- 寻道时间是只要矛盾

- 算法示例
	- 磁头开始位置：53
	- 请求队列=98,183,37,122,14,124,65,67

![[计算机科学/操作系统/attachments/2025-09-21_133159.svg]]


#### FCFS磁盘调度

FCFS磁盘调度：先来先服务
- 共跨越磁道：640

![[计算机科学/操作系统/attachments/2025-09-21_134333.svg]]

#### SSTF磁盘调度

SSTF磁盘调度：短寻道优先，谁离得近率先访问
- 共跨越磁道：236
- 由于请求是连续的，且数据大概率位于中间，磁头总是在中间，容易产生饥饿
![[计算机科学/操作系统/attachments/2025-09-21_135049.svg]]


#### SCAN磁盘调度

SCAN磁盘调度：SSTF+中途不回折，每个请求都有处理机会
- 共跨越磁道：236
![[计算机科学/操作系统/attachments/2025-09-21_135245.svg]]



#### C-SCAN磁盘调度

C-SCAN磁盘调度（电梯算法）：SCAN+字节移动到另一端，两端请求都能很快处理
- 共跨越磁道：157+200，200是快速复位

![[计算机科学/操作系统/attachments/2025-09-21_135415.svg]]
```c
static void make_request(){
	struct request*req;
	req=request+NR_REQUEST;
	req->sector=bh->b_blocknr<<1; // 每次读写2个扇区
	add_request(major+blk_dev,req);
}

static void add_request(struct blk_dev_struct *dev,struct request *req){
	struct requset*tmp = dev->current_request;
	req->next=NULL;
	cli();//关中断
	for(;tmp->next;tmp=tmp->next)
		if((IN_ORDER(tmp,req)||!IN_ORDER(tmp,tmp->next))&&IN_ORDER(req,tmp->next))
			break;
	req->next=tmp->next;
	tmp->next=req;
	sti();
}

#define IN_ORDER(s1,s2)\
	((s1)->dev<(s2)->dev)||(s1)->dev==(s2)->dev\
	&&(s1)->sector<(s2)->sector))
```

## 生磁盘的使用总结

1. 进程“得到盘块号”，计算出扇区号（sector）
2. 用扇区make_request，用电梯算法add_request
3. 进程sleep_on
4. 磁盘中断处理
5. do_hd_request算出cyl，head，sector
6. hd_out调用outp(...)完成端口写

```c
static void read_intr(void){
	end_request(1);
	do_hd_request();
}
```
