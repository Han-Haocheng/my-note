# 操作系统

操作系统 OS（Operating System）：计算机硬件和应用之间的一层软件，是用户和计算机硬件系统之间的接口，是[[计算机科学/基础/软件/应用软件|应用软件]]和[[计算机科学/基础/硬件/硬件|硬件]]间的接口，一组控制和管理计算机硬件和资源，合理的对各类作业进行调度，方便用户使用的计算机的集合

- 常见的操作系统：[[计算机科学/基础/Windows10/Win10|Win10]]、Unix、Linux、MacOS、Dos、鸿蒙 OS


## 操作系统的特征

### 并发性

并发性：*多个*程序*同时*(同一时间段、同一时间间隔)在系统中运行

> [!warning]+ 注意：并发和并行的区别
> - 并发：两个或多个事件在同一时间间隔内发生 
> - 并行性：两个或多个事件在同一时刻发生 
> 

### 共享性

共享性：操作系统内的*资源*(软件、硬件、数据)可以*被多个并发执行的进程*所使用
- 互斥共享：资源在同一段时间内只能由一个进程访问
- 同时访问：资源在同一段时间内能由不同的进程访问，宏观同时，微观交替


### 虚拟性

虚拟性：通过某种技术，把*一个物理实体*变成*若干个逻辑*上的对应
- 包括：空分复用、时分复用
- 虚拟内存：虚拟化内存技术，将物理存储设备虚拟化为逻辑内存技术

### 异步性

异步性：在*多个*程序并发运行的环境，每个程序的执行、暂停、推进速度和完成*时间都是不确定*的

## 操作系统的分类

### 根据对作业的处理方式

#### 批处理操作系统

批处理操作系统：用户将一批作业提交给操作系统后不再干预，由系统控制自动运行
*   **核心特征**：**不具有交互性**
*   **目的**：提高 CPU 的利用率。
  
单道批处理操作系统：内存中仅有**一道**程序运行，一次处理一个任务。
*   **自动性**：磁带上的一批作业能自动运行，无需人工干预。
*   **顺序性**：作业按顺序进入内存，先进入的先完成。
*   **单道性**：内存中仅有一道程序运行。
  
多道批处理操作系统 (多任务操作系统)：允许**多个**程序同时进入内存，交替使用 CPU
    *   **宏观上并行**：同时进入系统的多道程序都处于运行过程中（均已开始，均未结束）。
    *   **微观上串行**：内存中的多道程序轮流占有 CPU，交替执行。
*   **运行机制**：
    *   允许多个程序共享系统中的硬/软件资源。
    *   当一道程序因 I/O 请求暂停时，CPU 立即转去运行其他程序。
*   **优点**：
	*   **资源利用率高**：多道程序共享资源，充分利用各类资源。
	*   **系统吞吐量大**：单位时间内完成的总工作量大。作业成批调入内存，资源保持“忙碌”状态，作业切换不频繁，系统开销小。
*   **缺点**：
	*   **平均周转时间长**：作业从进入系统到完成并退出的时间较长（可能需几小时甚至几天），因所有作业需排队顺序处理。
	*   **无交互能力**：用户提交作业后无法干预运行过程，导致修改和调试程序极为不便。

#### 分时操作系统

分时操作系统：一台主机链接若干终端，采用时间片轮转方式处理服务请求
- 特点：多路性、交互性、独占性、及时性
- 系统举例：Windows、Unix、MacOS

#### 实时操作系统

实时操作系统：在严格的时间规定内完成对事件的处理
- 性质：及时性、可靠性

### 根据体系结构和使用环境分类

#### 嵌入式操作系统

嵌入式操作系统：运行在嵌入式系统环境中
- 性质：专用性、针对性、稳定性、实时高效、内存占用小

#### 个人计算机操作系统

系统举例：DOS、WinXP、Win 7、Win 10

#### 网络操作系统

网络操作系统：基于计算机网络系统
- 系统举例：Windows Server、Unix、Linux、NetWare

#### 分布式操作系统

分布式操作系统：由若干台无主从关系的处理机组成的系统，可以管理分散在多个地理位置上的计算机资源，可以并行运行同一个程序

### 根据所支持的用户数和任务数

- 单用户单任务系统：DOS
- 单用户多任务系统：Win XP
- 多用户多任务系统：Win 7、Win 10

### 根据用户界面

- 命令行界面操作系统：DOS
- 图形用户界面操作系统：Windows


## 操作系统的启动流程

操作系统启动流程(X86)

1. CPU 处于实模式
	- 实模式：寻址方式为 `CS<<4:IP`
2. `CS=0xFFFF;IP=0x0000`
	- 段寄存器(CS)：存储当前执行指令所在代码段的基地址
	- 偏移寄存器(IP)：指定内存地址的在代码端基地址的偏移量
3. 寻址 ROM 中的 BIOS 映射区起始地址
	- ROM BIOS 映射区：`0xFFFF0`
4. 检查 RAM、键盘、显示器、软硬磁盘等硬件设备
5. 将磁盘 0 磁道 0 扇区写入 `0x7C00`
	- 系统引导扇区(0 磁道 0 扇区)：启动设备的第一个扇区
6. 设置 `CS=0x07C0;IP=0x0000`
