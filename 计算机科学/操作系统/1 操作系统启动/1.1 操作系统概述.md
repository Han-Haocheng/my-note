# 操作系统概述



## 计算机数学模型

**计算机模型(图灵机)**：包含无限的数据存储器、读写数据器、状态寄存器和状态转移表的结构
- **通用图灵机**：在图灵机的基础上拥有可编程性、结构扩展性
- **无限长纸带**：划分为方格，每个方格可存储符号（如0、1或空白符）。
- **读写头**：可读取或修改当前方格的符号，并沿纸带左右移动。
- **状态寄存器**：记录当前状态（如初始状态、接受状态等）。
- **状态转移表**：根据当前状态和读取的符号，决定下一步操作（写入符号、移动方向、状态转换）
- **可编程性**：通过输入其他图灵机的描述（编码）和初始数据，通用图灵机可模拟其行为。
- **结构扩展**：在基础图灵机结构上，增加对程序编码的解析能力，使其能执行任意图灵机程序。

## 计算机物理模型

计算机物理模型(冯诺依曼结构)：由控制器、运算器、存储器、IO设备构成的系统

## 汇编

### 寄存器
在x86架构的汇编语言中，寄存器是CPU内部的高速存储单元，用于临时存储数据、地址和控制信息。以下是主要寄存器及其核心作用：

#### 通用寄存器


数据寄存器：用于算术逻辑和其他操作，可保存32位、16位、8位数据
- **累加器(Accumulator)** ：用于算术运算、函数返回值存储
- **基址寄存器(Base)** ：常用于内存寻址
- **计数器(Counter)** ：控制循环次数或字符串操作中的重复次数
- **辅助数据寄存器(Data)** ：用于I/O端口操作或与EAX配合处理64位数据


|         | 32位   | 16位  | 8上位  | 8下位  |
| ------- | ----- | ---- | ---- | ---- |
| 累加器     | `EAX` | `AX` | `AH` | `AL` |
| 基地址寄存器  | `EBX` | `BX` | `BH` | `BL` |
| 计数器     | `ECX` | `CX` | `CH` | `CL` |
| 辅助数据寄存器 | `EDX` | `DX` | `DH` | `DL` |

#### 段寄存器

- **代码段寄存器(Code Segment)** ：指向当前执行的代码段。
- **数据段寄存器(Data Segment)** ：指向程序的数据段（全局变量）
- **堆栈段寄存器(Stack Segment)** ：指向当前堆栈段。
- **附加段寄存器ES/FS/GS(Extra Segments)** ：用于扩展数据存储

| 代码段寄存器 | `CS`           |
| ------ | -------------- |
| 数据段寄存器 | `DS`           |
| 堆栈段寄存器 | `SS`           |
| 附加段寄存器 | `ES`/`FS`/`GS` |

#### 指针变址寄存器

- **基址指针(Base Pointer)** ：标记当前函数栈帧的基地址，用于访问局部变量和参数
- **堆栈指针(Stack Pointer)** ：指向栈顶地址，管理函数调用时的入栈/出栈操作
- **源变址寄存器(Source Index)** ：指向字符串/数组操作的源数据地址（如`MOVSB`指令）
- **目的变址寄存器(Destination Index)** ：指向字符串/数组操作的目标地址

| 基址指针    | `BP` |
| ------- | ---- |
| 堆栈指针    | `SP` |
| 源变址寄存器  | `SI` |
| 目的变址寄存器 | `DI` |

- 指令的源地址：`DS:SI`
- 指令的源地址：`ES:DI`

#### 控制寄存器

**标志寄存器(Flags)** ：存储状态标志和控制标志
- **指令指针(Instruction Pointer)** ：存储下一条待执行指令的地址。
- 条件标志寄存器
	- 零标志
	- 进位标志
- 控制标志寄存器
	- 中断允许
