# 系统调用


系统调用：用户程序取得操作系统服务的唯一途径。用户在程序中使用“访管指令”调用由操作系统提供的子功能集合


## 分类


- 设备管理 完成设备的请求/释放/启动 等功能
- 文件管理 完成文件的 读/写/创建/删除 等功能
- 进程控制 完成进程的 创建/撤销/阻塞/唤醒 等功能
- 进程通信 完成进程之间的 消息传递/信号传递 等功能
- 内存管理 完成内存的 分配/回收 等功能


- 用户程序通过执行“陷入指令（又称访管指令或 trap 指令）”来发起系统调用，以此请求操作系统提供服务。  
- 当用户程序执行访管指令时，CPU 的状态会从“用户模式”切换为“内核模式”；随后操作系统内核程序会对系统调用请求进行处理。处理完成后，内核程序再将 CPU 的使用权交还给用户程序（即 CPU 状态从“内核模式”返回“用户模式”）。  
- 这种系统设计的目的，是限制用户程序直接执行核心程序的操作，强制用户程序通过“系统调用”请求操作系统代为执行，从而保障系统的稳定性和安全性。  

**用户模式转内核模式的事例**包括：  
- 用户程序要求操作系统的服务（即“系统调用”）；  
- 发生一次中断；  
- 用户程序中产生了一个错误状态；  
- 用户程序中企图执行一条特权指令。  

**内核模式转用户模式**：通过一条特权指令实现，这条指令也称为“中断返回指令”。  

## 进入内核态

通过*中断指令*可以将 CS 的 CPL 改为 0，从而进入内核态
1. 用户程序包含一段含有中断指令的代码（库函数）
2. 操作系统实现中断处理程序，获取对应程序编号
3. 操作系统根据编号执行相应代码
