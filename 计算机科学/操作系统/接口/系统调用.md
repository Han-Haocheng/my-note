# 系统调用

存放在系统中的数据，应用程序不可直接访问

## 系统运行状态

严格划分内核模式与用户模式这两类程序以及各自的存储空间是
- 在执行的过程中，通过执行访问内核模式的命令(访管指令)
- 引起中断
- 再由中断系统转入操作系统内的相应程序

CPU 将内存分割为内核态和用户态，内核态可以访问任何数据，用户态不能访问内核数据
- 在通过 GDT 跳段时，CPU 通过检查存放在 GDT 结构中的 DPL 属性判断段的执行访问权限，DPL 越小，权限越高
- CPL：CS 段对应的 DPL
- RPL：DS 段对应的 DPL


### 用户态

**用户模式（目态、用户态）**：用户程序执行时机器所处的状态，具有较低特权的执行状态。 只执行*规定的指令*，只能访问*指定的寄存器和存储区*

### 内核态

**内核模式（管态、系统态）**：操作系统管理程序执行时机器所处的状态。 具有较高的特权，能执行包括特权指令的*一切指令*，能访问*所有寄存器和存储区*


在操作系统中，操作系统的内核包括一些与硬件关联较紧密的模块(如时钟管理、中断处理等) 以及运行频率较高的程序(如进程管理、存储器管理、设备管理等)，它们都工作在内核模式。


- 时钟管理：在计算机的各种部件中，时钟是最关键的部件。时钟的第一功能是计时，操作系统需要通过时钟管理，向用户提供标准的系统时间。另外，通过时钟中断的管理，可以实现进程的切换。
- 中断机制： 键盘或鼠标的输入、进程的管理和调度、系统调用、设备驱动、文件访问等，无不依赖于中断机制。
- 原语：操作系统的核心，是内核或微内核提供核外调用的过程或函数。
- 系统控制的数据结构及处理： 操作系统中需要一些用来登记状态信息的数据结构，还应该定义对这些数据结构的一系列操作：进程管理、存储器管理、设备管理。

## 进入内核态

通过中断指令可以将CS的CPL改为0，从而进入内核态
1. 用户程序包含一段含有中断指令的代码（库函数）
2. 操作系统实现中断处理程序，获取对应程序编号
3. 操作系统根据编号执行相应代码
