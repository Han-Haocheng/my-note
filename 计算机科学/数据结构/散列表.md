# 散列表

**散列表（哈希表）**：根据==散列函数==建立的表
- **散列函数（哈希Hash函数）**：记录存储位置和关键字之间的一个确定的对应（映射）关系$f$，可使每个关键字和一个唯一的存储位置相对应，这样在查找时只要根据这个对应关系找到给定值$K$的$f(K)$，即为记录的存储位置
- 散列查找（哈希查找）：利用散列表进行查找的过程

## 散列查找的特点

1. 散列函数是利用函数的映射来实现查找的
2. 对于某个函数$y=f(x)$，不同的$x$通过$f$获得的$y$有可能是相同的

地址冲突：不同的关键字有可能得到相同的散列地址

冲突通常无法完全避免
1. 如何构造一个合理的散列函数
2. 冲突的处理

## 散列函数


### 直接取值法

直接定址法：定义一个线性函数，取关键字对于该函数的函数值作为散列地址
- $H(\text{key}) = a \times \text{key} + b$
其中a，b为常数
- 特点：是一一对应函数，不会产生冲突，但要求地址集合与关键字集合的大小相同，且占用连续的地址空间，空间效率低，所以在实际生活中并不常用。

### 取余数法


取余数法（除留余数法）：选择一个常数P，取关键字除以P所得的余数作为散列地址
- $H(\text{key}) = \text{key} \mod P$
- 特点：若散列表表长为m，P≤m，且一般选用质数；简单、常用。

### 平方取中法

平方取中法：取关键字平方后的中间几位作为散列地址。

例：有一组关键字{3456,2564,3466,3454}，使用平法取中法取散列地址

## 散列函数冲突处理
冲突：当某个关键字$K_a$对应的散列地址是$i$，并且$K_a$占用该地址$i$时，如果另外某个关键字$K_b$对应的散列地址也是$i$

再探测：在冲突时继续选取一个函数，把该函数的值作为$K_b$的地址
- 如果在再探测过程中又发生了冲突，那么需要再次探测，直到不发生冲突为止

### 开放定址法

$$H_i = [H(\text{key}) + d_i]\mod m \ ,(1 \leq i < m)$$

- 散列函数$H(\text{key})$
- 散列表表长$m$
- 增量序列$d_i$

获取增量序列$d_i$
- 线性探测再散列：$d_i = 1,2,\cdots,m-1$
- 二次探测再散列：$d_i = 1^2,-1^2,2^2,-2^2,\cdots,k^2,-k^2 \ (k \leq m/2)$
- 伪随机探测再散列：$d_i =$ 伪随机数序列

聚集现象：指处理冲突过程中发生的两个第一个散列地址不同的记录，争夺同一个后继散列地址的现象

### 再散列法


$H_i = RH_i(\text{key}) \quad i=1,2,\ldots,k$
- 均不同的散列函数$RH_i$

再散列法：在产生地址冲突时换用另一个散列函数计算，直到冲突不再发生
- 特点：这种方法不易产生聚集现象，但增加了计算的时间。

### 链地址法

链地址法：将所有关键字为同义词（地址相同）的记录存储到同一链表中。将散列函数产生的每个散列地址都作为一个指针，指向一个链表，若散列表表长为m，则建立m个空链表，将散列函数对关键字进行计算转换为地址i后，映射到同一散列地址i的关键字均加入到地址i所指向的链表中，同一链表按关键字有序

## 散列查找的性能分析

$$\alpha = \frac{\text{哈希表中元素的个数}}{\text{哈希表长度}}$$

装填因子：反映散列表中关键字的数量，α越大，表中记录数越多，表装得越满，发生冲突的可能性越大，查找的效率越低

平均查找长度ASL

假定数据元素输入序列1234,1876,124,6876,6861,1135,2011。散列表长度是 12,散列函数 $H(x)=x \mod 10$,再探测的方法采用开放定址方法。计算查找成功时的平均查找次数。

解：查找成功时的平均查找次数:$13/7=1.86$。
- $H(1234)= 1234\%10=4$
- $H(1876)= 1876\%10=6$
- $H(124)= 124\%10=4$(冲突)
- $H(124)= (4+1)\%12=5$
- $H(6876)= 6876\%10=6$(冲突)
- $H(6876)= (6+1)\%12=7$
- $H(6861)= 6861\%10=1$
- $H(1135)= 1135\%10=5$(冲突)
- $H(1135)= (5+1)\%12=6$(冲突)
- $H(1135)= (5+2)\%12=7$(冲突)
- $H(1135)= (5+3)\%12=8$
- $H(2011)= 2011\%10=1$(冲突)
- $H(2011)= (1+1)\%12=2$
