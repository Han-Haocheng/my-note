# 树


树：一种描述==一对多==关系逻辑的==非线性结构==，节点之间有分支，具有层次结构。一种递归的数据结构，包含个结点的有限集合T，有且仅有一个根的结点，只有后继结点，没有前驱结点
- 树的深度：树中距离根最远的结点所处的层次
- 树的高度：高度与深度计算方向不同，但数值相同
- 树的度：树中所有结点的度的最大值


- 空树：当$n=0$时，集合T
- 非空树：当$n>0$时，集合T
- 根的子树：当$n>1$时，除根结点以外的其余结点分为$m(m > 0)$个==互不相交的有限集合==$T_1,T_2,\cdots,T_m$，其中==每个集合本身又是一棵树==
- 有序树：树中各个结点的各棵子树从左到右都是有次序的树；否则称为无序树



## 1 树的结点
结点：树的每一个元素对应一个结点，包含数据项和若干指向其他节点的分支
- 结点的度：节点的子树个数
- 结点的层次：根结点的层次为1，根结点的孩子的层次为2，依次类推
- **结点的权**：对结点赋予的一个有着某种意义的数值
- **结点的带权路径长度**：从树根结点到该结点之间的路径长度与该结点权值的乘积

- 叶子结点（终端结点）：度为0的结点
- 分支结点（非终端结点）：度不为0的结点
- 孩子结点：若结点X有子树，则子树的根结点即为结点X的孩子结点
- 双亲结点：若结点X有孩子，则X即为孩子的双亲结点
- 兄弟结点：同一双亲的孩子结点间互称为兄弟结点
- 堂兄弟结点：在树中层次相同，但双亲不同的结点
- 子孙结点：结点X的孩子，以及这些孩子的孩子都是X的子孙结点
- 祖先结点：从根结点到结点X所经分支上的所有结点，都称为X的祖先结点

## 2 树的路径

路径：从树的双亲结点移动到其孩子结点和其孩子孙结点所经过的路线
- 路径长度：路径上的分支数目，路径中==边的个数==
- **树的路径长度**：指从树根到每一结点的路径长度之和
- **树的带权路径长度**：树中所有叶子结点的带权路径长度之和，通常记作： $$WPL = \sum_{k=1}^{n} W_k \cdot L_k$$
	- $k$个叶子结点的权值$W_k$
	- 第$k$个叶子结点的路径长度$L_k$

## 3 树的性质

- 性质1: $节点个数=总度数+1$
- 性质2: 度为$m$，第$i$层最多有$m^{i-1},(i>0)$个节点
- 性质3: 高为$h$，$m$叉树，至多有$\frac{m^{h}-1}{m-1}$个结点，至少有$h$个节点
	- 通过等比数列求和公式 $\frac{1-q^{h}}{1-q}$计算
- 性质4: $n$个结点，$m$叉树的最小高度${\log_{m}{(n(m-1)})}$

## 4 树的抽象数据类型

### 4.1 数据对象
数据对象 D 是具有相同特性的数据元素的集合。
*  若 D 为空集，则称为空树
*   否则，D 中存在一个特定的称为**根**的数据元素 **root**，且其余元素可分为 m (m > 0) 个互不相交的有限集合 T₁, T₂, ..., Tm，其中每一个集合本身又是一棵树，称为根的子树

### 4.2 数据关系

数据关系 R 定义了 D 中元素之间的层级关系：
*   若 D 为空，则 R 为空集。
*   若 D 非空，则关系 R 满足以下条件：
    *   树中仅有一个没有前驱的节点，称为根节点。
    *   除根节点外，其余每个节点有且仅有一个前驱（即该节点的双亲/父节点）。
    *   树中每个节点可以有零个或多个后继（即该节点的孩子/子节点）。
    *   节点间的层次关系体现了“一对多”的非线性结构。

### 4.3 基本操作
基本操作列表（P 代表 Position/Tree，e 代表 Element，T 代表 Tree）：

1.  **InitTree(&T)**
    *   **初始条件**：无。
    *   **操作结果**：构造一棵空树 T。
2.  **DestroyTree(&T)**
    *   **初始条件**：树 T 存在。
    *   **操作结果**：销毁树 T，释放其占用的内存空间。
3.  **CreateTree(&T, definition)**
    *   **初始条件**：definition 给出树的定义。
    *   **操作结果**：按 definition 构造树 T。
4.  **ClearTree(&T)**
    *   **初始条件**：树 T 存在。
    *   **操作结果**：将树 T 清为空树，释放其内部节点内存，但不销毁树结构本身。
5.  **TreeEmpty(T)**
    *   **初始条件**：树 T 存在。
    *   **操作结果**：若 T 为空树，返回 TRUE，否则返回 FALSE。
6.  **TreeDepth(T)**
    *   **初始条件**：树 T 存在。
    *   **操作结果**：返回树 T 的深度（或高度）。
7.  **Root(T)**
    *   **初始条件**：树 T 存在。
    *   **操作结果**：返回 T 的根节点。若 T 为空，返回特定值（如 NULL 或 "ERROR"）。
8.  **Value(T, cur_e)**
    *   **初始条件**：树 T 存在，cur_e 是 T 中某个节点。
    *   **操作结果**：返回 cur_e 的值。
9.  **Assign(T, cur_e, value)**
    *   **初始条件**：树 T 存在，cur_e 是 T 中某个节点。
    *   **操作结果**：将节点 cur_e 赋值为 value。
10. **Parent(T, cur_e)**
    *   **初始条件**：树 T 存在，cur_e 是 T 中某个节点。
    *   **操作结果**：若 cur_e 是 T 的非根节点，则返回它的双亲；否则返回“空”。
11. **LeftChild(T, cur_e)**
    *   **初始条件**：树 T 存在，cur_e 是 T 中某个节点。
    *   **操作结果**：若 cur_e 有左孩子（或最左边的孩子），则返回它；否则返回“空”。
12. **RightSibling(T, cur_e)**
    *   **初始条件**：树 T 存在，cur_e 是 T 中某个节点。
    *   **操作结果**：若 cur_e 有右兄弟（即下一个兄弟），则返回它；否则返回“空”。
13. **InsertChild(&T, p, i, c)**
    *   **初始条件**：树 T 存在，p 指向 T 中某个节点，1 ≤ i ≤ p 指出的度 + 1，非空树 c 与 T 不相交。
    *   **操作结果**：插入树 c 为 T 中 p 所指节点的第 i 棵子树。
14. **DeleteChild(&T, p, i)**
    *   **初始条件**：树 T 存在，p 指向 T 中某个节点，1 ≤ i ≤ p 指出的度。
    *   **操作结果**：删除 T 中 p 所指节点的第 i 棵子树。
15. **TraverseTree(T, Visit())**
    *   **初始条件**：树 T 存在，Visit 是对节点操作的应用函数。
    *   **操作结果**：按某种规则（如先序、后序、层序）对树 T 中的每个节点调用函数 Visit() 一次且仅一次。一旦 Visit() 失败，则操作失败。

### 4.4 树的遍历

遍历：从根结点出发，按照某种次序访问树中所有结点，使得每个结点被访问一次且仅被访问一次。
- 先根遍历（先序遍历）：若树不空，则先根访问树的根结点，然后依次先根遍历根的每棵子树

- 后根遍历（后序遍历）：若树不空，则按照从左到右的顺序依次后根遍历根的每棵子树，然后访问根结点


## 5 树的存储结构

树的存储：要求既要存储结点的数据元素，又要存储结点之间的逻辑关系。
- ==顺序==存储方式：双亲存储
- ==链式==存储方式：孩子存储、孩子兄弟存储

### 5.1 双亲表示法 顺序存储

双亲表示法（父节点表示法）：顺序存储，用一组连续的空间存储结点，每个节点包含一个==指示器==指向其父节点的位置下标
- 特点：父节点好找，子节点难找

![[计算机科学/数据结构/attachments/2026-01-21_204253.svg]]


```c
#define MAX_TREE_SIZE 100

typedef struct{
	TElemType data;
	int parent;
}PTNode;

typedef struct{
	struct PTNode nodes[MAX_TREE_SIZE];
	int root;
	int n;
}PTree;
```


### 5.2 孩子表示法 链式存储

**多重链表表示法**：固定指针域，每个节点有多个指针域，每个指针域指向一颗子树的根节点
- 每个节点的指针域个数等于==树的度==
- 一颗n个节点的度为$k$的树有$n(k-1)+1$个==空链域==

**孩子链表表示法**：指针域只有一个指针指向一个==孩子节点位置组成的链表==



![[计算机科学/数据结构/attachments/2026-01-21_212211.svg]]


```c
typedef struct CTNode{
	int child;
	struct CTNode*next;
}*ChildPtr;

typedef struct{
	TElemType data;
	ChildPtr firstchild;
}CTBox;

typedef struct{
	CTBox nodes[MAX_TREE_SIZE];
	int n,r;
}CTree;

```

### 5.3 孩子兄弟表示法 链式存储

孩子兄弟表示法（二叉树表示法、二叉链表表示法）：以二叉链表作为树的存储结构，链表中结点的两个链域分别指向该结点的第一个孩子和下一个兄弟结点。


![[计算机科学/数据结构/attachments/2026-01-21_213558.svg]]
```c
typedef struct CSNode{
	ElemType data;
	struct CSNode *firstchild,*nextsibling;
}CSNode,*CSTree;
```
