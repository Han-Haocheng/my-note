# 线索二叉树

线索二叉树：利用二叉树的空指针域（n+1）存放指向该结点在某种遍历序列中的前驱和后继结点的位置信息，是加上线索的二叉树，是一种物理结构
- 原因：默认存储结构只能找到左右孩子信息，不能得到任意序列的前驱和后继，叶子节点的前驱或后继
- 目的：提高二叉树的遍历效率

- 线索：指向节点的前驱和后继的指针
- 线索化：对二叉树以某种次序遍历使其变为线索二叉树的过程

- ltag：标识lchild指示的是左孩子(0)还是前驱节点(1)
- rtag：标识rchild指示的是右孩子(0)还是后继节点(1)

## 1 中序线索二叉树

中序线索二叉树：相当于为二叉树建立了一个==双向链表==，既可以从第一个结点起顺后驱进行遍历，也可以从最后一个结点起顺前驱进行遍历

![[计算机科学/数据结构/attachments/2026-01-24_204543.svg]]

![[计算机科学/数据结构/attachments/Pasted image 20260124205321.png]]

带头结点的中序线索二叉树：为了简化线索链表的遍历，仿照线性表的存储结构，在二叉树的线索链表上也添加了一个头结点，并对相关结点的数据域的取值进行了约定
1. 头结点的lchild域存放指向线索链表的==根结点的指针==
2. 头结点的rchild域存放指向中序序列的==最后一个结点的指针==
3. 中序遍历序列的第一个结点的lchild域指针指向头结点。
4. 中序遍历序列的最后一个结点的rchild域指针指向头结点。

## 2 二叉树的线索化

由于线索二叉树构造的实质是将二叉链表中的空指针改为指向前驱或后继的线索，而前驱或后继的信息只有在遍历时才能得到，因此线索化的过程即为==在遍历的过程中修改空指针的过程==
- 对二叉树按照不同的遍历次序进行线索化，可以得到不同的线索二叉树

```c
BiThrTree*pre=NULL;

void InThreading(BiThrTree p){
	if(!p)
		return;
	InThreading(p->lchild);
	if(p->lchild){
		p->LTag = 0;
	}else{
		p->LTag=1;
		p->lchild=pre;
	}
	
	if(pre->rhild){
		pre->RTag=0;
	}else{
		pre->RTag=1;
		pre->rchild=p;
	}	
	pre=p;
	
	InThreading(p->rchild);
}

void InOrderThreading(BiThrTree&Thrt,BiThrTree T){
	Thrt = new BiThrNode;
	Thrt->LTag = 0;
	Thrt->RTag=1;
	Thrt-rchild=Thrt;
	if(!T){
		Thrt->lchild=Thrt;
		return;
	}
	
	Thrt->lchild=T;pre=Thrt;
	InThreading(T);
	pre->rchild=Thrt;
	pre->RTag=1;
	Thrt->rchild=pre;
}
```

## 3 线索二叉树的

对于上述链表结构，中序线索链表T的中序遍历算法基本步骤如下：
(1) 令p=T->lchild；p指向线索链表的根结点。
(2) 若线索链表非空，则执行：
    ① 顺着p的左孩子指针找到最左下结点（中序遍历的第一个结点），并访问之。
    ② 若p所指结点的右孩子域中为线索，则p的右孩子结点为后继结点。循环执行p=p->rchild，并访问p所指结点（在此循环中，顺着后继线索访问二叉树中的结点）。
    ③ 一旦“线索”中断，则p所指结点的右孩子域为右孩子指针，p=p->rchild，即p指向右孩子结点，返回(2)继续执行，直至链表为空
