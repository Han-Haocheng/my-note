# B树


B树（Balanced-tree）：减少外存访问次数，一种适用于外部查找的树，其特点是插入、删除时易于保持平衡，外部查找效率高，适用于组织磁盘文件的文件系统



![[计算机科学/数据结构/attachments/Pasted image 20260211162609.png]]

## B树的性质

阶为$h$的B树，可以为空树，或者是一棵满足下列性质的$m$叉树。
1. 树中每个结点==至多有m棵子树==
2. 若根结点不是叶子结点，则**至少有两棵子树**
3. 除根结点之外，所有非叶子结点==至少有$\left\lceil  \frac{m}{2}  \right\rceil$棵子树==
4. 有$s$个子树的非叶子结点具有$s-1$个关键字
	- 所有的非叶子结点中包含下列信息$(n,A_{0},K_{1},A_{1},K_{2},\dots,K_{n},A_{n})$
	- 其中$n$为关键字个数
	- $K_{i}, (i=1,2,\dots,n)$为关键字，$K_{i}<K_{i+1}$
	- $A_{i},(i=0,1,2,\dots,n)$为指向子树根结点的指针
	- 且指针$A_{i-1}$所指子树中所有结点的关键字均小于$K_{i}$
	- $A_{n}$所指子树中所有结点的关键字均大于$K_{n}$
5. B树总是树高平衡的，所有的叶子结点都在同一层，且不包含任何关键字信息。通常叶子结点也被称为查找失败的结点

平衡：所有叶子结点在同一层，左边和右边分布均匀。
有序：每个结点中有序，且左子树小于根，根小于右子树。
多路：多棵子树（如果最多子树个数为m，则称m阶B-tree）


关键字个数$=$子树个数$-1$
- 根结点：子树有$[2,m]$个，关键字有$[1,m-1]$个
- 非根结点：子树有$\left[ \left\lceil  \frac{m}{2}  \right\rceil,m \right]$个，关键字有$\left[ \left\lceil  \frac{m}{2}  \right\rceil-1,m-1 \right]$个


## B树的查找


将key与根结点中的$k[i]$进行比较
1. 若$\text{key} = k[i]$，则查找成功
2. 若$\text{key} < k[1]$，则沿着指针$A_0$所指的子树继续查找
3. 若$k[i] < \text{key} < k[i+1]$，则沿着指针$A_i$所指的子树继续查找
4. 若$\text{key} > k[n]$，则沿着指针$A_n$所指的子树继续查找

### 查找效率

深度为h+1的B树至少包含的==结点数==为：$1 + 2 + 2\lceil m/2 \rceil + \dots + 2(\lceil m/2 \rceil)^{h-1}$

若m阶B树中具有N个关键字，则叶子结点（查找不成功的结点）为N+1个，由此有：$N + 1 \geq 2(\lceil m/2 \rceil)^{h-1}$，可以得到：$h \leq \log_{\lceil m/2 \rceil} \left( \frac{N + 1}{2} \right) + 1$

在B树上进行一次查找，需访问的结点个数不超过$\log_{\lceil m/2 \rceil} \left( \frac{N + 1}{2} \right) + 1$
- 时间复杂度$O(\log_m N)$

### B树的插入


先直找，进入最底层结点
- 有空位，直接插入。该结点中原关键字个数$n < m-1$，有序插入到该结点的合适位置；
- 无空位，分裂。即原关键字个数$n = m-1$；

插入后确保至多$m-1$个关键字。

## B树的删除

先找到待删关键字key所在结点
- 若key在最下层结点中
	- 结点中关键字个数 > $\lceil m/2 \rceil$-1，则直接删除
	- 结点中关键字个数 == $\lceil m/2 \rceil$-1，且兄弟... > $\lceil m/2 \rceil$-1，则借调
	- 结点和其兄弟的关键字个数都 == $\lceil m/2 \rceil$-1，则合并
- 若key不在最下层，为了“中序有序”
	- 用key的左子树中的最大码或右子树中的最小码k取代key，然后从该子树中删除k（k一定在最下层）
删除后确保至少$\lceil m/2 \rceil$-1个关键字

结点中关节字个数$>\left\lceil  \frac{m}{2}  \right\rceil-1$，则直接删除$12$，3阶B树

## B+树

B+树（B-Plus Tree）：是在B树基础上优化的变种，主要应用于数据库索引的数据结构，是一种高效的平衡查找树（效率比B树高）

## B+树性质

- 在B+树中，具有n个关键字的结点含有n棵子树（也即指向子树的指针数量）。
- 所有叶结点包含了全部关键字，即在非叶结点中出现的关键字也会出现在叶结点中，及指向含这些关键字记录的指针，而且叶子结点本身按照关键字自小而大顺序连接。
- 叶结点包含信息，所有非叶结点仅起到索引作用，非叶节点中的每个索引项只含有对应子树的最大（或最小）关键字和指向该子树的指针，不含有该关键字对应记录的存储地址，但在B树中每个关键字对应一个记录的存储地址。

### B+树的查找
![[计算机科学/数据结构/attachments/Pasted image 20260212173831.png]]

B+树有两个头指，**一是指向B+树的根结点**，可进行随机查找；**另一个是指向关键字最小的叶子结点**，所有叶子结点链接成一个单链表，可进行顺序查找。
