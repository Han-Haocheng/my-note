# 线性表

线性表：具有**相同特性**的数据元素的一个**有限序列**，数据元素之间的**关系是线性关系**$$(a_{1},a_{2},\dots ,a_{i-1},a_{i},a_{i+1},\dots,a_{n})$$
  - $a_{1}$：线性起点/起始节点
  - $a_{n}$：线性终点/终端节点
  - $a_{i-1}$是$a_{i}$的直接前趋
  - $a_{i+1}$是$a_{i}$的直接后继
  - $1,2,\dots,n$：下标，表示元素在线性表中的位置
  - $n$：表长/元素的总个数
  - $n=0$：空线性表

线性表逻辑特征

- 开始节点：有且只有一个、没有直接前趋、仅有一个直接后继
- 终端节点：有且仅有一个、没有直接后继、仅有一个直接前趋
- 内部节点：有且仅有一个直接前趋、有且仅有一个直接后继


存储密度

存储密度：节点数据本身占的存储量和整个节点结构中所占的存储量之比
- $存储密度=\frac{{节点数据本身占用的空间}}{节点占用空间总量}$


## 1 线性表的抽象数据类型

### 1.1 线性表的数据对象

- $D=\{ a_{i}|a_{i}\in \mathrm{Elemeset}, i=1,2,\dots,n\}$

### 1.2 线性表的数据关系

- $R=\{ <a_{i-1},a_{i}>|a_{i-1},a_{i}\in D,(i=2,3,\dots,n) \}$

### 1.3 线性表的基本操作


| 操作                    | 含义                            |
| --------------------- | ----------------------------- |
| `InitList(&L)`        | 构造一个空线性表L                     |
| `DestroyList(&L)`     | 销毁线性表L                        |
| `ClearList(&L)`       | 将线性表L重置为空表                    |
| `ListLength(L)`       | 返回线性表L中的元素个数                  |
| `GetElem(L,i,&e)`     | 用e返回线性表L中第i个数据元素的值            |
| `ListInsert(&L,i,e)`  | 在L的第i个位置插入新元素e<br>L的长度加一      |
| `ListDelete(&L,i,&e)` | 删除L的第i个元素<br>用e返回其值<br>L的长度减一 |

构造线性表：`InitList(&L)`

- 初始条件：无
- 操作结果：**构造**一个空的线性表

析构线性表：`DestroyList(&L)`

- 初始条件：线性表L已经存在
- 操作结果：**销毁**线性表L

清空线性表：`ClearList(&L)`

- 初始条件：线性表L已经存在
- 操作结果：将线性表L**重置为空表**

判断线性表为空：`ListEmpty(L)`

- 初始条件：线性表L已经存在
- 操作结果：判断线性表**是否为空**
  - TRUE：空表
  - FALSE：非空表

获取线性表长度：`ListLength(L)`

- 初始条件：线性表L已经存在
- 操作结果：返回线性表L中的**元素个数**

获取元素：`GetElem(L,i,&e)`

- 初始条件：线性表L已经存在
- 操作结果：用e返回线性表L中第i个数据元素的值

按要求获取元素：`LocateElem(L,e,compare())`

- 初始条件
	- 线性表L已经存在
	- compare是数据元素判定函数
	- 存在与e满足compare()的数据元素
- 操作结果：L中第1个符合条件的结果

获取元素前趋：`PriorElem(L,cur_e,&pre_e)`
- 初始条件
	- 线性表L已经存在
	- `cur_e`是L的元素
	- `cur_e`不是L的第一个元素
- 操作结果：用`pre_e`返回`cur_e`的前趋

获取元素后继：`NextElem(L,cur_e,&next_e)`
- 初始条件
	- 线性表L已经存在
	- `cur_e`是L的元素
	- `cur_e`不是L的最后一个元素
- 操作结果：用`next_e`返回`cur_e`的后继

插入新元素：`ListInsert(&L,i,e)`

- 初始条件
	- 线性表L已经存在
	- $1\leq i\leq ListLength(L)+1$
- 操作结果
	- 在L的第i个位置插入新的元素e
	- L的长度加一

删除元素：`ListDelete(&L,i,&e)`

- 初始条件
	- 线性表L已经存在
	- $1\leq i\leq ListLength(L)+1$
- 操作结果
	- 删除L的第i个元素
	- 用e返回其值
	- L的长度减一

遍历元素：`ListRraverse(&L,visited())`

- 初始条件：线性表L已经存在
- 操作结果：依次对线性表中每个元素调用`visited()`

## 2 线性表的存储

### 2.1 线性表顺序存储


![[计算机科学/数据结构/attachments/20250410190959.svg]]

线性表顺序存储结构（线性表顺序映射、顺序表）：按逻辑顺序依次存放在一组地址连续的存储单元中
- 把逻**辑上相邻**的数据元素存储在**物理相邻**的存储单元中的存储结构
- 存储结构：**连续的存储空间**，随机存取
- 基地址/起始位置：线性表第一个元素的存储位置

#### 2.1.1 顺序存储结构特点

- 每个元素占用$l$个存储单元
- 已知第$i$个元素的位置，则第$i+1$个元素满足：$LOC(a_{i+1})=LOC(a_{i})+l$
- 已知第1个元素的位置，则第i个元素满足：$LOC(a_{i})=LOC(a_{1})+(i-1)l$
- 任意元素均可随机存取

#### 2.1.2 顺序线性表实现

- 存取元素的方法：随机存取法
- 存储密度：1
- 插入和删除操作时间长

静态数组表示

```c
#define LIST_INIT_SIZE 100 //定义线性表的最大长度
typedef struct{
    ElemType data[LIST_INIT_SIZE]; //用静态数组存储数据元素
```

### 2.2 链式存储结构



![[计算机科学/数据结构/attachments/20250404173032.svg]]

**链表**：用一组物*理位置任意的存储单元*来存放线性表的数据元素，元素的*逻辑次序和物理次序不一定相同*，n个结点由指针链组成，n=0时为空表
- **结点**：数据元素的存储映像，由数据域和指针域两部分组成
- **头指针**：指向链表中第一个结点的指针
- **首元结点**：链表中存储第一个数据元素$a_{1}$的结点
- **头结点**：链表的首元结点之前附设的一个结点


#### 2.2.1 表示空表方式

- 无头结点：头指针为空时，表示空表
- 有头结点：头结点指针域为空时表示空表
	- 便于处理首元结点：处理链表时，第一个位置上的操作与其他位置上一致
	- 便于处理空表和非空表：无论链表是否为空，头结点都是指向非空指针，处理统一

#### 2.2.2 链表类型

- **单链表**：结点只有一个指针域
- **循环链表**：首尾相接的链表，表中的最后一个结点的指针域指向头结点，形成一个环形
	- 特点：从任意一结点出发均可以找到表中的其他结点
	- 判断结束：和头指针/尾指针相比
	- 操作头尾：使用*尾指针*可快速锁定头尾结点
- **双链表**：结点有两个指针域，分别指向其直接前驱和直接后继
	- 优点：查找任意节点的前驱节点的时间复杂度为$O(1)$
- **双循环链表**：头结点前驱指针指向尾结点，尾结点的后继指针指向头结点


| 类型      | 查找表头结点<br>（首元结点） | 查找表尾结点 | 插节点`*p`的<br>前驱节点 |
| ------- | :--------------: | :----: | :--------------: |
| 单链表     |      $O(1)$      | $O(n)$ |      $N/A$       |
| 头指针循环链表 |      $O(1)$      | $O(n)$ |      $O(n)$      |
| 尾指针循环链表 |      $O(1)$      | $O(1)$ |      $O(n)$      |
| 双向循环链表  |      $O(1)$      | $O(1)$ |      $O(1)$      |

#### 2.2.3 链式存储结构特点

- 存储位置：任意位置
- 顺序存取：访问时只能通过头指针进入链表，并依次顺序扫描其余结点

链式存储结构的优点
- 节点空间可以动态申请和释放
- 数据元素的逻辑次序靠节点的指针来完成，插入和删除时，不需要移动数据元素

链式存储结构的缺点
- 存储密度小：每个结点的指针域需额外占用存储空间。当每个节点的数据域所占字节不多时，指针域所占存储空间的比重显得很大
- 非随机存取结构：对任意节点的操作都要从头指针按照指针链查找，增加了算法的复杂度

### 2.3 顺序表和链表的比较


| 类型     | 顺序表                | 链表                   |
| ------ | ------------------ | -------------------- |
| 存储空间   | 预先分配               | 动态分配                 |
| 存储密度   | $=1$               | $<1$                 |
| 存取时间   | $O(1)$             | $O(n)$               |
| 插入删除时间 | $O(n)$             | $O(n)$               |
| 使用情况   | 表长变化不大<br>频繁随机访问元素 | 长度变化较大<br>频繁进行插入删除操作 |

有序顺序表合并：已知线性表 $La$ 和$Lb$ 中的数据元素按值非递减有序排列，现要求将 $La$ 和 $Lb$ 归并为一个新的线性表 $Lc$，且 $Lc$ 中的数据元素仍按值非递减有序排列。

- 空间复杂度：$O(m+n)$

```c
	pa = La.elem; pb = Lb.elem;
	Lc.length = LA.length + Lb.Length;
	Lc.elem = new ElemType[Lc.length];
	pc= LC.elem;
	pa_last = La.elem + La.length - 1;
	pb_last = Lb.elem + Lb.length - 1;
	while(pa<=pa_last && pb<=pb_last)
		*pc++ = *pa <= *pb ? *pa++ : *pb++

	while(pb <= pb_last)
```

```c
void MergeList_L(LinkList& La, LinkList& Lb, LinkList&Lc){
	pa=La->next;
	pb=Lb->next;
	pc=Lc=La
	while(pa&&pb){
		if(pa->data<=pb->data){
			pc->next=pa;
			pc=pa;
			pa=pa->next;
		}else{
			pc->next=pb;
			pc=pb;
			pb=>pb->next;
		}
	}

}
```

#### 3.1.8 查找元素

线性表的顺序查找
- 时间复杂度：$T(n)=O(n)$
- 平均判断次数：$$ASL=\sum_{i=1}^{n}P_{i}C_{i}=\frac{1}{n}\sum_{i=1}^{n}=\frac{n+1}{2}$$
  - $P_{i}$：第i个记录被查找的概率
  - $C_{i}$：找到第i个记录需要比较的次数
```C
int LOCateElem(SqList L,ElemType e){
    // 核心逻辑
    for(int i=0;i<L.length;i++)
        if(L.data[i]==e) return i+1;
    return 0;
}
```


单链表获取指定元素位置
- 时间复杂度：$O(n)$
- 空间复杂度：$O(1)$

```C
Status GetElem_L(LinkList L,int i, ElemType &e){
    int j=1;
    LinkList p=L->next;
    while(p&&j<i){
        p=p->next;
        j++;
    }

    e=p->data;
    return OK;
}
```

#### 3.1.9 插入元素

顺序表插入：移动元素的次数依赖于表的长度和插入的位置
- 时间复杂度：$T(n)=O(n)$
- 平均移动次数：$E_{ins}=\frac{1}{n+1}\sum_{i=1}^{n+1}(n-i+1)=\frac{n}{2}$

```c
Status ListInsert_Sq(SqList *&L,int i,ElemType e)
{
	int j;
    //检查非法情况
    if(i<1||i>L.length+1) 
	    return ERROR;
    if(L.length==MAX_SIZE) 
	    return ERROR;
    i--;
    
    // 核心逻辑
    for(j=L.length;j>=i;j--)
        L->data[j]=L->data[j-1];
    L->data[i-1]=e;
    L->length++;
    return OK;
}
```

## 4 线性表的查找

时间：顺序查找最差，折半查找最好，分块查找介于两者之间
空间：分块最大，需要增加索引数据的空间

查找算法的特点
- 顺序查找对表没有特殊要求
- 分块时数据块之间在物理上可不连续。所以可以达到插入、删除数据只涉及对应的块；另外，增加了索引的维护
- 折半查找要求表有序，所以若表的元素的插入与删除很频繁，维持表有序的工作量极大
- 在表不大时，一般直接使用顺序查找

### 4.1 顺序查找

顺序查找过程：从表的一端开始，依次将记录的关键字和给定值进行比较
- 若某个记录的关键字和给定值相等，则查找成功
- 若扫描整个表后，未找到关键字和给定值相等的记录，则查找失败


- 适用：线性表的顺序存储结构、链式存储结构。

#### 4.1.1 顺序查找算法分析
算法分析：假设每个记录的查找概率相等，则：$ASL=\sum_{i=1}^{n} P_i C_i=\frac{1}{n}\sum_{i=1}^{n}(n-i+1)=\frac{n+1}{2}$
- 优点：算法简单，对表的结构没有要求
- 缺点：n很大时，查找效率非常低

#### 4.1.2 顺序查找代码

```c
typedef struct {
    KeyType key;        // 关键字域
    InfoType otherinfo;  // 其他域
} ElemType;

// 顺序表的定义
typedef struct {
    ElemType *R;        // 存储空间基地址,0号单元留空
    int length;         // 当前长度
    KeyType key;
    Str.Ri0
} SSTable;

// 顺序查找算法
int Search_Seq (SSTable ST, int key) {
	if(ST.R==NULL||ST.lelngth==0)
		return;
	
    // 在顺序表ST中顺序查找其关键字等于key的数据元素
    // 若找到,则函数值为该元素在表中的位置,否则为0
    for (int i=ST.length; i>=1; --i)
        if (ST.R[i].key == key) return i;  // 从后往前找
    return 0;
}

```

### 4.2 分块查找


![[计算机科学/数据结构/attachments/Pasted image 20260204203650.png]]

分块查找（索引顺序查找）：顺序查找改进方法。
1. ==块内无序，块间有序==：将顺序表分成若干子表，要求每个子表中的数值都比后一块中数值小（子表内部未必有序）
2. 然后将各子表中的最大关键字构成一个索引表，表中还要包含每个子表的起始地址（头指针）

#### 4.2.1 分块查找过程
1. 对于待查值$key$，在索引表中按某种查找算法将$key$与各个子表的最大关键字如$k_i$，$k_{j}$进行比较
	- 若$k_{i} < key \leq k_{j}$，则$key$可能在$k_{j}$所对应的子表中
2. 在1所找到的子表中进行顺序查找
	- 若找到关键字与$key$相等的记录，则查找成功，返回该记录所在的位置
	- 否则查找失败，返回0

#### 4.2.2 分块查找算法分析

$ASL=L_b+L_w$
- 对索引表的平均查找长度$L_b$
- 在子表中查找元素的平均查找长度$L_w$
- 若将长度为$n$的表均匀地分为$b=n/s$块，每块有$s$个记录，并设表中每个记录地查找概率相等
	- 用顺序查找确定块的位置：$ASL=\frac{1}{b}\sum_{i=1}^{b}i+\frac{1}{s}\sum_{j=1}^{s}j=\frac{b+1}{2}+\frac{s+1}{2}=\frac{1}{2}\left(\frac{n}{s}+s\right)+1$
	- 用折半查找确定块的位置：$ASL=\log_2\left(\frac{n}{s}+1\right)-1+\frac{s+1}{2}\approx\log_2\left(\frac{n}{s}+1\right)+\frac{s}{2}$

### 4.3 二分查找

折半查找（二分查找）
- 适用条件：采用==顺序==存储结构的==有序==表

#### 4.3.1 二分查找描述
二分查找描述：在表ST中取位于中间的记录作为比较对象
1. 若中间记录的关键字与给定值相等，则查找成功；
2. 若中间记录的关键字大于给定值，则在中间记录的左半区继续查找；
3. 若中间记录的关键字小于给定值，则在中间记录的右半区继续查找
4. 不断重复上述过程，直到查找成功或者所查找的区域无记录，即查找失败

#### 4.3.2 二分查找过程
二分查找过程：设表长为$n$，$low$、$high$和$mid$分别指向待查记录所在区间的下界、上界和中间点，$key$为给定的待查值
1. 设置初始区间指针：$low=1$，$high=n$；
2. 若$low>high$，查找失败，返回查找失败信息；否则$mid = \left\lfloor \frac{low + high}{2} \right\rfloor$，并执行以下操作：
	1. 若`key<ST.elm[mid].key`，则`high=mid-1`；重新执行2
	2. 若`key> ST.elm[mid].key`，则`low=mid+1`；重新执行2
	3. 若`key=ST.elm[mid].key`，查找成功，返回记录在表中的位置

#### 4.3.3 二分查找代码


```c
int Search_Bin (SSTable ST, KeyType key) {
    // 在有序表中折半查找其关键字等于key的数据元素
    // 若找到,则函数值为该元素在表中的位置,否则为0
    low=1; high=ST.length; // 置查找区间初值
    while (low<=high) {
        mid=(low+high)/2;
        if (key == ST.R[mid].key) 
	        return mid;    // 找到待查元素
        else if (key<ST.R[mid].key) 
	        high=mid-1;  // 继续在前一子表进行查找
        else 
	        low=mid+1;   // 继续在后一子表进行查找
    }
    return 0; // 表中不存在待查元素
}
```

#### 4.3.5 折半查找算法分析

借助折半查找的判定树，求取平均查找长度，假定有序表的长度$n=2^h-1$，则判定树深度为$h=\log_2(n+1)$，假设表中每个记录的查找概率相等，则查找成功时折半查找的平均查找长度为：
$$ASL=\frac{1}{n}\sum_{j=1}^{h} j \cdot 2^{j-1} = \frac{n+1}{n} \log_2(n + 1) - 1 \approx \log_2(n + 1) - 1$$
