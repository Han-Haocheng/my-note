
# 主存储器

![[计算机科学/组成原理/attachments/2026-03-11_200042.svg]]

主存：机器指令直接操作的存储器，需要主存地址对其进行随机访问
- 存储体：由许多存储单元组成
- 存储单元：大小相同的最小存储位置，每一个位置都有唯一地址表示
- 存储字长：存储单元的长度

MDR 的长度与存储字长相等


## 存储器容量的扩展

### 线选法

线选法：**直接**通过 CPU 上的地址线对存储芯片进行连接扩展，每根线对应一个芯片，通常选中为 0 的芯片
- 优点：线路简单实现起来比较容易，不需要译码
- 缺点：地址是不连续

### 译码器片选法

译码器片选法：通过**译码器间接**连接存储芯片进行扩展，一般高位是片地址，低位是片内地址
- 优点：地址连续，不再要求 CPU 用于扩展的地址线同一时间只能有一条有效信号线，译码器已经进行帮助 CPU 做了片选的工作

### 映射对应的二进制地址码


系统程序地址：地址数量= $2^{11}=2K$

| $A15-A8$ | $A7-A0$  |
| -------- | -------- |
| 01100000 | 00000000 |
| 01100111 | 11111111 |

应用程序地址：地址数量= $2^{10}=1K$

| $A15-A8$ | $A7-A0$  |
| -------- | -------- |
| 01101000 | 00000000 |
| 01101011 | 11111111 |

### 确定芯片类型
- 系统程序：$2K×8位$ ROM 芯片×1
- 应用程序：$1K×4位$ RAM 芯片×2

### 分配地址线

分配地址线 ：根据地址范围，138 译码器只会输出 $Y_4$ 或 $Y_5$

- $A_{10}\sim A_0$ 接到 $2K×8位$ ROM 的地址线
- $A_9\sim A_0$ 接到 $1K×4位$ RAM 的地址线
- $A_{11}$ 接到 138 译码器的 A 端
- $A_{12}$ 接到 138 译码器的 B 端
- $A_{13}$ 接到 138 译码器的 C 端
- $A_{14}$ 接到 138 译码器 $G_1$ 端
- $A_{15}$ 接到 138 译码器 $G_{2A}$ 端
- $\overline{MREQ}$ 接到 138 译码器的 $G_{2B}$ 端

### 片选信号连接
- 138 译码器 $Y_4$ 连接到 $2K×8位$ ROM芯片上
- 138 译码器 $Y_5$ 与地址 $A_10$ 进行与运算后，片选到 $1K×4位$ RAM芯片上

### 数据线连接

- $2K×8位$ ROM 芯片接到数据 $D_7\sim D_0$
- 2 片 $1K×4位$ RAM 芯片分别接到 $D_3\sim D_0$ 和 $D_7\sim D_4$

### 高性能存储芯片

- SDRAM（同步 DRAM）：在系统时钟的控制下进行读出和写入，CPU 无需等待。 
- RDRAM：由 Rambus 开发，主要解决存储器带宽问题。
- 带 Cache 的 DRAM：在 DRAM 的芯片内集成了一个由 SRAM 组成的 Cache，有利于猝发式读取。


## 存储器的校验

存储器的校验：由于环境当中的电磁干扰，可能对存储单元所存储的信息造成干涉，导致数据失效。

合法代码集合

1. 普通存储 {000, 001, 010, 011, 100, 101, 110, 111} 检错 0 位、纠错 0 位
2. 奇偶存储 {000, 011, 101, 110} 检错 1 位、纠错 0 位
3. 三倍冗余 {000, 111}  检错 1 位、纠错 1 位
4. 四倍冗余 {0000, 1111} 检错 2 位、纠错 1 位
5. 五倍冗余 {00000, 11111} 检错 2 位、纠错 2 位

> 编码的检测能力和纠错能力与任意两组合法代码之间**二进制位**的**最少差异数**有关。

### 编码的最小距离

任意两组合法代码之间*二进制位*的*最少差异*，编码的纠错检错能力与编码的最小距离有关
- $L-1=D+C(D≥C)$
- 编码的最小距离 $L$
- 检错的位数 $D$
- 纠错的位数 $C$ 

### 汉明码的组成

![image.png](计算机科学/组成原理/attachments/4.2.6image.png)

汉明码：采用奇偶校验和分组校验，校验位放置于 $2^n$ 的位置
- 分组校验：按照特定规则进行分组，并在规则内添加一位校验位
- 汉明码的分组是一种非划分方式（组和组之间存在交叉）


汉明码的三要素
- 需添加：$2^k≥n+k+1$ 位检测
- 检测位位置：$2^i(i=0, 1, 2, ...)$
- 检测位取值：检测位的取值与该位所在的检测小组承担的奇偶校验任务有关。 

各检测为 $C_i$ 所承担的检测小组

| 检测位   | 组编号   | 包含位置 |
| ----- | ----- | --------------------------------------------------------------------------------------- |
| $C_1$ | $g_1$ | $01_{0001}, 03_{0011}, 05_{0101}, 07_{0111}, 09_{1001}, 11_{1011}, 13_{1101}, 15_{1111}$ |
| $C_2$ | $g_2$ | $02_{0010}, 03_{0011}, 06_{0110}, 07_{0111}, 10_{1010}, 11_{1011}, 14_{1110}, 15_{1111}$ |
| $C_4$ | $g_3$ | $04_{0100}, 05_{0101}, 06_{0110}, 07_{0111}, 12_{1100}, 13_{1101}, 14_{1110}, 15_{1111}$ |
| $C_8$ | $g_4$ | $08_{1000}, 09_{1001}, 10_{1010}, 11_{1011}, 12_{1100}, 13_{1101}, 14_{1110}, 15_{1111}$ |

- $g_i$ 小组独占 $2^{i-1}$ 位
- $g_i$ 和 $g_j$ 小组共占第 $2^{i-1}+2^{j-1}$ 位
- $g_i$、$g_j$ 和 $g_l$ 小组共占第 $2^{i-1}+2^{j-1}+2^{l-1}$ 位



举例：求 0101 按“偶校验”配置的汉明码

- 因为 n=4
- 根据 $2^k≥n+k+1$
- 得到 $k=3$
- 汉明码的排序如下

|二进制序号|1|2|3|4|5|6|7|
|-|-|-|-|-|-|-|-|
|名称| $C_1$ | $C_2$ |0| $C_4$ |1|0|1|
|汉明码|0|1|0|0|1|0|1|


纠错过程：形成新的检测位 $P_i$，其位数与增添的检测位有关，如增添 3 位(k=3)，新的检测位为 $P_4P_2P_1$。


举例：假如 k=3，数据为 0011，偶编码汉明码为 1000011，错误数据 90%为一个，$P_i$ 的取值为
- $P_1=1\oplus3\oplus5\oplus7=1\oplus0\oplus0\oplus1=0$
- $P_2=2\oplus3\oplus6\oplus7=0\oplus0\oplus1\oplus1=0$
- $P_4=4\oplus5\oplus6\oplus7=0\oplus0\oplus1\oplus1=0$
- 若结果都为 0，数据无措，若结果为 1，则数据在 $P_4P_2P_1$ 的位置出错
- 汉明码编码的最小距离：$L=D(2)+C(0)+1=3$
