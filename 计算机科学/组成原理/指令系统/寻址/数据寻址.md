# 数据寻址

`|操作码|寻址特征|形式地址A|`

- 形式地址：指令字中的地址
- 有效地址：操作数的真实地址

约定：指令字长=存储字长=机器字长

| 类型    | 含义                              | 访存  | 特点                                         |
| ----- | ------------------------------- | --- | ------------------------------------------ |
| 立即寻址  | A 是操作数本身                        | 0   | A 的位限制了立即数的范围                              |
| 直接寻址  | A是操作数效地址                        | 1   | - A 的位数限制该指令操作数的寻址范围<br>- 操作数地址不易修改，必须修改 A |
| 隐含寻址  | - 两个操作数<br>- A 是其中之一<br>- 另一个隐函 | 1   | 可减少指令字段长度                                  |
| 间接寻址  | - A 是操作数地址的地址                   | 2   | - 可扩大寻址范围<br>- 便于编址程序                      |
| 寄存器寻址 |                                 |     |                                            |

## 立即寻址

立即寻址：形式地址 A 就是操作数
- 格式 `|OP|#|A|`
	- 立即寻址特征\#
	- 立即数 A：可正可负、补码
- 特点
	- 执行时不需要再次访存
	- A 的位限制了立即数的范围

## 直接寻址
![[计算机科学/组成原理/attachments/Pasted image 20240301094827.png]]


直接寻址：形式地址就是有效地址 `EA=A`
- 格式 `|OP|#|A|`
- 特点
	- 访存 1 次
	- A 的位数限制该指令操作数的寻址范围
	- 操作数地址不易修改，必须修改 A

## 隐含寻址


隐含寻址：操作两个数，其中一个操作数地址隐含在操作码中，可减少指令字段长度
- 格式：`[OP][#][A]`

如 8086 指令集

- `MUL` 指令：被乘数隐含在 AX（16 位）或 AL（8 中
- `MOVS` 指令：源操作数的地址隐含在 SI 中，目的的操作数隐含在 DI 中

## 寄存器寻址

有效地址是寄存器编号，寄存器直接寻址（$EA=R_i$）
![20240301101331.png](计算机科学/组成原理/attachments/Pasted%20image%2020240301101331.png)

- 执行阶段补访存，只访问寄存器，执行速度块
- 寄存器个数有限，可缩短指令字长

## 寄存器间接寻址

操作数保存在内存单元，操作上的地址保存在寄存器中（`EA=(R_i)`）

![20240301101714.png](计算机科学/组成原理/attachments/Pasted%20image%2020240301101714.png)

- 执行阶段只需要进行一次访存
- 便于编制循环程序

## 基址寻址

`EA=(BR)+A`

- 可扩大寻址范围
- 有利于多道程序

采用专用寄存器作基址寄存器
- BR 为基址寄存器
- 内容由操作系统或管理程序确定
- BR 内容不变，只能修改形式地址 A

![20240301102128.png](计算机科学/组成原理/attachments/Pasted%20image%2020240301102128.png)



采用通用寄存器作基址寄存器
- R_0 作为基址寄存器
- 用户确定作为基址的寄存器
- 基址寄存器的内容由操作系统决定
- R_0 内容不变，形式地址 A 可变

![20240301102459.png](计算机科学/组成原理/attachments/Pasted%20image%2020240301102459.png)

## 变址寻址

`EA=(IX)+A`，IX 为变址寄存器(专用或通用)
![20240301102928.png](计算机科学/组成原理/attachments/Pasted%20image%2020240301102928.png)

- 可扩大寻址范围
- IX 的内容由用户给定
- 在程序 执行过程中 IX 内容可变，形式地址 A 不变
- 便于处理数组问题

## 相对寻址

`EA=(PC)+A`
A 是相对于当前指令的位移量（可正可负，补码）

![20240301104322.png](计算机科学/组成原理/attachments/Pasted%20image%2020240301104322.png)

- A 的位数决定操作数的寻址范围
- 程序浮动，广泛用于转移指令

## 堆栈寻址

### 堆栈的特点

堆栈

- 硬堆栈：多个寄存器
- 软堆栈：指定的存储空间

先进后出（一个入出口）栈顶地址由 SP 指出

### 堆栈寻址举例

- 进栈：`(SP)-1->SP`
![20240301105539.png](计算机科学/组成原理/attachments/Pasted%20image%2020240301105539.png)
- 出栈：`(SP)+1->SP`
![20240301105551.png](计算机科学/组成原理/attachments/Pasted%20image%2020240301105551.png)

### SP 的修改与主存编址方法有关

- 按字编址：sp 每次修改 1
- 按字节编址：SP 每次修改 存储字长的字节数

> 思考：采用如此多的寻址方式，应该是为了应对不同类型的程序运行，不同的寻址方式的特点和适用范围不同
