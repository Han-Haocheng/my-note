## 计算机软硬件

软硬件在逻辑上是等效的

以“乘法的实现”为例

|      | 硬件              | 软件                 |
| ---- | --------------- | ------------------ |
| 特性   | 硬件乘法器           | 软件模拟乘法             |
| 速度   | 快（1个时钟周期完成）     | 慢（需要多次循环和加法）       |
| 资源占用 | 需要专用电路（面积大、功耗高） | 无需额外硬件（节省芯片面积）     |
| 灵活性  | 固定功能（只能做乘法）     | 可通过编程修改算法（如优化速度）   |
|      | 电路实现            | 编程实现               |
|      | 性能好、成本高、不易拓展    | 性能低（相对于硬件）、成本低、易拓展 |



**硬件**：计算机的实体，如主机外设等

**软件**：由具有各类特殊功能的信息（程序）组成
- **系统软件**：用来管理整个计算机系统
	- 举例：语言处理程序、操作系统、服务性程序、数据库管理系统、网络软件
	- 语言处理程序：高级语言、汇编语言、机器语言
- **应用软件**：按任务需要编制成的各种程序

### 从程序语言角度

![[计算机科学/组成原理/attachments/Pasted image 20260213141414.png]]

| 特性   | 高级语言      | 汇编语言       | 机器语言      |
| ---- | --------- | ---------- | --------- |
| 抽象程度 | 高（隐藏硬件细节） | 中（符号化机器指令） | 低（纯二进制）   |
| 可读性  | 高（接近自然语言） | 中（需要硬件知识）  | 低（仅0/1序列） |
| 执行方式 | 需编译器/解释器  | 需汇编器       | CPU直接执行   |
| 开发效率 | 高         | 低          | 极低        |
| 应用场景 | 应用程序开发    | 驱动开发、性能优化  | 硬件控制      |

1. 虚拟机器M4(高级语言)：用编译程序翻译成汇编语言程序，抽象程度高
2. 虚拟机器M3(汇编语言)：用汇编程序翻译成机器语言程序
3. 虚拟机器M2(操作系统)：用机器语言程序解释操作系统
4. 实际机器M1(机器语言用)：微指令解释机器指令
5. 微程序机器M0(微指令系统)：由硬件直接执行为指令


## 计算机体系结构

计算机体系结构(抽象层)：程序员(机器语言程序员)所见到的计算机系统的属性(指令系统、数据类型、寻址技术、io机理)，概念性的结构与功能特性

## 计算机组成

计算机组成(实现层)：实现计算机体系结构所体现的属性(具体指令的实现)
