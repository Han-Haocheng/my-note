# 计算机网络体系结构

## 计算机网络体系结构的发展

#计算机/网络/结构 目标

两台计算机要互相传送文件需解决的问题
- 必须有一条传送数据的通路.
- 发起方必须激活通路。
- 要告诉网络如何识别接收方。
- 发起方要清楚对方是否已开机，且与网络连接正常。
- 发起方要清楚对方是否准备好接收和存储文件。
- 若文件格式不兼容，要完成格式的转换。
- 要处理各种差错和意外事故，保证收到正确的文件。

#计算机/网络/结构 发展

1. 最初的 ARPANET 设计时提出了分层的设计方法
	- 分层：将庞大而复杂的问题，转化为若干较小的局部问题。
2. 1974 年，IBM 按照分层的方法制定并提出了系统网络体系结构SNA(System Network Architecture).
3. 此后其他一些公司也相继推出了具有不同名称的体系结构
4. 国际标准化组织OISO提出的OSI/RM(Open SystemsInterconnection Reference Model)是使各种计算机在世界范围内互连成网的标准框架。
	- OSI/RM：是个抽象的概念
5. 1983年，形成了著名的ISO 7498 国际标准，即七层协议的体系结构。
	- OSI 试图达到一种理想境界：全球计算机网络都遵循这个统一标准，使全球的计算机将能够很方便地进行互连和交换数据。

## 层次网络结构

对于非常复杂的计算机网络协议，其结构应该是<font color="#c0504d">层次式</font>的
- 对等层交互
- 优点：各层之间是独立的、灵活性好、结构上可分割开、易于实现和维护、能促进标准化工作
- 常各层所要完成的功能：差错控制、流量控制、分段和重装、复用和分用、连接建立和释放

##  实体

#计算机/网络/实体 实体entity：任何可发送或接收信息的<font color="#c0504d">硬件或软件进程</font>

## 网络协议

#计算机/网络/协议 网络协议network protocol：为进行网络中的数据交换而建立的规则、标准或约定，控制<font color="#c0504d">两个对等</font>实体（或多个实体）进行通信的规则的集合，各个对等层之间传递数据时的各项规定
- 对等层peer layers：把数据（即数据单元加上控制信息）通过水平虚线直接传递给对方

#计算机/网络/协议 组成
- 语法：数据与控制信息的结构或格式，信号电平
- 语义：需要发出何种控制信息，完成何种动作以及做出何种响应，差错控制
- 同步（时序）：事件实现顺序的详细说明，速度匹配


#计算机/网络/协议 形式：文字描述、程序代码
### 协议数据单元

#计算机/网络/协议 协议数据单元PDU(Protocol Data Unit)：OSI 参考模型把对等层次之间传送的数据单位

## 服务

#计算机/网络/服务 服务Service：在协议的控制下，两个对等实体间的通信使得本层能够向上一层提供服务。要实现本层协议，还需要使用下面一层所提供的服务。

### 服务访问点

#计算机/网络/服务 服务访问点SAP(Service Access Point)：在同一系统中相邻两层的实体进行交互（即交换信息）的地方
- 逻辑接口

### 服务数据单元

#计算机/网络/服务 服务数据单元SDU(Service Data Unit)：OSI 把层与层之间交换的数据的单位
- 可以是多个 SDU 合成为一个 PDU
- 也可以是一个 SDU 划分为几个 PDU
