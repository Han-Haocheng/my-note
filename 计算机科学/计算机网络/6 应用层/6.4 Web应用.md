# Web应用

#计算机/网络/应用层/Web Web(World Wide Web)：网页以及网页间的相互连接


网页WebPage：由对象Object组成
- 对象Object：HTML文件、JPEG图片、视频文件、动态脚本
- 基本HTML文件：包含对其他对象引用的链接

对象寻址Addressing：通过统一资源定位器URL（Uniform Resoure Locator）^[RFC-1738]寻址
- URL格式：Scheme://host:port/path

## HTTP协议

![[计算机科学/计算机网络/6 应用层/attachments/Pasted image 20251020142029.png]]

#计算机/网络/应用层/Web 超文本传输协议HTTP（HyperText Transfer Protocol）：C/S架构，无状态协议，使用TCP传输服务，请求响应模式，默认端口为80
- 浏览器Brower：客户端进程，用于请求、接收、展示Web对象
- 网络服务WebServer：服务端进程，用于响应客户的请求、发送对象
- 无状态stateless：服务器不维护任何有关客户端过去所发请求的信息

HTTP协议流程
1. 服务器在80端口等待客户请求
2. 创建套接字Socket：浏览器发起到服务器的TCP连接
3. 服务器接收来自浏览器的TCP连接
4. 浏览器与服务器交换HTTP消息
5. 关闭TCP连接


## HTTP连接

#计算机/网络/应用层/Web/连接 HTTP连接：非持久性连接、持久性连接

### 非持久性连接
![[计算机科学/计算机网络/6 应用层/2025-10-20_144236.svg]]

#计算机/网络/应用层/Web/连接 非持久性连接Nonpersistent HTTP：每个TCP连接最多允许传输一个对象
- 响应时间=2RTT+文件发送时间
- 问题：每个对象需要2个RTT，操作系统需要为每个TCP连接开销资源overhead

![[计算机科学/计算机网络/6 应用层/attachments/Pasted image 20251020145201.png]]

### 持久性连接

#计算机/网络/应用层/Web/连接 持久性连接Persistent HTTP：每个TCP连接允许传输多个对象，发送响应后，服务器保持TCP连接打开，后续的HTTP消息可以通过这个连接发送
- 无流水pipelining：客户端只有收到前一个响应后才发送新的请求，每个被引用的对象耗时1个RTT
- 有流水：客户端只要遇到引用对象就发出请求，理想情况，所有的引用对象消耗1个RTT

## HTTP消息格式


#计算机/网络/应用层/Web/消息 HTTP消息格式：请求消息、响应消息

### 请求消息
![[计算机科学/计算机网络/6 应用层/2025-10-20_151030.svg]]

#计算机/网络/应用层/Web/消息 请求消息request：ASCII形式
- 请求行：客户端请求指令
	- GET：输入信息通过request行的URL字段上传
	- POST：网页经常需要填写表格form，在请求消息的消息体中上传客户端的输入
	- HEAD：请Server不要将所请求的对象放入响应消息中，测试用
	- PUT：将消息体中的文件上传到URL字段所指定的路径
	- DELETE：删除URL字段所指定的文件
- 头部行：客户端请求控制信息
	- 主机Host
	- 浏览器类型User-agent
	- 连接状态Connection
	- 接受语言Accept-language
- 消息体Entity body

```http
GET /somedir/page.html HTTP/1.1
Host: www.someschool.edu
User-agent: Mozilla/4.0
Connection: close
Accept-language:fr
```
### 响应消息

#计算机/网络/应用层/Web/消息 响应消息response：ASCII形式
- 状态行：协议、状态码、状态短语
- 头部行：服务端相应的控制信息
	- 连接状态Connection
	- 响应生成时间Date
	- 服务器类型Server
	- 上次修改时间Last-Modified
	- 内容长度Content-Length
	- 内容类型Content-Type
- 响应体

响应状态代码

| 状态码 | 状态短语                       | 含义        |
| --- | -------------------------- | --------- |
| 200 | OK                         | 响应成功      |
| 301 | Moved Permanently          | 已经永久移动    |
| 400 | Bad Request                | 错误请求      |
| 404 | Not Found                  | 无法找到      |
| 505 | HTTP Version Not Supported | HTTP版本不支持 |

```http
HTTP/1.1 200 OK
Connection: close
Date: Thu, 06 Aug 1998 12:00:15 GMT
Server: Apache/1.3.0 (Unix)
Last-Modified: mon, 22 Jun 1998 ...
Content-Length: 6821
Content-Type: text/html

data data data data data ...
```

## Cookie技术

很多应用需要服务器掌握客户端的状态


#计算机/网络/应用层/Web/技术 Cookie技术：某些网站为了辨别用户身份、进行session跟踪而储存在用户本地终端上的数据（通常经过加密）^[RFC-6265]


### Cookie组件

- HTTP响应消息的cookie头部行
- HTTP请求消息的cookie头部行
- 保存在客户端主机上的cookie文件，由浏览器管理
- Web服务器端的后台数据库

### Cookie原理

1. 客户端向服务器发送普通的请求
2. 服务器可以设置客户端的Cookie文件
3. 再次访问服务器时可以将Cookie信息包含进请求信息中
4. 服务器响应对应Cookie状态的数据
   
### Cookie作用

Cookie作用：身份认证、购物车、推荐、Web e-mail
- 问题：隐私问题


## Web缓存技术

![[计算机科学/计算机网络/6 应用层/attachments/Pasted image 20251020165100.png]]

#计算机/网络/应用层/Web/技术 Web缓存技术（代理服务器技术）：在不访问服务器的前提下满足客户端的HTTP请求
- 目的：缩短客户请求的响应时间、减少机构组织的流量、在大范围内实现有效的内容分发
- 缓存服务器：即是客户端也是服务器，通常由ISP架设

1. 用户设定浏览器通过缓存进行Web访问
2. 浏览器向缓存/代理服务器发送所有的HTTP请求
	- 如果所请求对象在缓存中，缓存返回对象
	- 否则，缓存服务器向原始服务器发送HTTP请求，获取对象，然后返回给客户端并保存该对象
3. 通过条件性GET同步缓存服务器和源服务器数据


条件性GET方法：如果缓存有最新版本，则不需要返回请求对象
- 缓存：在HTTP请求消息中声明所有版本的日期，`if-modified-since: <date>`
- 服务器：如果缓存是最新的，则响应不包含对象，`304 Not Modified`
